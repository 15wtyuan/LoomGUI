# v1c.4 Click 增强 + v1c 收尾 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Click 判定全面对齐 fgui（双击 / downTargets / ClickTest / Move 取消 / Canceled / CancelClick）+ `stopImmediatePropagation` + Stationary hover 跟随 + C# 路由测回尾，收尾 v1c。

**Architecture:** Click 判定全部在 core `loomgui_core::input::PointerState`（照 fgui `TouchInfo`），C# 仅路由/转发。时间复用已有 `loomgui_stage_tick(dt)`（C# 传 `Time.unscaledDeltaTime`，core `Stage::advance_time` 累积 `time_s`），clickCount 复用 `EventRecord` 的 pad 字节（20B 不变）。`Canceled` = `PointerKind::Canceled=3`，偏离 fgui quirk（隐式 CancelClick，reset clickCount）。

**Tech Stack:** Rust（loomgui_core + loomgui_ffi_c，csbindgen 1.9）、Unity 6.5 C#（LoomEventHandler/LoomInputCollector/LoomStage）、fgui 参考实现 `temp/FairyGUI-unity/Assets/Scripts/Core/Stage.cs`。

## Global Constraints

- **版本串**：`loomgui_version()` 返 `"v1c.4"`（T5 改）。
- **ABI 不增字节**：`PointerEvent` 16B、`EventRecord` 20B、`PointerKind` `#[repr(u8)]`。`EventRecord.pad:[u8;3]` → `click_count:u8` + `pad:[u8;2]`（20B 不变）。C# `LoomEvent` 加 `byte clickCount`（offset 5）。
- **fgui 阈值（固定像素，无缩放）**：mouse 10 / touch 50（per-axis `|dx|>t || |dy|>t`）、Move 取消硬编码 50、双击 0.35s。
- **Canceled 偏离**（spec §0.6，用户确认）：`PointerKind::Canceled` 置 `click_cancelled=true` → 发 Up、不发 Click、reset `click_count=1`+`last_click_time=0`。
- **双击 clickCount**：`1→2→1` 循环，统一核心 350ms 逻辑（不依赖 Unity `tapCount`）。
- **工作流**：本机编码。任何 Rust 改动（T1-T5）后必 `cargo build -p loomgui_ffi_c --release` 重编 + 拷 .dll 到 `loomgui_unity/Assets/Plugins/LoomGUI/` + commit .dll + .meta（家里机 PlayMode 才能验）。
- **Ponytail**：最短 diff，YAGNI。简化处标 `ponytail:` 注释。时间用 f32（千秒内精度足够 350ms 窗口）。
- **spec**：`docs/superpowers/specs/2026-06-24-v1c.4-click-design.md`（真相源，本 plan 不复述其论证）。

```rust
    /// click 目标判定（照 fgui ClickTest）。返 Click 应派发的节点；None=不产 Click。
    /// cancelled（Move>50/CancelClick/Canceled）→ None。位移 per-axis 超阈值 → None。
    /// 否则优先 down_targets[0]（按下叶，"still on stage"≈索引有效）；叶失效则沿当前 hit 祖先兜底。
    fn click_test(slot: &TouchSlot, scene: &Scene, current_hit: Option<NodeId>) -> Option<NodeId> {
        if slot.click_cancelled {
            return None;
        }
        let t = click_threshold(slot.touch_id);
        let dx = slot.last_pos.0 - slot.down_pos.0;
        let dy = slot.last_pos.1 - slot.down_pos.1;
        if dx.abs() > t || dy.abs() > t {
            return None;
        }
        if let Some(&leaf) = slot.down_targets.first() {
            if leaf.0 < scene.nodes.len() {
                return Some(leaf);
            }
        }
        let mut cur = current_hit;
        while let Some(id) = cur {
            if id.0 >= scene.nodes.len() {
                break;
            }
            if slot.down_targets.contains(&id) {
                return Some(id);
            }
            cur = scene.nodes[id.0].parent;
        }
        None
    }
```

- [ ] **Step 3: 加 bump_click_count（照 fgui End @1745-1755）**

`impl PointerState` 内加（time_s 作参数传，避免 &mut self 与 &mut slot 借用冲突）：

```rust
    /// 双击 clickCount 累进（照 fgui End：350ms + per-axis 位置 + 同键 → 1→2→1 循环）。
    /// 返回本次 click_count 并更新 slot 的 last_click_* 状态。
    fn bump_click_count(slot: &mut TouchSlot, button: u8, time_s: f32) -> u8 {
        let t = click_threshold(slot.touch_id);
        let within_time = time_s - slot.last_click_time < DOUBLE_CLICK_TIME;
        let within_pos = (slot.last_pos.0 - slot.last_click_pos.0).abs() < t
            && (slot.last_pos.1 - slot.last_click_pos.1).abs() < t;
        let same_button = slot.last_click_button == button;
        let count = if within_time && within_pos && same_button {
            if slot.click_count == 2 { 1 } else { slot.click_count + 1 }   // 1→2→1 循环
        } else {
            1
        };
        slot.click_count = count;
        slot.last_click_time = time_s;
        slot.last_click_pos = slot.last_pos;
        slot.last_click_button = button;
        count
    }
```
