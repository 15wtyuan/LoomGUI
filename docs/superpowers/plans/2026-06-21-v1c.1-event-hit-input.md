# v1c.1 事件/命中/输入最小交互闭环 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在现有渲染链之上加交互闭环——指针输入→命中→click/hover/leave 事件 + `:hover/:active/:disabled` 伪类重匹配 + `is_pointer_on_ui`，命中 v1 验收 #3（hover/active 反馈）/#5（UI 挡点击）。

**Architecture:** 核心新增 input/hit/rematch 三模块；打包器产动态规则表进 .pkg.bin（v2→v3）；FFI 加 set_input/borrow_events/is_pointer_on_ui/set_node_disabled 四函数（pull 模式）；Unity 加输入采集 + C# 侧 listener 派发。listener 在 C#（核心只算命中+产事件+伪类重匹配），绕开 IL2CPP 回调坑。

**Tech Stack:** Rust core（serde/bincode, 现有 taffy/ttf-parser/cssparser）+ csbindgen FFI + Unity 6.5 URP C# 后端。

## Global Constraints

- **零改 frame blob v3（FrameBlob.cs）/ MirrorPool / MaterialManager / TextRasterizer / shader**——v1c.1 不动渲染树 SOA。包格式改是 .pkg.bin（runtime 包）非 frame blob，两者独立（PKG_MAGIC=0x474B504C "LPKG" vs frame blob "LOOM"，两套 version 字段）。
- `measure_text` / `Stage::new` / `build_render_nodes` / `solve` 签名**不变**——命中/rematch 在 tick 内加步骤，不改渲染入口签名。
- **.pkg.bin `PKG_FORMAT_VERSION` 2→3，MIN=MAX=3**（旧 v2 拒）。`write_package` 签名加 `dynamic: &DynamicRuleSection` 参数（同 v1b.3 加 atlas 参数先例）。`read_package` 把 DynamicRuleSection 填进 `Scene.dynamic_rules`（返回元组不变）。
- **FFI 4 新函数全常驻（不 gate parse feature）**——输入/事件是 runtime 稳定入口。`PointerEvent`/`EventRecord`/`PointerState` 定义在 `loomgui_core::input`，core 常驻导出。`--no-default-features` 构建的 .dll 仍有这 4 函数（坑 21：删/加 pub API 必验 workspace）。
- 伪类重匹配**全量不缓存**（§5.3）——每帧 tick 内命中 diff 后无条件全量节点重匹配（仅动态规则子集）。500 节点微秒级。
- 命中用 `layout_rect` AABB + clip 门控，**不做 transform world_to_local**（transform defer v1d，v1c.1 无动画故无影响）。
- **disabled 仍参与命中但 active/click 抑制**（LoomGUI 有意偏离 fgui——支持 `:disabled` 伪类 hover 反馈；fgui `enabled=false` 完全跳过 hit-test）。
- **inline 路径不支持伪类**（dynamic_rules 空）——伪类重匹配测走包路径；命中/事件测可用 inline。
- 事件产出**确定性**：同输入序列产同事件序列（状态机无随机/时间依赖，click 阈值 ~10px 常量）。
- **事件回传拉取式**（borrow_events，同 borrow_frame pull 模式）——绕开 §14.2 IL2CPP 回调坑。listener 在 C# 侧显式 AddListener(nodeId, type, cb)，核心不持 C# 回调。
- 事件回调改状态走 FFI 命令式 API（set_node_disabled 等）回传核心，天然延下帧 solve，无反馈环（§15）。
- **v1c.1 事件无冒泡**——业务回调只收目标事件（RollOver/Out/Click 均目标直派）；冒泡 v1c.2 加。
- 命中逆等效绘制序读 `node.style.order`（ResolvedStyle 已有 order 字段），**不新加 Node.sorting_order**——避免重复字段 + rematch 同步风险。order 运行时不可变（动态规则不改 order）。
- Rust→C# 回调必须 static + `[MonoPInvokeCallback]`（§14.2）——v1c.1 选 pull 模式故不涉回调。
- 改 .pkg.bin 格式必重编 .dll（坑 10）+ bump version 级联所有手搓包测 fixture（坑 17）+ .meta 补齐（坑 13）。
- `cargo test --workspace` 必绿（坑 21：单 crate 绿 ≠ workspace 绿）。

```rust
use crate::scene::node::{Node, NodeKind};

/// 运行时版 compound 匹配（消费 Node 而非 ElementData，运行时无 ElementTree）。
/// 匹配 tag/classes/id（不含伪类状态——状态由 T6 match_element_with_state 门控）。
/// **依赖 T3 给 Node 加 classes/id 字段**——T2 先占位：tag 映射 div/button/img/span，
/// class/id 暂返回 false（T3 填 Node.classes/id 后改）。
pub fn compound_matches_node(c: &Compound, node: &Node) -> bool {
    if let Some(t) = &c.tag {
        let kind_tag = match &node.kind {
            NodeKind::Container => "div",
            NodeKind::Button => "button",
            NodeKind::Image { .. } => "img",
            NodeKind::Text { .. } => "span",
        };
        if kind_tag != t.as_str() {
            return false;
        }
    }
    if let Some(_id) = &c.id {
        // Node 暂无 id 字段——T3 加 Node.id 后填实。暂返回 false（保守不命中）。
        return false;
    }
    for _cls in &c.classes {
        // Node 暂无 classes 字段——T3 加 Node.classes 后填实。暂返回 false。
        return false;
    }
    true
}
```

```rust
pub fn compound_matches_node(c: &Compound, node: &Node) -> bool {
    if let Some(t) = &c.tag {
        let kind_tag = match &node.kind {
            NodeKind::Container => "div",
            NodeKind::Button => "button",

            return false;
        }
    }

            return false;
        }
    }
```

```rust
//! 命中测试（§10.1）：输入 design 坐标点 → 返回命中 NodeId。
//! 逆等效绘制序遍历（顶层优先），layout_rect AABB + clip 门控 + pointer-events。
//! v1c.1 不做 transform world_to_local（defer v1d，无动画故无影响）。

use crate::scene::node::{Node, NodeId, Rect, Scene};

/// 点是否在 Rect 内（含边界，design 坐标）。
fn point_in_rect(point: (f32, f32), r: Rect) -> bool {
    point.0 >= r.x && point.0 <= r.x + r.w && point.1 >= r.y && point.1 <= r.y + r.h
}

/// children 按 style.order 降序稳定排（大值=顶层在前；同 order 保 children 原序）。
fn effective_draw_order(scene: &Scene, parent: NodeId) -> Vec<NodeId> {
    let mut kids: Vec<NodeId> = scene.nodes[parent.0].children.clone();
    kids.sort_by_key(|&c| -scene.nodes[c.0].style.order);  // 负号=降序，stable 保原序
    kids
}

/// §10.1 命中测试。逆等效绘制序遍历，第一个命中即返回（顶层优先）。
pub fn hit_test(scene: &Scene, point: (f32, f32)) -> Option<NodeId> {
    // 从 roots 逐棵 DFS。多个 root 按顺序，后 root 顶层（与渲染序一致）。
    for &root in &scene.roots {
        if let Some(hit) = hit_subtree(scene, root, point) {
            return Some(hit);
        }
    }
    None
}

/// 递归测某子树。先测子（逆等效序，顶层先），子命中返回子的；子都不命中→自身 fallback。
fn hit_subtree(scene: &Scene, id: NodeId, point: (f32, f32)) -> Option<NodeId> {
    let node = &scene.nodes[id.0];
    // clip 门控：有 clip_rect 且点不在 clip 内 → 整个子树不命中
    if let Some(clip) = node.clip_rect {
        if !point_in_rect(point, clip) {
            return None;
        }
    }
    // 先测子（逆等效绘制序 = 顶层先）
    for &c in &effective_draw_order(scene, id) {
        if let Some(hit) = hit_subtree(scene, c, point) {
            return Some(hit);
        }
    }
    // 子都不命中 → 自身 fallback：touchable + AABB 含点
    if node.touchable && point_in_rect(point, node.layout_rect) {
        return Some(id);
    }
    None
}
```

```rust
    pub last_hovered: Option<NodeId>,
}

impl Default for PointerState {
    fn default() -> Self {
        Self {
            last_pos: (0.0, 0.0),
            is_down: false,
            down_node: None,
            down_pos: (0.0, 0.0),
            last_hovered: None,
        }
    }
}

impl PointerState {
    pub fn new() -> Self { Self::default() }

    /// 消费本帧输入事件 + 命中 → 产 EventRecord 序列。
    /// events 空时仍跑 hover diff（指针位置沿用 last_pos，§6.3）。
    pub fn process(&mut self, scene: &Scene, events: &[PointerEvent]) -> Vec<EventRecord> {
        let mut out: Vec<EventRecord> = Vec::new();
        // 更新 last_pos（Move/Down/Up 都更新；空 events 沿用）
        for ev in events {
            self.last_pos = (ev.x, ev.y);
            let hit = hit_test(scene, self.last_pos);
            match ev.kind {
                PointerKind::Move => {
                    if let Some(n) = hit {
                        out.push(EventRecord { node_id: n.0 as u32, event_type: EVT_MOVE, x: ev.x, y: ev.y });
                    }
                }
                PointerKind::Down => {
                    self.is_down = true;
                    self.down_pos = (ev.x, ev.y);
                    self.down_node = hit;
                    if let Some(n) = hit {
                        let node = &scene.nodes[n.0];
                        if !node.disabled {
                            node.active = true;  // 命中非 disabled → 设 active
                            out.push(EventRecord { node_id: n.0 as u32, event_type: EVT_DOWN, x: ev.x, y: ev.y });
                        }
                    }
                }
                PointerKind::Up => {
                    self.is_down = false;
                    if let Some(n) = hit {
                        let node = &scene.nodes[n.0];
                        if !node.disabled {
                            node.active = false;
                            out.push(EventRecord { node_id: n.0 as u32, event_type: EVT_UP, x: ev.x, y: ev.y });
                            // click 判定：down_node == hit 且位移 < 阈值
                            if self.down_node == Some(n) {
                                let dx = ev.x - self.down_pos.0;
                                let dy = ev.y - self.down_pos.1;
                                if (dx * dx + dy * dy).sqrt() <= CLICK_THRESHOLD_PX {
                                    out.push(EventRecord { node_id: n.0 as u32, event_type: EVT_CLICK, x: ev.x, y: ev.y });
                                }
                            }
                        }
                    }
                    self.down_node = None;
                }
            }
        }
        // hover diff（每帧，用 last_pos）——即使无 events 也跑（鼠标静止 hover 保持）
        let cur_hover = hit_test(scene, self.last_pos);
        if cur_hover != self.last_hovered {
            if let Some(old) = self.last_hovered {
                out.push(EventRecord { node_id: old.0 as u32, event_type: EVT_ROLL_OUT, x: self.last_pos.0, y: self.last_pos.1 });
            }
            if let Some(new) = cur_hover {
                out.push(EventRecord { node_id: new.0 as u32, event_type: EVT_ROLL_OVER, x: self.last_pos.0, y: self.last_pos.1 });
            }
            // 更新 Node.hovered
            if let Some(old) = self.last_hovered {
                scene.nodes[old.0].hovered = false;
            }
            if let Some(new) = cur_hover {
                scene.nodes[new.0].hovered = true;
            }
            self.last_hovered = cur_hover;
        }
        out
    }
}
```

```rust
use crate::parse::selector::{compound_matches_node, Combinator, Compound, ParsedSelector};
use crate::scene::node::{NodeId, Scene};
use crate::style::mapping::apply_decl;

/// 判定 compound 是否匹配 node + 伪类状态门。
/// pseudo_hover → node.hovered；pseudo_active → node.active；pseudo_disabled → node.disabled。
fn compound_matches_with_state(c: &Compound, node_id: NodeId, scene: &Scene) -> bool {
    let node = &scene.nodes[node_id.0];
    if c.pseudo_hover && !node.hovered { return false; }
    if c.pseudo_active && !node.active { return false; }
    if c.pseudo_disabled && !node.disabled { return false; }
    compound_matches_node(c, node)
}

/// 完整后代链匹配（从右往左，复用 selector.rs matches 算法，消费 Node/Scene）。
pub fn match_element_with_state(sel: &ParsedSelector, node_id: NodeId, scene: &Scene) -> bool {
    let comps = &sel.compound;
    if comps.is_empty() { return false; }
    let last = &comps[comps.len() - 1];
    if !compound_matches_with_state(last, node_id, scene) { return false; }
    if comps.len() == 1 { return true; }
    match_chain_with_state(comps, comps.len() - 1, node_id, scene)
}

/// 递归匹配 comps[0..end_idx] 在 start_el 的祖先链上（同 selector.rs match_compound_chain）。
fn match_chain_with_state(comps: &[Compound], end_idx: usize, start_el: NodeId, scene: &Scene) -> bool {
    if end_idx == 0 { return true; }
    let target_comp = &comps[end_idx - 1];
    let combinator = comps[end_idx].combinator;
    match combinator {
        Combinator::Child => {
            match scene.nodes[start_el.0].parent {
                Some(parent) => {
                    compound_matches_with_state(target_comp, parent, scene)
                        && match_chain_with_state(comps, end_idx - 1, parent, scene)
                }
                None => false,
            }
        }
        Combinator::Descendant => {
            let mut cur = scene.nodes[start_el.0].parent;
            while let Some(ancestor) = cur {
                if compound_matches_with_state(target_comp, ancestor, scene) {
                    if match_chain_with_state(comps, end_idx - 1, ancestor, scene) {
                        return true;
                    }
                }
                cur = scene.nodes[ancestor.0].parent;
            }
            false
        }
    }
}
```

```rust
/// 全量节点重匹配（仅动态规则子集，§4.1）。
/// 每节点从 base_style 重叠命中的动态规则（specificity 升序）→ 写 Node.style。
/// 返回是否有任何节点 layout 字段变（v1c.1 solve 每帧全量，返回值供观测/测试）。
pub fn rematch_pseudo_classes(scene: &mut Scene) -> bool {
    let mut any_layout_dirty = false;
    // 收集规则 + specificity（避免每节点重算 parse）
    let rules_with_spec: Vec<(u32, u32, u32, &DynamicRule)> = scene.dynamic_rules.rules.iter()
        .map(|r| (r.selector.specificity.0, r.selector.specificity.1, r.selector.specificity.2, r))
        .collect();
    for i in 0..scene.nodes.len() {
        let node_id = NodeId(i);
        // 从 base_style 重起
        let mut new_style = scene.nodes[i].base_style.clone();
        // 收集命中规则，按 specificity 升序排（高 specificity 后 apply 胜出）
        let mut matched: Vec<(u32, u32, u32, &DynamicRule)> = Vec::new();
        for r in &rules_with_spec {
            if match_element_with_state(&r.3.selector, node_id, scene) {
                matched.push(*r);
            }
        }
        matched.sort_by_key(|r| (r.0, r.1, r.2));  // specificity 升序
        for (_, _, _, r) in &matched {
            for decl in &r.declarations {
                apply_decl(&mut new_style, &decl.prop, &decl.value);
            }
        }
        // 对比 layout 字段是否变（taffy_style 整体比较 + order）
        let old = &scene.nodes[i].style;
        let layout_changed = new_style.taffy_style != old.taffy_style || new_style.order != old.order;
```

```rust
/// 注入本帧指针事件（扁平 PointerEvent 数组）。tick 前调。
/// null/len=0 = 本帧无输入事件（hover diff 仍跑，指针位置沿用上帧）。
#[no_mangle]
pub extern "C" fn loomgui_stage_set_input(
    h: *mut StageHandle,
    events: *const PointerEvent,
    len: usize,
) {
    if h.is_null() { return; }
    let sh = unsafe { &mut *h };
    if events.is_null() || len == 0 {
        sh.stage.set_input(&[]);
        return;
    }
    let evs = unsafe { std::slice::from_raw_parts(events, len) };
    sh.stage.set_input(evs);
}

/// 拉取本帧事件 SOA（pull，同 borrow_frame）。返 EventRecord[] as_ptr + len。
/// null 句柄或未 tick → null + len=0。指针下 tick 失效。
#[no_mangle]
pub extern "C" fn loomgui_stage_borrow_events(
    h: *mut StageHandle,
    out_len: *mut usize,
) -> *const u8 {
    if h.is_null() {
        if !out_len.is_null() { unsafe { *out_len = 0; } }
        return std::ptr::null();
    }
    let sh = unsafe { &*h };
    let events = sh.stage.last_events();
    if events.is_empty() {
        if !out_len.is_null() { unsafe { *out_len = 0; } }
        return std::ptr::null();
    }
    if !out_len.is_null() { unsafe { *out_len = events.len(); } }
    events.as_ptr() as *const u8
}
```

```csharp
using System;
using System.Collections.Generic;
using System.Runtime.InteropServices;

namespace LoomGUI
{
    public enum EventType : byte { Down = 0, Up = 1, Move = 2, Click = 3, RollOver = 4, RollOut = 5 }

    [StructLayout(LayoutKind.Sequential)]
    public struct LoomEvent
    {
        public uint nodeId;
        public EventType type;
        public float x;
        public float y;
    }

    /// C# 侧 listener 表 + 派发（对齐 fgui listener 在 C# 域，核心只产事件）。
    public class LoomEventHandler
    {
        readonly Dictionary<uint, Dictionary<EventType, Action<LoomEvent>>> _listeners = new();

        public void AddListener(uint nodeId, EventType type, Action<LoomEvent> cb)
        {
            if (!_listeners.TryGetValue(nodeId, out var byType))
                _listeners[nodeId] = byType = new Dictionary<EventType, Action<LoomEvent>>();
            byType[type] = cb;  // v1c.1 单 callback/type（覆盖；v1c.2 多 callback 用 List）
        }

        public void RemoveListener(uint nodeId, EventType type)
        {
            if (_listeners.TryGetValue(nodeId, out var byType))
                byType.Remove(type);
        }

        /// 读 EventRecord[] buffer → 逐条查表派发。ptr = loomgui_stage_borrow_events 返回。
        public void DispatchPending(IntPtr ptr, int count)
        {
            if (ptr == IntPtr.Zero || count == 0) return;
            int recSize = Marshal.SizeOf<LoomEvent>();
            for (int i = 0; i < count; i++)
            {
                var evt = Marshal.PtrToStructure<LoomEvent>(ptr + i * recSize);
                if (_listeners.TryGetValue(evt.nodeId, out var byType) && byType.TryGetValue(evt.type, out var cb))
                    cb(evt);
            }
        }
    }
```
