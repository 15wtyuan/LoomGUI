# v1.1 background-image Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 div/button 容器支持 `background-image:url(x)` + `background-size:cover|contain|100%`，复用现有 `<img>` 图集采样链，零新 payload 变体/shader/blob 列。

**Architecture:** ResolvedStyle 加 `background_image: Option<String>` + `background_size: BackgroundSize` 两字段；mapping.rs 解析 url()/size；loomgui_pkg 收集链额外扫 style.background_image 进同 atlas；render/mod.rs Container 分支条件产带图 quad（复用 `NodePayload::Mesh`，texture 从 0 改实际 tex_id + fit_uv 算 UV）。FFI/Unity 零改。

**Tech Stack:** Rust（loomgui_core + loomgui_pkg，taffy/cssparser/scraper/image）+ Unity C#（零改）+ HTML/CSS showcase。

## Global Constraints

- **围栏子集**：`background-image:url(x)`/`url("x")`/`url('x')` + `background-size:cover|contain|100%`。砍 `background-position`/`background-repeat`/`auto`/多背景（v1-scope §2）。
- **默认 background-size = Stretch**（`100%` 语义；未设时默认 Stretch，非 CSS auto）。
- **节点范围**：仅 div/button 容器；span 文本节点不画背景图（Text 节点不改）。
- **色图共存**：background-color + background-image 共存（CSS 标准），shader `mainTexture × vertexColor` 自然叠加。
- **url 走 atlas**：与 `<img>` 同一条 TextureRegistry + 打包器收集链；只支持 atlas 相对路径，不支持 data URI/绝对路径。
- **Rust build + commit + push 在本机**；Unity PlayMode 验收在家里机（改 Rust 后必重编+commit .dll）。
- **main 直推**。
- **Ponytail 全开**：最短差异，YAGNI，无未请求抽象。
- **不建新 payload 变体 / 不改 shader / 不改 FFI blob SOA / 不改 NodeAnim**。

```rust
/// 解析 CSS `background-image: url(...)` 值，提取括号内路径（去可选引号 + 首尾空格）。
/// 支持 `url(x)` / `url("x")` / `url('x')` / `url( x )`。非 url() 格式或空 → None。
pub fn parse_url(value: &str) -> Option<String> {
    let v = value.trim();
    let inner = v.strip_prefix("url(")?.strip_suffix(")")?;
    let inner = inner.trim();
    let len = inner.len();
    if len == 0 { return None; }
    // 去首尾配对引号
    let path = if len >= 2 && (inner.starts_with('"') && inner.ends_with('"'))
        || (inner.starts_with('\'') && inner.ends_with('\''))
    {
        &inner[1..len - 1]
    } else {
        inner
    };
    let path = path.trim();
    if path.is_empty() { None } else { Some(path.to_string()) }
}
```

在 `render/mod.rs` 的 `build_render_nodes` 函数之后（约第 204 行后）、`resolve_lp` 之前加：

```rust
/// 按 background-size 算 rect → atlas UV 映射。
///
/// cover/contain 用统一 span 公式，区别在缩放比 s 取 max（cover）/ min（contain）：
/// - Cover：s=max，UV 跨度 ≤ 子区（内收），取子区中央可见部分，图边缘被裁。
/// - Contain：s=min，UV 跨度 ≥ 子区（外扩），rect 边缘采样落到子区外 → clamp 到图边缘
///   透明像素 → 留白透出 background-color。
/// - Stretch：整子区拉伸填满。
fn fit_uv(size: crate::style::resolved::BackgroundSize, rect: &Rect, meta: &TexMeta) -> ([f32; 2], [f32; 2]) {
    use crate::style::resolved::BackgroundSize;
    let (rw, rh) = (rect.w, rect.h);
    let (iw, ih) = (meta.width as f32, meta.height as f32);
    let (uv_min, uv_max) = (meta.uv_min, meta.uv_max);
    if iw <= 0.0 || ih <= 0.0 { return (uv_min, uv_max); }
    let (auw, auh) = (uv_max[0] - uv_min[0], uv_max[1] - uv_min[1]);
    let (cu, cv) = ((uv_min[0] + uv_max[0]) / 2.0, (uv_min[1] + uv_max[1]) / 2.0);
    match size {
        BackgroundSize::Stretch => (uv_min, uv_max),
        BackgroundSize::Cover | BackgroundSize::Contain => {
            let s = match size {
                BackgroundSize::Cover => (rw / iw).max(rh / ih),
                BackgroundSize::Contain => (rw / iw).min(rh / ih),
                _ => unreachable!(),
            };
            let u_span = auw * rw / (iw * s);
            let v_span = auh * rh / (ih * s);
            ([cu - u_span / 2.0, cv - v_span / 2.0],
             [cu + u_span / 2.0, cv + v_span / 2.0])
        }
    }
}
```
