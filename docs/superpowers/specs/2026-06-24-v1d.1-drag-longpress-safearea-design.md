# v1d.1 设计：拖拽 + 长按 + safe-area

> 范围来源：`docs/roadmap/v1d-plan.md` v1d.1（覆盖 defer #1/#2/#11/#12）。
> fgui 对照（已核实源码）：GObject.cs:2001-2138（drag）、UIConfig.cs:131/136（阈值）、LongPressGesture.cs（长按）。
> 两机串行：Rust 改本机 cargo test + 重编 .dll commit；C# 家里机验。

## 0. 决策（已确认）

1. **检测放 core**（all-core）：drag + longpress 都在 `input.rs` 状态机检测，emit 事件，C# 只路由。理由：drag 要和 `click_cancelled` 耦合（drag-start 必取消 click），放 core 最干净；longpress 放 core 只用 tick 计时，与 click/hover/active 既有 pattern 一致。备选 fgui 式 C# 手势类否决（drag-click 耦合跨 FFI 别扭）。
2. **drag 只发事件，不跟手**：emit DragStart/Move/End（位置/delta），**不自动移动节点**。跟手（对象跟手指）依赖 transform.translate（v1d.3）或 `position:absolute`（v1.x）——LoomGUI 节点是 CSS/taffy 布局驱动，位置是 layout 输出非自由输入，现在塞临时偏移会被 v1d.3 吞掉。**跟手留 v1d.3**。
3. **longpress 与 click 独立**（照 fgui）：longpress 不自动取消 click；业务要消费调既有 `CancelClick`。
4. **longpress universal**（不加 DSL flag）：任何节点按住 1.5s 触发；无 listener 时 C# 跳过（零成本）。drag 反之 **opt-in**（需 `draggable`，否则会破坏 click 容忍——见 §3）。
5. **safe-area 根 letterbox**：LoomStage 用 `Screen.safeArea` 做 shrink-to-fit+居中，内容自动避刘海。纯 Unity 侧，**core 零改**。CSS `env(safe-area-inset-*)` defer。
6. **机制/阈值镜像 fgui**：drag 阈值 mouse 2 / touch 10（per-axis OR）；longpress trigger 1.5s、半径 50px。

## 1. 架构

事件流不变（`PointerEvent[] → core process() → EventRecord[] → FFI → C# DispatchPending`），只加事件类型 + per-slot 状态：

```
Down  → 记 drag_target(最近 draggable 祖先) + drag_testing；记 down_time
Move  → 超 drag 阈值 → DragStart(cancel click) → 后续 DragMove
       (并行：超 50px → longpress_cancelled)
tick  → is_down 槽 time_s-down_time≥1.5s 且未取消 → LongPress(一次)
Up    → 若 dragging → DragEnd
```

drag/longpress 状态都在 `TouchSlot`（已有），不新增状态机类。

## 2. core 契约

### 2.1 Node（scene/node.rs）
- `+ draggable: bool`（默认 false）。parse 读 HTML `draggable="true"` 属性（AI 原生，照 `<img draggable>`）。pkg.bin round-trip（§6）。
- `disabled`/`touchable` 既有：disabled 节点不发起 drag/longpress（§3.4）；`touchable=false` 已被 hit_test 排除（不会成 down_node）。

### 2.2 TouchSlot（input.rs）新增字段
```
drag_testing: bool        // Down 在 draggable 链上置 true
dragging: bool            // DragStart 后置 true
drag_target: Option<NodeId>  // Down 时=down_targets 中最近 draggable；None 则无 drag
down_time: f32            // Down 时=time_s（longpress 用）
longpress_fired: bool     // 触发后置 true（本 press 不再发）
longpress_cancelled: bool // 位移>50px 置 true（本 press 不再发）
```

### 2.3 常量（input.rs）
```
DRAG_THRESHOLD_MOUSE: f32 = 2.0    // = fgui clickDragSensitivity
DRAG_THRESHOLD_TOUCH: f32 = 10.0   // = fgui touchDragSensitivity
LONGPRESS_TRIGGER: f32 = 1.5       // = fgui LongPressGesture.TRIGGER（秒）
LONGPRESS_RADIUS: f32 = 50.0       // = fgui holdRangeRadius（与 MOVE_CANCEL_PX 同值，独立常量明义）
```

### 2.4 事件类型（EventRecord.event_type，u8，不增字节）
```
EVT_DRAG_START  = 6
EVT_DRAG_MOVE   = 7
EVT_DRAG_END    = 8
EVT_LONG_PRESS  = 9
```
EventRecord 仍 20B（`node_id, event_type, click_count, pad[2], touch_id, x, y`），drag/longpress 复用 x/y=指针位（delta 业务自算）、click_count=0。

## 3. drag 状态机（input.rs process）

### 3.1 Down
- 照既有：置 `is_down/down_pos/down_node/down_targets`。
- **新增**：`drag_target = down_targets.iter().find(|n| scene[*n].draggable)`（最近 draggable 祖先，含 down_node）；`drag_testing = drag_target.is_some()`；`dragging=false`。

### 3.2 Move（既有 arm 扩展）
- 既有：`is_down && per-axis>50 → click_cancelled=true`。
- **新增 longpress 取消**：同处置 `longpress_cancelled=true`（位移>50 即取消长按）。
- **新增 drag**：
  - 若 `drag_testing && !dragging && drag_target.is_some()` 且 per-axis `|dx|>t || |dy|>t`（t=mouse?2:touch10）→ `dragging=true; click_cancelled=true;` emit `EVT_DRAG_START(drag_target)`。
  - 若 `dragging` → emit `EVT_DRAG_MOVE(drag_target)`。
- 既有 EVT_MOVE（touch_monitors）不动；drag 事件独立 emit。

### 3.3 Up / Canceled（既有 arm 扩展）
- **新增**：若 `dragging` → emit `EVT_DRAG_END(drag_target)`（Canceled 也发 DragEnd，照 fgui onTouchEnd）。
- 清 drag 状态：`drag_testing=dragging=false; drag_target=None`。
- 既有 click_test/bump_click_count 不动（drag 已置 click_cancelled，click 自然不发）。

### 3.4 抑制
- down_node `disabled` → drag_target 视同 None（不发 drag；照 click 抑制）。
- `drag_target` 走 down_targets 链（叶子+祖先），支持「draggable 容器拖整棵子树」。

### 3.5 路由
- DragStart/Move/End 全部 `drag_target` 起 **bubble**（照 Down/Up/Click，祖先可接，为 v1d.5 ScrollPane 服务）。
- 无需 touch_monitor：drag_target 固定存于 slot，后续 Move/Up 恒指向它（即便手指离开对象）。

## 4. longpress（input.rs，tick 驱动）

### 4.1 Down
- `down_time = time_s; longpress_fired=false; longpress_cancelled=false`。

### 4.2 Move
- `is_down && per-axis>50 → longpress_cancelled=true`（§3.2 同处）。

### 4.3 tick 检查（process 头部，每帧跑——照 v1c.4 stationary hover pattern，空事件也跑）
- 遍历 is_down 槽：若 `down_node.is_some() && !longpress_fired && !longpress_cancelled && time_s - down_time >= LONGPRESS_TRIGGER` → emit `EVT_LONG_PRESS(down_node)`；`longpress_fired=true`。
- down_node `disabled` → 跳过（不发）。

### 4.4 Up
- 无特殊（fired 与否业务已知；click 独立发）。

### 4.5 路由
- `down_node` 起 bubble。universal（任何节点按住即得，listener 自愿）。

## 5. safe-area（loomgui_unity LoomStage.cs，纯 Unity 侧）

### 5.1 ConfigureTransforms 改造
- 新增 `[SerializeField] bool _safeArea = true;`。
- `_safeArea=true` 时：用 `Screen.safeArea`（x/y/width/height，像素，左下原点）替代全屏：
  - `sf = min(safeArea.width / designW, safeArea.height / designH)`（shrink-to-fit 到 safe 区）
  - `localPosition = (safeArea.center - screenCenter)` 对齐（design 中心↔safe 中心；y-flip 在 localScale=-sf 不变）
  - **正交相机 orthoSize 不变**（仍=屏高/2，覆盖全屏）；root transform 把 design 映射进 safe 区，safe 区外留 letterbox 空白。公式精确值在 plan 落。
  - 无刘海屏 `safeArea==全屏` → 行为与现状一致（零回归）；默认 `_safeArea=true` 安全。
- `_safeArea=false` → 现有全屏行为（回归）。

### 5.2 输入对齐（关键协调点）
- `LoomInputCollector` 的 screen→design 映射**必须用与 ConfigureTransforms 相同的 sf + offset**，否则触摸偏移。
- 实现：抽出公共「design↔screen 变换」计算（safe 区 + shrink-to-fit），ConfigureTransforms 和 Collect 共用。plan 落实。

### 5.3 core 零改
- safe-area 完全在 Unity 侧（root transform + 输入映射），core 不涉。

## 6. FFI / ABI / pkg

### 6.1 FFI（loomgui_ffi_c）
- 无新 FFI 函数（drag/longpress 走既有 `borrow_events` 通道）。
- version 串 `v1d.1`。
- ABI 不增字节：EventRecord 20B（新事件类型用 event_type 空位 6-9）。

### 6.2 pkg.bin（Node.draggable round-trip）
- NodeBlock 加 `draggable`：**优先用现有 flags 字节的空闲 bit**（照 v1c.1 touchable/disabled 存法），避免 formatVersion bump；若无空闲 bit → +1 byte + formatVersion v3→v4。plan 读当前 NodeBlock 布局定。
- inline 路径（parse）直接填 Node.draggable。
- 改 pkg 格式必重编 .dll + commit（坑 10）。

## 7. C# 镜像（loomgui_unity）

### 7.1 EventType 枚举（LoomEventHandler.cs）
- `+ DragStart=6, DragMove=7, DragEnd=8, LongPress=9`。

### 7.2 DispatchPending 路由
- DragStart/Move/End/LongPress → `BubbleRoute`（照 Down/Up/Click）。
- EventContext 透传 x/y/clickCount（既有，无新字段；delta 业务自算）。

### 7.3 无 C# 手势类
- 检测全在 core，C# 不新增手势识别器（否决 fgui LongPressGesture C# 类方案）。

## 8. 测试

### 8.1 core 单测（Rust，input.rs）
**drag**：
- draggable Down + Move>阈值 → DragStart + click_cancelled（无 Click）
- DragStart 后续 Move → DragMove；Up → DragEnd
- 非 draggable Down + Move → 无 drag 事件（仅既有 MOVE）
- 阈值 per-axis：mouse 2 / touch 10（对角线、边界）
- 祖先 draggable：down 叶非 draggable 但祖先 draggable → drag_target=祖先，发 drag
- disabled draggable → 不发 drag
- 阈值内 Move（<阈值）+ Up → 不发 drag，正常 Click（drag 不破坏 click 容忍）

**longpress**：
- Down + time≥1.5s（无 move）→ LongPress 一次
- Down + time<1.5s + Up → 无 LongPress
- Down + Move>50px → longpress_cancelled，1.5s 后不发
- LongPress 不重复（fired 后再 tick 不发）
- LongPress + Up 后 Click 照发（独立）
- disabled → 不发 LongPress

### 8.2 abi 测（ffi）
- version 返 `v1d.1`
- EventRecord sizeof==20（新事件类型不增字节）
- LoomEvent（C#）Marshal.SizeOf==20

### 8.3 C# EditMode 测（LoomEventHandlerTests.cs）
- DragStart/Move/End bubble 路由（祖先接）
- LongPress 路由
- 复用 v1c.4 BuildStage/DispatchOne helper（font_path 家里机填）

### 8.4 PlayMode / 家里机（interact sample + safe-area）
- drag：draggable 元素按下拖动 → DragStart/Move/End 日志；阈值外才发；非 draggable 不发
- longpress：按住 1.5s → LongPress 日志；移动取消
- safe-area：Game 视图模拟刘海（设备模拟器/`Screen.safeArea` override）→ 内容避刘海；关 `_safeArea` 回归全屏
- 回归 v1c.1-v1c.4（hover/click/双击/capture 不破）

## 9. defer（不在 v1d.1）
- **drag 跟手（自动移动节点）→ v1d.3**（transform.translate 启用）
- **longpress onBegin/onAction 重复/onEnd → v1x**（fgui 全套手势语义）
- **CSS `env(safe-area-inset-*)`（per-element 内边距）→ v1x**（要 CSS env() 解析 + Unity→core 传 insets）
- **DragDropManager（拖放 DnD）→ v1x**（fgui DragDropManager，拾取+投放语义，独立于通用 drag）
- **其它手势（swipe/pinch/pan）→ v1x**（fgui Gesture/ 目录）
- scroll 滚动 / 滚轮 / 嵌套仲裁 → v1d.5（消费本轮 drag 事件）

## 10. 风险

| 风险 | 缓解 |
|---|---|
| drag 阈值 mouse=2 过敏（桌面点击抖动误触发 drag） | 仅 draggable 节点生效；非 draggable 走 click 容忍 10px 不受影响（照 fgui） |
| universal longpress 对无 listener 节点空发事件 | C# 无 listener 即跳过，零成本；语义独立不破坏 click |
| safe-area 输入映射与 root transform 不一致 → 触摸偏移 | §5.2 抽公共变换，ConfigureTransforms 与 Collect 共用 |
| pkg 加 draggable 触发 formatVersion bump | 优先 flags bit；plan 读 NodeBlock 布局定 |
| longpress tick 检查每帧跑（性能） | 仅遍历 is_down 槽（≤5），O(1) per tick，500 节点无影响 |
| C# 本机不编译（无 Unity） | 静态审 + 家里机跑 EditMode/PlayMode（照 v1c.4） |
