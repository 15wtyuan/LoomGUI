# v1c.1 — 事件/命中/输入最小交互闭环 设计

> 主设计文档 §10 定事件契约；本 spec 定 v1c.1 实现。v1c 分 spec 路线见 `.superpowers/sdd/progress.md`。
> 范围 = v1c 第一步（最小交互闭环）：输入桥 + 命中 + click/hover/leave + `:hover/:active/:disabled` 伪类 + `is_pointer_on_ui`。命中验收 #3（hover/active 反馈）、#5（UI 挡点击）。

## 1. 架构总览

v1c.1 在现有「渲染链」之上加一条平行的「交互链」，三层职责切分：

```
Unity (C#)                          核心 (Rust)                         打包器 (Rust)
─────────                           ──────────                          ─────────
LoomInputCollector                  Stage                               pack()
 旧输入系统 → PointerEvent[]          set_input(events[])                  抽 :hover/:active/:disabled
 ────────────────────────→  hit_test() → 命中 NodeId              规则 → DynamicRuleSection
                                  diff hover/active 状态              写进 .pkg.bin (v2→v3)
                                  伪类重匹配(全量重 cascade)            ↓
                                  产事件 SOA                          read_package()
LoomEventHandler                  borrow_events() ←────────────────  Scene 持 {base_style, dynamic_rules}
 NodeId→listener 表                                                    set_node_disabled(id,bool)
 拉事件 → 派发回业务
```

**核心新增三模块**（`loomgui_core/src/`）：
- `input.rs`：`PointerEvent`/`EventRecord` 结构 + 单指针状态机（down/up/move → click 判定 + rollover/out diff）。
- `hit.rs`：`hit_test(scene, point) -> Option<NodeId>`——逆等效绘制序遍历，layout_rect AABB + clip 门控。
- `style/rematch.rs`：运行时伪类重匹配——消费 `Scene.dynamic_rules` + 节点状态（hovered/active/disabled），全量重 cascade 叠到 base_style。

**现有改动**：
- `Scene`/`Node`：加运行时状态字段（`hovered/active/disabled/touchable` bool）+ `base_style` + 持 `dynamic_rules`（等效绘制序读 `style.order` 不加字段，§2.2）。
- `Stage`：加 `set_input` + `is_pointer_on_ui` + tick 内「命中 → 状态 diff → 伪类重匹配 → 产事件」管线。
- `style/cascade.rs`：抽出「叠规则」逻辑供 rematch 复用。
- `parse/selector.rs`：`Compound` 加伪类标志（`hovered/active/disabled: bool`），`parse_selector` 解析 `:hover` 等，`match_element` 加状态感知版。
- `asset/mod.rs`（打包器+包格式）：DynamicRuleSection 进 .pkg.bin，`PKG_FORMAT_VERSION` 2→3。

**FFI 新增**（loomgui_ffi_c）：`loomgui_stage_set_input` / `loomgui_stage_borrow_events` / `loomgui_stage_is_pointer_on_ui` / `loomgui_stage_set_node_disabled`。

**Unity 新增**（loomgui_unity）：`LoomInputCollector`（采集）+ `LoomEventHandler`（listener 表 + 派发）。

**零改契约**：frame blob v3（FrameBlob.cs）/ MirrorPool / MaterialManager / TextRasterizer / shader **零改**——v1c.1 只新增交互链 + 包格式加段，不动渲染树 SOA。包格式改是 .pkg.bin（runtime 包）非 frame blob，两者独立。

## 2. 数据结构

### 2.1 PointerEvent（输入，跨 FFI 扁平 POD）

```rust
// loomgui_core/src/input.rs
#[repr(C)]
pub struct PointerEvent {
    pub kind: PointerKind,   // u8: 0=Down,1=Up,2=Move
    pub x: f32,              // design 坐标（后端已做 screen→design 映射）
    pub y: f32,              // 左上原点（§10.5，翻转在后端根 Stage）
    pub button: u8,          // 0=左键（v1c.1 只用 0，预留）
}
#[repr(C)]
pub enum PointerKind { Down = 0, Up = 1, Move = 2 }
```
`#[repr(C)]` 保证跨 FFI 字节布局定死（§14.3 扁平 POD，禁嵌套结构跨 FFI）。C# 侧 `[StructLayout(LayoutKind.Sequential)]` 直接映。

### 2.2 Node 运行时状态字段（scene/node.rs 加）

```rust
pub struct Node {
    // ...现有字段（id/parent/kind/style/taffy_id/layout_rect/clip_rect/children/dirty_mesh/dirty_text）...
    pub base_style: ResolvedStyle,  // 打包期 resolve_styles 产物（不变，rematch 基线）
    pub touchable: bool,             // pointer-events:auto=true / none=false（v1c.1 补解析，见下）
    pub hovered: bool,               // 当前帧命中（运行时，每帧命中 diff 更新）
    pub active: bool,                // 指针按下且命中（运行时状态机）
    pub disabled: bool,              // 业务设（set_node_disabled），伪类源 + active/click 抑制
}
```
`style` 字段保留为**运行时 rematch 覆写值**（初始 = base_style.clone()）。`hovered/active` 每帧重算，`disabled` 业务设持久状态。

**等效绘制序不新加字段**：hit.rs 命中逆序遍历直接读 `node.style.order`（ResolvedStyle 已有 `order` 字段，围栏 §2 已支持），**不新加 `Node.sorting_order`**——避免与 style.order 重复字段 + rematch 改 style.order 后 sorting_order 过时的同步风险。order 运行时不可变（动态规则不改 order，§4 伪类规则只改视觉/尺寸不涉及 order），建树时 style.order 已定。

**`touchable`（pointer-events）**：围栏 §2 列了 `pointer-events:auto/none` 但现有 `apply_decl` 未实现（走 default 静默忽略）。v1c.1 补：`mapping.rs` apply_decl 加 `"pointer-events" => style.touchable = value != "none"`（auto/默认=true，none=false）+ `ResolvedStyle` 加 `touchable: bool`（默认 true）+ 建树时填 `Node.touchable`。围栏 §2 已列属 v1 范围，v1c.1 命中门控需要它（否则 `pointer-events:none` 节点挡命中）。

### 2.3 Scene 持动态规则表（scene/mod.rs 加）

```rust
pub struct Scene {
    // ...现有（roots/nodes）...
    pub dynamic_rules: DynamicRuleTable,  // 包加载时填；inline 路径空（§5.4）
}
```

### 2.4 DynamicRuleTable（伪类规则，打包器产 → 包加载填）

```rust
// loomgui_core/src/style/dynamic.rs
pub struct DynamicRuleTable {
    pub rules: Vec<DynamicRule>,
}
pub struct DynamicRule {
    pub selector: ParsedSelector,        // 复用 parse::selector::ParsedSelector（含伪类标志）
    pub declarations: Vec<Declaration>,  // (prop:String, value:String)，apply_decl 消费
}
```
复用现有 `parse::selector::ParsedSelector`——v1c.1 先扩展它加伪类标志（§2.6），不新造选择器类型。打包器从 StyleSheet 抽**含 `:hover/:active/:disabled` 的规则**填这里。运行时 rematch 用 `match_element_with_state` + 节点当前状态判定。

### 2.5 事件 SOA（输出，跨 FFI 扁平 POD）

```rust
// loomgui_core/src/input.rs
#[repr(C)]
pub struct EventRecord {
    pub node_id: u32,      // 目标节点
    pub event_type: u8,    // 0=Down,1=Up,2=Move,3=Click,4=RollOver,5=RollOut
    pub x: f32,            // 事件坐标（design）
    pub y: f32,
}
```
每帧 `Vec<EventRecord>` → FFI `borrow_events` 返回扁平 buffer（`as_ptr`，定长 POD 数组，C# 直接 `Span<byte>` 读，非手写二进制）。事件按生成序排列（同帧多事件保序）。

### 2.6 ParsedSelector 加伪类标志（parse/selector.rs 扩展）

现有 `Compound` 无伪类字段（selector.rs:4 注释「v0 不含伪类」）。v1c.1 扩展：
```rust
pub struct Compound {
    pub tag: Option<String>,
    pub classes: Vec<String>,
    pub id: Option<String>,
    pub combinator: Combinator,
    pub pseudo_hover: bool,    // NEW：含 :hover
    pub pseudo_active: bool,   // NEW：含 :active
    pub pseudo_disabled: bool, // NEW：含 :disabled
}
```
`parse_selector` 解析 `:hover`/`:active`/`:disabled` 填标志。

**Serialize/Deserialize derive（待加，非现状）**：现状 `ParsedSelector`/`Compound`/`Combinator`（selector.rs:5-24）+ `Declaration`（css.rs）**均无** `Serialize/Deserialize` derive。v1c.1 **必须给这四个类型加** `#[derive(Serialize, Deserialize)]`——DynamicRuleSection 用 bincode 序列化 `ParsedSelector`（含嵌套 `Vec<Compound>` + `Combinator` enum）进 .pkg.bin，runtime 反序列化直接用结构，不重新 parse（不依赖 parse feature，§5.2/坑 21）。spec 列为必改动项，plan 期不能假设现状已有。

`match_element` 加状态感知版 `match_element_with_state`：选择器含 `pseudo_hover` 时只当 `node.hovered=true` 才匹配，`pseudo_active`/`pseudo_disabled` 同理。

### 2.7 PointerState（单指针状态机，Stage 持）

```rust
// loomgui_core/src/input.rs
pub struct PointerState {
    pub last_pos: (f32, f32),          // 最后已知指针位置（design；set_input len=0 时沿用，§6.3）
    pub is_down: bool,                  // 当前按下
    pub down_node: Option<NodeId>,      // down 时命中的节点（click 判定 + active 源）
    pub down_pos: (f32, f32),           // down 坐标（click 阈值判定，§10.3 ~10px）
    pub last_hovered: Option<NodeId>,   // 上一帧命中（rollover/out diff 源）
}
```
v1c.1 单指针，Stage 持一个 `PointerState` 实例。click 判定：`is_down→up` 且 `down_node == up_hit` 且 `|up_pos - down_pos| < 10px`（§10.3 鼠标阈值）。rollover/out：本帧 `hit_test(last_pos)` vs `last_hovered` diff。`last_pos` 在每次 `set_input`（含 len=0 之外的 Move/Down/Up）更新；len=0 时沿用，保证鼠标静止 hover 保持（§6.3）。

## 3. 命中测试（hit.rs）

### 3.1 算法

```rust
/// §10.1：输入 design 坐标点 → 返回命中 NodeId（None = 命中根外/空）。
/// 逆等效绘制序遍历，第一个命中即返回（顶层优先）。
pub fn hit_test(scene: &Scene, point: (f32, f32)) -> Option<NodeId>
```

单节点命中判定 `point_hits_node(scene, id, point) -> bool`：
1. **visible + touchable 门控**：v1c.1 `visible` 恒 true（无 display:none 运行时切换），`touchable` = `Node.touchable`（来自 CSS `pointer-events`，v1c.1 补解析见 §2.2）。`pointer-events:none`（touchable=false） → 跳过该节点**但继续测其子**（CSS 语义：none 不挡子节点命中）。
2. **clip 门控**：有 `clip_rect`（overflow:hidden）且点不在 clip 内 → 该子树整体不命中（裁掉了）。
3. **AABB 测试**：`layout_rect` 含点 → 候选命中。
4. **子节点逆序优先**：先递归测子（顶层先），子命中则返回子的；子都不命中 → 当前节点自身 fallback 命中（容器自身 AABB 含点）。

### 3.2 等效绘制序（order 重排）

§10.1：children 顺序经 `order` 重排，非零排前（顶层）。命中遍历按重排后逆序（顶层先测）。
```rust
fn effective_draw_order(scene: &Scene, parent: NodeId) -> Vec<NodeId>
```
v1c.1 多数节点 `style.order=0` → 退化为 children 原序逆序（与现有渲染 DFS 顺序一致）。`order` 来源：CSS `order`（围栏 §2 已支持，ResolvedStyle 有 `order` 字段）——hit.rs **直接读 `node.style.order`**，不新加 Node 字段（§2.2 已述：避免重复 + 同步风险）。无 CSS order 则 0。

### 3.3 坐标系

命中全在 **design 坐标系**（§14.3：layout_rect 绝对 design，根 Stage 一次性 design→world）。`set_input` 传的 `PointerEvent.x/y` 必须是 design 坐标（Unity 侧 screen→design 映射）。核心不做翻转（§10.5 翻转在后端根 Stage一次性）。

### 3.4 不做 transform（v1c.1 边界）

§10.1 第 1 步「累计 transform 逆矩阵 world_to_local」——v1c.1 不做（transform defer v1d）。命中直接用 `layout_rect` AABB。v1c.1 无动画故无影响。v1d 上 transform 后加 world_to_local。

### 3.5 命中结果用途

每帧 tick 内 `hit_test(scene, pointer_pos)` → `cur_hit`：①hover diff（cur_hit vs last_hovered → RollOver/RollOut + 更新 Node.hovered）；②active 状态机（down 时 down_node=cur_hit，active = down && cur_hit==down_node）；③is_pointer_on_ui（cur_hit 非空且非根）。

## 4. 伪类重匹配（rematch.rs）

### 4.1 执行模型（全量重 cascade，§5.3 既定）

每帧 tick 内命中 diff 后**无条件全量节点重匹配（仅动态规则子集）**（不做「状态没变就跳过」的指纹缓存——§5.3 明确不做）。500 节点全量 rematch 微秒级。
```rust
/// base_style + 动态规则（按节点当前 hovered/active/disabled 匹配）。
/// 全量重算每节点 style 写回 Node.style。返回是否触发 layout dirty。
pub fn rematch_pseudo_classes(scene: &mut Scene) -> bool
```

### 4.2 算法

对每个节点：
1. 取 `base_style`（Node.base_style，打包期产物，不变）。
2. 遍历 `scene.dynamic_rules.rules`，每条用 `match_element_with_state(rule.selector, node, scene)` 判定（含伪类状态门）。
3. 命中动态规则按 specificity 升序 apply（同现有 cascade 顺序），叠到 base_style 得新 `Node.style`。
4. 对比新旧 style：layout 相关字段（taffy_style：size/flex/padding/margin）变 → 标 `layout_dirty`；仅视觉字段（background_color/color/opacity）变 → 标 `mesh_dirty`。

base_style 不动——rematch 每帧从 base_style 重叠动态规则，不累积。

### 4.3 打包器分流（§5.5）

含 `:hover/:active/:disabled` 的规则 → dynamic_rules；纯静态规则 → 烤进 base_style（现有打包期 resolve_styles 不变）。rematch 只跑动态规则子集（全量节点 × 动态规则子集）。

### 4.4 disabled 状态源

`set_node_disabled(scene, node_id, bool)` → 写 `Node.disabled` → 下帧 rematch `:disabled` 生效。disabled 节点**仍参与命中，但 active/click 抑制**——hit_test 不跳 disabled 节点（仍可被 hover diff 触发 `:disabled` 伪类显示 disabled 态），但 down 时若命中 disabled 节点 → 不设 active、up 不产 click。

**LoomGUI 有意偏离 fgui**：fgui `enabled=false` = `touchable=false`，hit-test **完全跳过** disabled 节点（既不命中也不 hover）。LoomGUI 因选了 CSS `:disabled` 伪类 DSL，需要 disabled 节点仍被 hover diff 触发才能显示 disabled 态（`:disabled` 伪类的 hover 反馈是 CSS 设计动机），故保留命中但抑制 active/click。fgui 无伪类 DSL 不需要此行为。**这与 `pointer-events:none`（§3.1 跳过自身+继续测子）不同**——disabled 不改 touchable，touchable 仍 true，hit_test 正常返回 disabled 节点，只是状态机层抑制 active/click。

### 4.5 触发时机（tick 管线，§15 v1c.1 版）

```
set_input(events) → hit_test → 状态 diff（hover/active 更新 Node 字段）
  → rematch_pseudo_classes（全量重 cascade，写 Node.style + dirty）
  → layout dirty → taffy solve（子树）
  → 渲染 pass（mesh dirty 重生成）
  → 产事件 SOA
```
rematch 在 hit 后、solve 前——伪类改 layout 属性能本帧 solve（核心状态 diff 驱动，不延下帧）。**事件回调改的布局才延下帧**（§15 防反馈环）。

## 5. 包格式演进（.pkg.bin v2 → v3）

### 5.1 改动总览

v1b.3 的 v2 结构：Header + StringTable + NodeBlock + AtlasSection。v1c.1 加一段 + bump version：
```
Header (version 2→3, MIN=MAX=3，旧 v2 拒)
StringTable
NodeBlock        ← bincode(ResolvedStyle) 语义改为 base_style（结构不变）
AtlasSection
DynamicRuleSection  ← NEW（含伪类规则集）
```

### 5.2 DynamicRuleSection 结构

```rust
// loomgui_core/src/asset/mod.rs
pub struct DynamicRuleSection {
    pub rules: Vec<DynamicRuleEntry>,
}
pub struct DynamicRuleEntry {
    pub selector: ParsedSelector,        // bincode 序列化（含伪类标志）
    pub declarations: Vec<Declaration>,  // (prop, value) 同静态规则
}
```
**selector 走 bincode 不走 string**：运行时 rematch 要查选择器结构（含伪类标志），存 selector_text 每帧重新 parse 浪费且 parse feature 在 runtime 可能关。bincode 预序列化 → runtime 直接用结构，**不依赖 parse feature**（与包格式「runtime 稳定入口不依赖 parse」纪律一致，坑 21 同源）。

### 5.3 base_style 进 NodeBlock

现状 NodeBlock 每节点 `bincode::serialize(&n.style)`。v1c.1 这就是 `base_style`——结构不变，语义明确为「静态烤好的基础样式」。read_package 建 Node 时 `base_style = 反序列化值`，`style = base_style.clone()`。

### 5.4 inline 路径不支持伪类（§2 决策）

inline（load_inline）走 parse → resolve_styles，**v1c.1 不抽 dynamic_rules**（空）。inline 伪类规则在 resolve_styles 当静态匹配（现有行为）——hover/active/disabled 在 inline 路径不生效。inline 仍支持命中+事件（hit_test 不依赖 dynamic_rules），只是伪类视觉反馈不生效。**伪类重匹配测用包路径**；命中/事件测可用 inline。

### 5.5 formatVersion bump 级联（坑 17 纪律）

1. Rust `asset/mod.rs`：`PKG_FORMAT_VERSION` 2→3 + `MIN_VERSION`/`MAX_VERSION` 同步。
2. `read_package`：解 DynamicRuleSection（version>=3）。
3. `write_package` 签名加 `dynamic_rules: &DynamicRuleSection` 参数（同加 atlas 参数的先例）。
4. grep 所有手搓包测 fixture：升 version + 补 DynamicRuleSection（空也行，结构要对）。
5. **frame blob（FrameBlob.cs）不动**——.pkg.bin 改与 frame blob 独立。

### 5.6 零改边界

- frame blob v3（FrameBlob.cs）零改：v1c.1 不动渲染树 SOA。
- MirrorPool/TextRasterizer/shader 零改。
- .pkg.bin 改要重编 .dll（坑 10）。C# 侧只透传包 bytes 不解析包（现状），故 C# 侧零改——只 Rust 侧 read_package/write_package 变。

## 6. FFI 接口（loomgui_ffi_c）

### 6.1 新增函数

```rust
/// 注入本帧指针事件（扁平 PointerEvent 数组）。tick 前调。
/// null/len=0 = 本帧无输入事件（指针位置沿用上帧，hover diff 仍跑）。
#[no_mangle]
pub extern "C" fn loomgui_stage_set_input(h: *mut StageHandle, events: *const PointerEvent, len: usize)

/// 拉取本帧事件 SOA（pull，同 borrow_frame 模式）。
/// 写 len 到 out_len，返回 EventRecord[] 缓存指针（下 tick 失效）。
/// null 句柄或未 tick → null + len=0。
#[no_mangle]
pub extern "C" fn loomgui_stage_borrow_events(h: *mut StageHandle, out_len: *mut usize) -> *const u8

/// UI 挡住时游戏不响应点击（§10.6）。= 当前指针命中非空且非根。
#[no_mangle]
pub extern "C" fn loomgui_stage_is_pointer_on_ui(h: *const StageHandle) -> bool

/// 业务设节点 disabled 状态（伪类源 + active/click 抑制）。
#[no_mangle]
pub extern "C" fn loomgui_stage_set_node_disabled(h: *mut StageHandle, node_id: u32, disabled: bool)
```

### 6.2 StageHandle 加缓存

```rust
pub struct StageHandle {
    stage: Stage,
    frame_blob: Vec<u8>,   // 现有
    event_blob: Vec<u8>,   // NEW：borrow_events 返回 &this[..]，下 tick 覆盖
}
```
`event_blob` = `Vec<EventRecord>` 的 `as_ptr`/len（`#[repr(C)]` POD，C# 直接 `Span<byte>` 读，无需序列化）。

### 6.3 生命周期与失效语义

- 顺序：`set_input → tick → borrow_frame + borrow_events`。
- `borrow_events` 指针下 tick 失效（同 borrow_frame）。
- `set_input` len=0：本帧无输入事件，指针位置沿用上帧最后已知位置（hover diff 仍跑，鼠标静止停按钮上 hover 保持，§10.3 语义）。
- `is_pointer_on_ui`：tick 后调，读 Stage 缓存的本帧命中结果。

### 6.4 输入坐标契约

`PointerEvent.x/y` 必须 **design 坐标 + 左上原点**（§10.5/§3.3）。Unity 侧做 screen→design 映射。核心不做翻转。

### 6.5 常驻 vs parse-gate

4 个新函数**全常驻（不 gate）**——输入/事件是 runtime 稳定入口，不依赖 parse feature（同 load_package 纪律）。`PointerEvent`/`EventRecord` 定义在 `loomgui_core::input`，core 常驻导出。`--no-default-features` 构建的 .dll 仍有这 4 函数（坑 21）。

### 6.6 version 字符串

`loomgui_version()` 的 `"v1b.3"` → `"v1c.1"`（工具链 round-trip 测用）。

## 7. Unity 后端（loomgui_unity）

### 7.1 LoomInputCollector（输入采集 MonoBehaviour）

```csharp
[ExecuteAlways]
public class LoomInputCollector : MonoBehaviour
{
    // v1c.1 用 Unity 旧输入系统（Input.mousePosition/GetMouseButtonDown/Up）
    // 桌面 Mono backend 最简；新输入系统留 v1d+（多触摸/手势需要时换）
    // 每帧采集 → PointerEvent[] → loomgui_stage_set_input
}
```
采集：`Input.mousePosition` → screen→design 映射（`/Screen.width*rootSize.x`, `/Screen.height*rootSize.y`）→ Move；`GetMouseButtonDown(0)` → Down；`GetMouseButtonUp(0)` → Up。每帧产 0-2 个 PointerEvent（move 恒产 + down/up 边沿产）。Update 开头采集 → set_input → LoomStage.tick → borrow。

**坐标 y-flip**：`Input.mousePosition` 左下原点，LoomGUI 要左上原点 design。`design_y = rootSize.y - (mouse.y / Screen.height * rootSize.y)`。输入采集侧自己翻到 design 左上（与渲染根 Stage (1,-1,1) 独立，§8.1）。

### 7.2 LoomEventHandler（listener 表 + 派发）

```csharp
public class LoomEventHandler
{
    Dictionary<uint, Dictionary<EventType, Action<LoomEvent>>> _listeners;
    public void AddListener(uint nodeId, EventType type, Action<LoomEvent> cb);
    public void RemoveListener(uint nodeId, EventType type, Action<LoomEvent> cb);
    public void DispatchPending(LoomStage stage);  // borrow_events → 逐事件查表派发
}
public enum EventType : byte { Down=0, Up=1, Move=2, Click=3, RollOver=4, RollOut=5 }
public struct LoomEvent { public uint nodeId; public EventType type; public float x; public float y; }
```
`DispatchPending`：调 `loomgui_stage_borrow_events` 拿 `EventRecord[]` → `Span<byte>` 逐条读（§14.3 禁 `Marshal.PtrToStructure`）→ 查 `_listeners[nodeId][type]` 派发。无 listener 的事件丢弃。

### 7.3 业务用法

```csharp
var handler = new LoomEventHandler();
handler.AddListener(buttonNodeId, EventType.Click, e => Debug.Log("clicked"));
// 每帧：collector.Collect(stage); stage.Tick(dt); handler.DispatchPending(stage);
```
对齐 fgui `btn.onClick.Add(callback)`（listener 在 C# 域，核心只产事件）。v1c.1 不做 fgui `EventListener` 属性式 API（`onRollOver`），用显式 `AddListener(nodeId, type, cb)`——v1c.2 路由完整化时再考虑 EventBridge/ListenerId。

**事件回调改状态路径**：v1c.1 listener 在 C# 侧，回调里改节点状态（如点 button 禁用另一个 button）走 FFI 命令式 API（`loomgui_stage_set_node_disabled` 等）回传核心——这是跨 FFI 调用，不在事件回调的同帧 solve 循环里，天然延下帧 solve，无反馈环风险（§15）。v1c.1 不提供「回调里直接改样式」的同步路径（改样式 = 改状态 = 走 FFI 命令式 + 延下帧 rematch）。

### 7.4 LoomStage 接线

现有 `LoomStage.cs` 加：持 `LoomInputCollector` + `LoomEventHandler`；tick 流程 `collector.Collect → set_input → tick(dt) → borrow_frame + borrow_events → handler.DispatchPending`；`is_pointer_on_ui` 暴露给游戏。

### 7.5 零改边界

MirrorPool/MaterialManager/FrameBlob/TextRasterizer/shader 零改——只加采集 + 派发两新 class。.dll 重编（坑 10）。.meta 补齐（坑 13）。

### 9.1 v1c.1 做

| 能力 | 落点 |
|---|---|
| 扁平指针事件输入（down/move/up+x,y） | core input.rs + FFI set_input |
| 命中测试（逆等效绘制序，layout_rect AABB+clip+pointer-events） | core hit.rs |
| 单指针状态机（click ~10px + rollover/out diff） | core input.rs，Stage 持 PointerState |
| 事件产出（Down/Up/Move/Click/RollOver/RollOut 六种 SOA） | core + FFI borrow_events |
| `:hover/:active/:disabled` 伪类重匹配（全量重 cascade） | core rematch.rs |
| 打包器动态规则表进 .pkg.bin（v2→v3，§5.5） | loomgui_pkg + asset/mod.rs |
| `is_pointer_on_ui()` / `set_node_disabled` | core + FFI |
| Unity 输入采集（旧输入系统）+ listener 派发（C# 侧） | LoomInputCollector + LoomEventHandler |
| Node 运行时状态字段 + base_style + touchable（等效序读 style.order） | scene/node.rs + style/mapping.rs |

| interact sample | loomgui_pkg/samples/interact/ |

### 9.2 v1c.1 不做（in-v1 但 defer）

| 能力 | defer 到 | 原因 |
|---|---|---|
| DOM 三阶段路由 + stop_propagation + EventBridge/ListenerId | v1c.2 | v1c.1 用 C# 显式 AddListener 直派 |
| 多触摸槽 + capture_touch 多 monitor | v1c.3 | v1c.1 单指针 |
| transform 字段 + world_to_local 命中 | v1d | transform 随动画/拖拽，无动画故无影响 |
| GTween 时钟 | v1d | 无动画 |
| ScrollPane + 拖拽 + 拖拽/滚动仲裁 | v1d | 仲裁耦合 ScrollPane（§10.4） |

### 9.3 v1.x defer（不在 v1）

Controller/Gear/Transition（§11.2-11.4）、`[data-page]`（§5.3，随 Controller）、shape mask / hit_area trait（§10.1，v1c.1 只 rect clip）、命中跨帧缓存（§10.1 按帧号缓存；v1c.1 每帧 tick 内 hit_test 算一次，结果存 Stage 供 is_pointer_on_ui 读，不跨帧缓存）。

### 9.4 零改契约

- frame blob v3（FrameBlob.cs）/ MirrorPool / MaterialManager / TextRasterizer / shader 零改。
- `measure_text` / `Stage::new` / `build_render_nodes` 签名不变（命中/rematch 在 tick 内加）。
- 改动面：core 加 3 模块 + Scene/Node 加字段 + 包格式 v3 + FFI 4 函数 + Unity 2 新 class + sample。

### 9.5 AI 可预测性约束

- 伪类重匹配全量不缓存（§5.3）——状态变即重算，无 invalidation 黑盒。
- 事件产出确定性：同输入序列产同事件序列（状态机无随机/时间依赖，click 阈值常量）。
- 命中逆等效绘制序（与渲染 DFS 一致）——顶层优先可预测。
- listener 在 C# 显式注册——谁监听谁收，无核心侧隐式派发。

### 9.6 已知简化（围栏文档/sample 须告知）

- inline 路径不支持伪类（dynamic_rules 空）——开发期用包路径验伪类。
- v1c.1 命中对 transform 偏移元素不准（无 transform，但无动画故无影响）——v1d 修。
- disabled 仍参与命中但 active/click 抑制（LoomGUI 有意偏离 fgui——支持 `:disabled` 伪类 hover 反馈；fgui `enabled=false` 完全跳过 hit-test，见 §4.4）。
