# v1.3 动态树重构（Dynamic Tree）设计

> **状态**：设计定稿待 review
> **日期**：2026-07-01
> **前置**：v1.3（ColorFilter + 九宫格 + profiling）已完成；本 spec 是 roadmap §1 v1.x 编号下的 **v1.3+**（插在 v1.4 列表之前的地基重构）。
> **依据**：brainstorm 会话（2026-07-01）+ static-tree 影响面评估（Explore 全量扫描）+ fgui GList/GObject 对照。

---

## 0. 为什么有这一项

v1 的 scene 树是 **static-tree pipeline**：`load_package`/`load_inline` 一次性建整棵树并冻结，之后只 tick。这套能撑过 showcase 46 卡，因为 showcase 是静态样张。

但游戏 UI 的常态是动态的：背包 item 随玩家变、列表虚拟化、弹窗 add/remove 子树、聊天消息流式追加。v1.4（列表）、v1.5（Controller/Transition）、v1.6（富文本）、v1.7（IME）**每一个**都会撞同一堵墙——它们都需要运行时改树/改内容。v1.4 只是第一个撞上的。

**本项不是 v1.4 的一部分，是它的地基**：把"v1 为了快速走通而冻结的树"解冻，让宿主能运行时建/挂/删/改节点子树。地基打好后，v1.4 列表虚拟化、v1.5+ 全部动态需求都在其上构建，不再各自打补丁绕过 static-tree。

**设计约束**（项目级，优先于工作量）：LoomGUI 是个人兴趣项目，**设计合理 > 工作量**。架构决策选最干净最严谨的方案，"改动大"不是反对理由。YAGNI 仍适用（不为真不存在的需求建抽象），但"为 v1.4+ 铺路的通用动态基础"值得一次到位，不做只服务 v1.4 的最小版。

## 1. 目标与非目标

### 目标

1. **代际 NodeId**：节点可运行时增删，删除后旧 NodeId 安全失效（悬空检测），无全架构 NodeId 错位风险。
2. **动态树 API**：宿主经 FFI 运行时建根/建节点/挂载/摘除/删除/改内容/改样式。
3. **附属状态联动清理**：删节点时 anim/scroll/tween 附属状态同步清，无残留、无悬空。
4. **架构清理（代际化强制要求）**：base_style/style 语义分离；world_transforms 时机前移解决结构变更帧 hit_test 越界 + 保留 scroll_pos 同帧进 world matrix。
5. **v1 既有行为零回归**：static-tree 下的 pkg/inline 加载、渲染、滚动、输入、动画、伪类全部不变。

### 非目标（推迟）

- **列表虚拟化 / slot 复用 / soft clip**：v1.4，在本项动态树之上构建。
- **Controller / Gear / Transition**：v1.5。
- **runtime CSS stylesheet 重解析 / class 增删触发整树 rematch**：本项 set_style 只改单节点 base_style；class 增删 + 整树重匹配推后。
- **代际 NodeId 跨 .pkg.bin 持久化**：pkg 是静态打包产物，load 时重建 NodeId，无持久化代际需求。

## 2. 已定架构决策（brainstorm 确认）

| # | 决策 | 选择 | 关键理由 |
|---|---|---|---|
| D1 | NodeId 表示 | `u32` = 高 20 bit index + 低 12 bit generation | 对外 u32 不透明句柄（FFI 零签名变化），代际是内部实现；100 万节点 + 4096 代足够 |
| D2 | 节点存储 | `SlotMap<NodeId, Node>`（`new_key_type!`） | slotmap 已在依赖树（taffy 带），零新依赖；成熟、wrap 安全、悬空检测现成 |
| D3 | 并行数组策略 | A4：anim/scroll 用 `Vec<Option<T>>` + 删时联动清；world_transforms/text_layouts 保持 `Vec<T>` | 代际安全由 slotmap 保证，附属数据不重复付代际开销；正确性靠"删除联动"显式不变量非访问顺序约定 |
| D4 | 动态 API 入参 | CSS 字符串（create_root/create_node/set_style） | 复用 mapping.rs parse_*（默认编译，零新依赖）；贴 CSS 模型 + AI 可预测性；建节点低频无 parse 开销顾虑 |
| D5 | base/style 语义 | base_style（源）+ style（派生，每帧 rematch） | 代际化 + set_style 要求；当前架构已隐式此模型，显式化低风险 |
| D6 | world_transforms 时机 | 前移到 process/scroll 之后、render 之前（每帧 1 次，删末尾冗余 + 首帧 guard） | 消除结构变更帧 hit_test 越界（长度 = 当前 nodes）；scroll_pos 同帧进 world matrix 保留；hit_test 仍用上帧值（1 帧延迟本已认）。详见 §9.3 |
| D7 | TweenManager 联动 | remove_node 时杀该节点所有 tween | 当前 `t.node.0 >= n` 长度检查代际化后失效，改主动杀 |

## 3. 代际 NodeId 内部表示（D1）

### 3.1 物理布局

```rust
/// 不透明节点句柄。对外 u32（FFI/C# 透明），内部 = 高 20 bit index + 低 12 bit generation。
/// sentinel 0xFFFF_FFFF = 无效（同 v1，FFI 返 None/0 时用）。
#[derive(Copy, Clone, Eq, PartialEq, Hash, Debug)]
pub struct NodeId(pub u32);

impl NodeId {
    pub const INVALID: NodeId = NodeId(0xFFFF_FFFF);
    pub fn is_valid(self) -> bool { self.0 != 0xFFFF_FFFF }
    /// slotmap 槽位号（高 20 bit）。用于并行数组（anim/scroll/world_transforms）索引。
    pub fn index(self) -> usize { (self.0 >> 12) as usize }
    /// generation（低 12 bit）。slotmap 内部校验用，业务代码不直接读。
    pub fn gen(self) -> u16 { (self.0 & 0xFFF) as u16 }
}
```

**位宽划分 20/12**（brainstorm 确认）：
- index 20 bit = ~100 万节点上限。游戏 UI 单屏节点几千量级，绰绰有余。
- generation 12 bit = 4096 代。同一槽复用 4096 次才 wrap，配合 slotmap 内部 wrap 检测足够安全。
- 24/8 被否决：256 代在列表频繁复用槽时有 wrap 风险。

### 3.2 与 slotmap Key 的衔接

slotmap 的 `new_key_type!` 生成自定义 Key 类型，其内部布局由 slotmap 决定（含 index + version）。**NodeId 与 slotmap Key 的关系**：

- `SlotMap<NodeId, Node>`：`new_key_type!` 让 slotmap 直接用 `NodeId` 作为 Key 类型。slotmap 的 `insert` 返回 `NodeId`，`get(NodeId)` 内部解包 index + 校验 version。
- slotmap Key 的 version 位宽由 slotmap 自己管（默认足够），与我们 NodeId 的 12-bit generation **不必字面一致**——NodeId.0 的具体编码由 slotmap 的 Key 序列化决定，我们只保证"对外是 u32 + sentinel 0xFFFF_FFFF"这个契约。
- **实现期校准点**：`new_key_type!` 生成的 Key 能否保证 `0xFFFF_FFFF` 永不作为合法 Key 返回（即 sentinel 安全）。若 slotmap 的编码可能撞 0xFFFF_FFFF，需在 `insert` 后断言或预留。实现时验证。

### 3.3 FFI / C# 透明性

当前 FFI 全程 `node_id: u32`（`NodeId(node_id as usize)` 转换），sentinel `0xFFFF_FFFF`。代际化后：
- FFI 签名**零变化**：仍是 `node_id: u32`。
- Rust 侧 `NodeId(u32)` 直接传 slotmap，不再 `as usize`（`NodeId(pub usize)` → `NodeId(pub u32)`）。
- C# 侧 `uint` 透明，仍当不透明句柄存/传。
- **事件 target / tween node_id / find_node_by_id 返回 / borrow_events 里的 NodeId**：全都是 u32，代际化对 C# 不可见。删节点后，C# 持有的旧 u32 句柄再用时，Rust 侧 slotmap.get 返 None → FFI no-op（沿用 v1 越界静默跳过语义）。

## 4. slotmap 接入（D2）

### 4.1 Scene 字段变更

```rust
pub struct Scene {
    pub roots: Vec<NodeId>,                              // 不变（Vec<NodeId>）
    pub nodes: SlotMap<NodeId, Node>,                    // Vec<Node> → SlotMap
    pub dynamic_rules: DynamicRuleTable,                 // 不变
    pub focused_node: Option<NodeId>,                    // 不变
    pub world_transforms: Vec<Affine2>,                  // 不变（每帧重建，按 index 索引）
    pub anim: AnimTable,                                 // 内部 Vec<NodeAnim> → Vec<Option<NodeAnim>>（见 §5）
    pub scroll: ScrollTable,                             // 已是 Vec<Option<ScrollPaneState>>，不变
    pub text_layouts: Vec<Option<TextLayout>>,           // 不变（每帧重建）
}
```

### 4.2 Node 字段变更

```rust
pub struct Node {
    pub id: NodeId,                  // 保留（slotmap key 的镜像，build/insert 时填，便于 n.id 直接用）
    pub parent: Option<NodeId>,      // 不变（slotmap key，代际安全）
    pub kind: NodeKind,              // 不变
    pub base_style: ResolvedStyle,   // 原 style 改名：源，build/create/set_style 写
    pub style: ResolvedStyle,        // 派生：每帧 rematch 从 base_style 起算 + 叠伪类
    pub taffy_id: Option<taffy::NodeId>,
    pub layout_rect: Rect,
    pub clip_rect: Option<Rect>,
    pub children: Vec<NodeId>,
    pub dirty_mesh: bool,
    pub dirty_text: bool,
    // ...其余字段不变
}
```

**`Node.id` 保留**：代码大量用 `n.id`（如写进 RenderNode.node_id）。slotmap 的 key 存在外部，但 Node 镜像一份避免到处传 key。insert 时填，与 slotmap key 一致。

### 4.3 访问模式变更：`scene.nodes[id.0]` → `scene.nodes.get(id)`

当前全代码 `scene.nodes[id.0]` 直接索引（无校验，70+ 处）。代际化后必须改：

- **生产代码**：`scene.nodes[id.0]` → `scene.nodes.get(id).expect("live node")`（确信 live 时）或 `scene.nodes.get(id)?`（可能悬空时）。slotmap.get 自带 gen 校验，悬空返 None。
- **DFS 遍历**：`for c in &node.children { rec(scene, *c, ...) }` 不变（children 存的是 live NodeId，遍历时校验）。但 `rec` 内 `scene.nodes.get(id)` 要处理 None（理论上不应发生，因为父删子必先摘，但防御性 `expect` 或 `return`）。
- **`0..scene.nodes.len()` 整表遍历**：slotmap 的 `len()` 返回**存活节点数**（非槽位总数）。整表遍历改 `for (id, node) in &scene.nodes`（slotmap 迭代器只 yield live 槽，自动跳空洞）。

### 4.4 Scene::build / build_scene 改造

当前 `Scene::build(entries)` 一次性 push 进 Vec。slotmap 下：
- `entries` 顺序遍历，`scene.nodes.insert(node)` 返回 NodeId，填回 `node.id` + 建 parent/children。
- **NodeId 不再是数组下标连续整数**——entries 第 i 个的 NodeId 由 slotmap 分配（前 N 个无删除时仍是 0..N，但语义上不再保证连续）。
- 既有 70 处 `Scene { ... }` 字面量构造（测试）全要改：`nodes: vec![...]` → 用 `Scene::build(entries)` 或建个 test helper `Scene::from_nodes(...)`。

## 5. 并行数组代际化策略（D3 = A4）

### 5.1 数组分类与策略

| 数组 | 性质 | 代际化策略 |
|---|---|---|
| `nodes` | 持久（slotmap） | slotmap，代际安全 |
| `world_transforms` | **每帧重建**（compute_world_transforms `vec![IDENTITY; n]`） | 保持 `Vec<Affine2>`，按 `NodeId.index()` 索引，空洞位置 IDENTITY 无害（无 live NodeId 引用） |
| `text_layouts` | **每帧重建**（solve `vec![None; n]`） | 保持 `Vec<Option<TextLayout>>`，按 `NodeId.index()` 索引，空洞 None 无害 |
| `anim` | 持久（ensure 只增不减） | `Vec<NodeAnim>` → `Vec<Option<NodeAnim>>`，按 `NodeId.index()` 索引，**删节点联动清 None** |
| `scroll` | 持久（ensure 只增不减） | 已是 `Vec<Option<ScrollPaneState>>`，按 `NodeId.index()` 索引，**删节点联动清 None** |

**为何 world_transforms/text_layouts 不用 Option**：它们每帧 `vec![default; n]` 全量重建，n = slotmap 槽位总数（`scene.nodes.capacity()` 或用 `index()` 最大值 +1）。空洞位置是 IDENTITY/None，永远不被读到（live NodeId 的 index 才被访问，空洞对应已删节点，slotmap.get 先拦）。无跨帧状态，无残留风险。

**为何 anim/scroll 要 Option + 联动清**：它们跨帧累积（scroll_pos/velocity/opacity 插值中间值）。删节点后若不清，槽位复用时新节点读到上个租客的旧 scroll_pos/anim override = 花屏/错位。`Option` + 删时清 None 保证复用槽是新数据。

### 5.2 联动清理不变量（同构坑 88 教训）

**删节点 = 全附属表同步清事件**。`remove_node(id)` 内部：

```rust
fn remove_node(&mut self, id: NodeId) {
    // 1. 从父摘除（parent.children remove + 调整 roots 若是根）
    // 2. 递归删子节点（先删子再删己，DFS）
    // 3. 联动清附属状态（本节点的，子的在递归时已清）：
    self.anim.clear_at(id.index());       // anim[idx] = None
    self.scroll.clear_at(id.index());     // scroll[idx] = None
    self.tweens.kill_node(id);            // 杀该节点所有 tween（见 §7）
    // 4. scene.nodes.remove(id)（slotmap remove，gen++）
}
```

**所有"按 NodeId.index() 索引的持久附属表"都要在 remove_node 联动清**。final review 追数据流时逐附属表核对（同坑 88"program 号变更 = 全消费点同步"）。当前附属表 = anim + scroll + tween。后续若加新附属表（如 v1.5 Gear 存储），remove_node 必须同步加清。

### 5.3 AnimTable 改造

```rust
// 前：pub struct AnimTable(pub Vec<NodeAnim>);
// 后：
pub struct AnimTable(pub Vec<Option<NodeAnim>>);

impl AnimTable {
    pub fn get(&self, id: NodeId) -> Option<&NodeAnim> {
        self.0.get(id.index())?.as_ref()
    }
    pub fn ensure(&mut self, id: NodeId) -> &mut NodeAnim {
        let idx = id.index();
        if idx >= self.0.len() { self.0.resize(idx + 1, None); }
        self.0[idx].get_or_insert_with(NodeAnim::default)
    }
    pub fn clear_at(&mut self, idx: usize) { if idx < self.0.len() { self.0[idx] = None; } }
    pub fn clear_node(&mut self, id: NodeId) { self.clear_at(id.index()); }  // 语义同前
    // clear_prop 不变（改 clear_at 内部的 NodeAnim 字段）
}
```

**注意**：`ensure` 用 `id.index()` 而非 `id.0`（NodeId 现在是 u32，index 是高 20 bit）。所有 `node.0`/`id.0` 作下标处改 `node.index()`/`id.index()`。

### 5.4 ScrollTable 改造

已是 `Vec<Option<ScrollPaneState>>`，结构不变。`get`/`get_mut`/`ensure` 的下标从 `id.0` 改 `id.index()`。新增 `clear_at(idx)` 供 remove_node 联动清。

### 5.5 world_transforms / text_layouts 容量对齐

每帧重建时，长度要覆盖所有 live 节点的 `index()`。slotmap 删除留洞后，`index()` 最大值可能 > `len()`（存活数）。两个选择：
- **(a) 长度 = slotmap 槽位总数**（`scene.nodes.capacity()` 或维护一个 `max_index`）：空洞位置浪费少量内存，但 `world_transforms[id.index()]` 永不越界。
- **(b) 长度 = `index()` 最大值 + 1**：每帧扫一遍 live 节点求 max。O(n) 但 n 已是遍历成本的一部分。

实现期选 (a)（slotmap 的 capacity 即槽位总数，直接用，零额外扫描）。空洞位置的 IDENTITY 是几字节浪费，可忽略。

## 6. base_style / style 语义分离（D5）

### 6.1 当前混用问题

v1 的 `node.style: ResolvedStyle` 身份混乱：
- `Scene::build` 时填 cascade 后的非伪类最终值（当 base 用）。
- `rematch_pseudo_classes` 每 tick 在它上面叠加伪类规则改 bg_color 等（当可变最终值用）。
- 无"源/派生"区分。

代际化 + set_style 要求明确：set_style 改什么？改 base 还是派生？若改派生，下帧 rematch 从 base 重算会冲掉 set_style 的值。

### 6.2 分离后语义

- **`node.base_style: ResolvedStyle`** — 源。build 时 cascade 填（非伪类规则合并）、`create_node` 时 parse CSS 填、`set_style` 时改。这是"节点的内联样式 + 匹配的非伪类规则的合并"。
- **`node.style: ResolvedStyle`** — 派生。每帧 `rematch_pseudo_classes` 从 `base_style` 起算，叠加匹配的伪类规则，产出 `style`。render/transform 读 `style`。

**set_style 改 base_style**，下帧 rematch 自动重算 style，伪类规则每帧从 base 重算不会被冲掉。

### 6.3 低风险依据

调研发现 `cascade.rs:115-128` 伪类规则**已故意排除在 base cascade 之外**（base 只算非伪类规则，伪类走 dynamic rematch）。所以当前 base_style 本就是"非伪类合并值"，伪类是每帧叠加——**架构已隐式此模型，只是字段没分开**。代际化显式化：
- build 时填 `base_style`（原来填 style 的非伪类值）。
- rematch 从 `base_style` 起算（原来从 style 起算，但 build 后 style==非伪类值，等价）。
- 派生 `style` 每帧由 rematch 产出。

计算逻辑不变，只是字段拆分 + 明确源/派生。

### 6.4 动态创建节点的 base_style

`create_node(kind, css)`：parse css 得声明，应用到 `ResolvedStyle::default()` 上（复用 mapping.rs `parse_*`），作为新节点的 `base_style`。`style` 初始 = `base_style`（首帧 rematch 前的占位，rematch 后由核心填）。

动态创建的节点无 class/id 匹配的非伪类规则（除非 set_style 给）——base_style 就是 create_node 传入的 CSS parse 结果。伪类规则若匹配（如 :hover）由 dynamic_rules 每帧 rematch 叠加，同静态节点。

## 7. 动态树 API（D4）

### 7.1 场景驱动（brainstorm 确认的三场景）

1. **层级骨架**：初始化 stage 无 UI，程序建根 + 几个空 Container 当 layer（弹窗/飘字/HUD 层）。
2. **锚点挂载**：设计师 pkg 放锚点节点（带 id），程序 find_node_by_id 找到，动态建 UI 挂上去。
3. **动态图/文**：程序动态建 Image（set_src）/ Text（set_text）。

### 7.2 API 形态（Stage 方法 + FFI）

| Stage 方法 | FFI | 说明 |
|---|---|---|
| `create_root(kind, css) -> NodeId` | `loomgui_stage_create_root(h, kind, kind_len, css, css_len) -> u32` | 建根节点并设为 roots[0]。stage 无 scene 时用（初始化建根）。已有根则报错/替换。 |
| `create_node(kind, css) -> NodeId` | `loomgui_stage_create_node(h, kind, kind_len, css, css_len) -> u32` | 建孤立节点（未挂树），返回 NodeId。 |
| `append_child(parent, child)` | `loomgui_stage_append_child(h, parent, child)` | 挂 child 为 parent 末子。child 必须当前无父（孤立）。 |
| `insert_before(parent, child, ref)` | `loomgui_stage_insert_before(h, parent, child, ref)` | 挂 child 到 parent 中 ref 之前。ref=INVALID 末尾追加。 |
| `remove_child(parent, child)` | `loomgui_stage_remove_child(h, parent, child)` | 从 parent 摘 child（child 变孤立，节点仍存活）。 |
| `remove_node(node)` | `loomgui_stage_remove_node(h, node)` | 删节点：递归删子 + 联动清 anim/scroll/tween + slotmap remove。 |
| `set_text(node, text)` | `loomgui_stage_set_text(h, node, text, len)` | 改 Text 节点 content（非 Text 静默跳过/报错）。标 dirty_text。 |
| `set_src(node, src)` | `loomgui_stage_set_src(h, node, src, len)` | 改 Image 节点 src。标 dirty_mesh。 |
| `set_style(node, css)` | `loomgui_stage_set_style(h, node, css, css_len)` | parse css 改 base_style 指定字段。标 dirty。下帧 rematch 重算 style。 |

**入参 CSS 字符串**：`create_root`/`create_node`/`set_style` 接 CSS 声明串（如 `"width:100px;background:#f00;overflow:hidden"`）。复用 mapping.rs 的 `parse_*`（默认编译，零新依赖）。极简内联声明分割（`split(';')` + `split(':')`）不依赖 cssparser 全家桶/parse feature。

**入参 kind 字符串**：`"div"`/`"l-container"`/`"button"`/`"img"`/`"span"`，复用 dom.rs 的 tag→NodeKind 映射逻辑（提取为 runtime 可用的 `kind_from_tag(&str) -> NodeKind`，不依赖 parse feature）。

### 7.3 set_style 语义

`set_style(node, css)` parse css 得声明，逐字段应用到 `node.base_style`（覆盖指定字段，未出现的字段保留）。标节点 dirty，下帧 rematch 从新 base_style 重算 style。

**不重跑整树 cascade**：set_style 只改单节点 base_style，不触发 class/id 重新匹配。class 增删 + 整树重匹配推后（非目标）。

### 7.4 挂载/摘除的 layout 联动

`append_child`/`insert_before`/`remove_child` 改 `parent.children` + `child.parent`。下一帧 `solve` 重建 taffy 树（已每帧从零建，天然跟进），自动重算 layout_rect。无额外 layout 标脏——solve 每帧全量重算本就跟进结构变更。

**taffy_id 每帧重建**：solve 已每帧建新 taffy_ids 映射（调研 §3 确认），结构变更天然跟进。

### 7.5 悬空安全

代际 NodeId 保证：删节点后，C# 持有的旧 u32 句柄再用时，Rust 侧 `scene.nodes.get(id)` 返 None → 各 FFI no-op（沿用 v1 越界静默跳过）。tween 持有的旧 NodeId 由 §8 联动杀。input 持有的旧 NodeId（down_targets/last_hovered_chain）由现有 OOB guard + 祖先回退兜底（调研 §7 确认已有 `down_leaf_destroyed_fallback_to_ancestor` 测试）。

## 8. TweenManager 删节点联动（D7）

### 8.1 当前失效的防悬空

`tween.rs:175`：`if t.killed || t.node.0 >= n { continue; }` — 长度检查。代际化后 NodeId 不再是连续 usize，`t.node.0 >= n` 语义失效（u32 编码值远大于 n，恒真或恒假都不对）。

### 8.2 改为主动杀

`remove_node(id)` 联动调 `self.tweens.kill_node(id)`：杀该节点所有 prop 的 tween（标记 killed，update 跳过）。同 v1 既有 `Stage::clear_anim` 清 override 思路。

```rust
// TweenManager 新增
pub fn kill_node(&mut self, node: NodeId) {
    for t in &mut self.tweens { if t.node == node { t.killed = true; } }
}
```

**update 的 length guard 保留但语义改变**：`t.node` 存 NodeId（slotmap key）。update 时 `scene.anim.ensure(t.node)` 用 `t.node.index()`——若 t.node 已被 kill（联动杀），killed 标志先拦。若未 kill 但节点已删（不应发生，因 remove_node 必联动杀），slotmap.get(t.node) 返 None → ensure 仍能写（anim 是 Vec<Option> 按 index），但写到的位置已 clear_at 过 = 新 NodeAnim，无危害（孤儿 tween 写个没人读的 override）。**保险起见**，update 内加 `if scene.nodes.get(t.node).is_none() { t.killed = true; continue; }` 双保险。

## 9. world_transforms 时机前移（D6）

### 9.1 当前问题

v1 tick 顺序：`tween → focus → solve → refresh_content_sizes → [首帧guard: world_transforms] → process(hit_test 读 world_transforms) → scroll → keys → world_transforms → rematch → render`。

- `process` 用上帧 world_transforms（1 帧延迟，已认）。
- **结构变更帧**：上帧 world_transforms 长度 < 当前 nodes，hit_test `world_transforms[id.index()]` 越界。

### 9.2 前移修法

把"末尾 compute_world_transforms"挪到"solve + refresh_content_sizes 之后、process 之前"。新顺序：

```
tween → focus → solve → refresh_content_sizes → compute_world_transforms 
→ process(用本帧 world_transforms) → scroll → keys → rematch → render
```

- **消除 1 帧延迟**：hit_test 用本帧最新几何（solve 后）+ 最新 scroll_pos（refresh_content_sizes 后）。拖拽跟手更准。
- **结构变更安全**：compute_world_transforms 在 process 前，长度 = 当前 nodes 槽位数，hit_test 不越界。
- **删首帧 guard**：`if scene.world_transforms.is_empty() { compute_world_transforms }` 不再需要——每帧都算。
- **末尾不再算**：原 stage.rs:289 的 compute_world_transforms 删除（前移后末尾那次冗余）。

### 9.3 scroll_pos 同帧生效保留

原末尾算 world_transforms 的理由是"读 scroll_pos 同帧进 world matrix（零拖拽延迟）"。前移后：process（含 drag_follow 写 scroll_pos）在 compute_world_transforms **之前**？——不行，drag_follow 写的 scroll_pos 要进 world matrix。

**精确顺序**（保留 scroll_pos 同帧生效）：
```
tween → focus → solve → refresh_content_sizes 
→ process(drag_follow 写 scroll_pos) → scroll.advance → keys 
→ compute_world_transforms(读 scroll_pos) → rematch → render
```

即 compute_world_transforms 在 process/scroll 之后、render 之前。这样：
- world_transforms 长度 = 当前 nodes（结构变更安全，因 solve 在最前已建好）。
- scroll_pos 同帧进 world matrix（process 写、compute 读，同帧）。
- **hit_test 仍用上帧 world_transforms**：process 内 hit_test 在 compute_world_transforms 之前跑，读上帧。1 帧延迟保留（hit_test 1 帧延迟本就已认，clip 门控用 viewport 固定主导）。

**关键**：world_transforms 容量在 solve 后确定（nodes 结构定），compute 在 process 后算值。hit_test 读上帧值（长度对，因上一帧也 compute 过）。结构变更帧：上一帧 world_transforms 长度 = 上一帧 nodes，若本帧增了节点，hit_test 用上帧长度会越界？

**解决**：hit_test 访问 `world_transforms[id.index()]` 前，若 `id.index() >= world_transforms.len()`，该节点视为未命中（跳过）。即 hit_test 加 bounds guard（本就该有，调研 §7 指出 hit.rs:81 是 raw index 无 guard）。结构变更帧新增的节点本帧不参与命中（下一帧才算上），1 帧延迟语义一致。

## 12. 风险与不变量

### 12.1 风险

| 风险 | 缓解 |
|---|---|
| 70 处 Scene 字面量改 `Scene::build`/helper 工作量大 | 提 test helper `Scene::from_nodes(vec![...])` 统一改，机械批量 |
| slotmap `new_key_type!` 的 Key 编码可能撞 sentinel 0xFFFF_FFFF | 实现期验证；slotmap Key 编码通常含 version 高位，全 1 极不可能，但断言确认 |
| world_transforms 前移改变 hit_test 1 帧延迟语义 | hit_test 本就 1 帧延迟（已认），前移只改 world_transforms 长度安全，不恶化延迟 |
| set_style parse CSS 复用 mapping.rs 的覆盖语义（部分 parse 还是整字段替换） | 实现期定：set_style 覆盖出现的字段，未出现保留（内联样式语义） |
| 代际化后 NodeId.index() 与 slotmap capacity 的关系 | world_transforms/text_layouts 长度 = slotmap capacity（含空洞），按 index 索引不越界 |

### 12.2 核心不变量（final review 核对清单）

1. **NodeId 代际安全**：删节点后旧 NodeId 任何 API 用到 → no-op/None（slotmap.get 校验 gen）。
2. **remove_node 联动清**：anim/scroll/tween 全清（§5.2）。新附属表加入时必须同步加 remove_node 联动。
3. **base 是源 style 是派生**：set_style 改 base，rematch 每 frame 从 base 产 style，伪类不被冲。
4. **持久附属表 Option 化**：anim/scroll 是 `Vec<Option<T>>`，删时清 None，复用槽是新数据。
5. **临时数组每帧重建**：world_transforms/text_layouts 每帧 `vec![default; capacity]`，空洞无害。
6. **world_transforms 长度安全**：process 前 world_transforms 长度 ≥ 任何 live NodeId.index()；hit_test 加 bounds guard。
7. **FFI/C# 透明**：NodeId 仍是 u32，sentinel 0xFFFF_FFFF，C# 侧零感知。
