## 1. 声明面（CSS overflow 扩展）

扩展现有 `overflow` 值域 + 新增 per-axis longhand（浏览器标准，AI 先验最强，复用 overflow→clip_rect 路径）：

| CSS | 含义 | clip_rect |
|---|---|---|
| `overflow: visible`（默认） | 不裁剪不滚动 | None |
| `overflow: hidden` | 只裁剪（现状，零改） | Some(viewport) |
| `overflow: scroll` | 裁剪 + **始终可滚动** | Some(viewport) |
| `overflow: auto` | 裁剪 + **溢出时才可滚动** | Some(viewport) |
| `overflow-x` / `overflow-y` | per-axis longhand，覆盖 shorthand 同轴 | per-axis |

### 1.1 ResolvedStyle 字段变更

`ResolvedStyle.overflow_hidden: bool` → **删除**，替换为两字段：

```rust
#[derive(Debug, Clone, Copy, PartialEq, Default, Serialize, Deserialize)]
#[repr(u8)]
pub enum OverflowMode {
    #[default] Visible = 0,
    Hidden = 1,
    Scroll = 2,
    Auto = 3,
}
// ResolvedStyle:
//   pub overflow_x: OverflowMode,   // default Visible
//   pub overflow_y: OverflowMode,   // default Visible
```

- `#[repr(u8)]`（FFI/序列化稳定，坑 34 教训）+ `Serialize/Deserialize`（bincode 进 pkg，§3.7）。
- **pkg formatVersion 6→7**（ResolvedStyle bincode 布局变）。坑 17/43：bump 后 grep 全仓 fixture/构造点同步。
- `Default`：两轴均 `Visible`（与现状 `overflow_hidden=false` 等价 → 零回归）。

### 1.2 CSS 解析（`style/mapping.rs`）

- `overflow` shorthand：`visible/hidden/scroll/auto` → **同时**设 `overflow_x = overflow_y`。
- `overflow-x` / `overflow-y` longhand：设对应单轴（后于 shorthand 应用，覆盖）。
- 未知值 → `Visible`（宽松，不报错——与现有 overflow 解析一致）。

### 1.3 clip_rect 填充（`layout/mod.rs`，小改）

现状：`overflow_hidden` → `clip_rect = Some(自身 border 框)`。改为：**任一轴 ≠ Visible → `clip_rect = Some(border 框)`**（绝对坐标，layout solve 后填；border 框 ≥ content 框，clip 宽松无害）。clip 按容器整体 border 框（不 per-axis）——固定，滚动只平移子内容，shader 按此 clip。**scroll 数学用的 `viewport_size` = content box**（§2.3，layout_rect 减 padding/border），与子内容原点对齐；clip（border 框）仅作可见性边界。v1 简化：scroll 容器建议 `padding:0`（padding≠0 时 thumb/越界计算有微小偏移，完整 padding-edge 处理 v1.x）。

### 1.4 scroll 能力 derive（不另存 scroll_type）

每帧 solve 后，对每个 `overflow_x≠Visible || overflow_y≠Visible` 的节点算：
- `capable_x = overflow_x ∈ {Scroll, Auto}`（该轴**允许**滚动）
- `capable_y = overflow_y ∈ {Scroll, Auto}`
- `effective_x = capable_x && (overflow_x == Scroll || content_w > viewport_w)`（该轴**实际**可滚——有溢出）
- `effective_y` 同理

**仲裁/物理/滚动条统一用 `effective`**（该轴实际可滚）作单一判据；`capable` 仅作内部中间量。`overflow:auto` 无溢出 → effective=false → 不当候选、手势穿过（浏览器一致）；`overflow:scroll` → effective=true（overlap=0 时仍候选但滚不动，fgui 同款）。轴锁按 effective 轴判（effective_y && !effective_x 的容器遇水平手势让出）。

> **AI 可预测性**：`overflow:auto` 内容不溢出 = 不滚动、不显条（浏览器一致）；`overflow:scroll` 始终可滚（即使无溢出，浏览器一致）。derive 模型兑现两语义。

## 2. 数据模型（新 `loomgui_core/src/scroll.rs`）

镜像 TweenManager（§2.22）模式：transient 状态表 + Stage 持有/驱动。

### 2.1 ScrollPaneState（每滚动容器）

```rust
#[derive(Debug, Clone, Default)]
pub struct ScrollPaneState {
    // 几何（solve 后填，跨 tick 持续——物理依赖上帧 content_size）
    pub content_size: Vec2,      // 直接子节点 layout_rect AABB 的并（w,h）
    pub viewport_size: Vec2,     // 容器 content box 尺寸（layout_rect 减 padding/border）
    pub overlap: Vec2,           // max(0, content_size - viewport_size)
    // 偏移（运行时核心，物理/拖拽/滚轮写）
    pub scroll_pos: Vec2,        // 已滚过像素；x/y ∈ [-bounce, overlap+bounce]（回弹可越界）
    // 物理状态
    pub velocity: Vec2,          // 拖拽中记录的速度（exp 平滑）
    pub tweening: u8,            // 0=无, 1=编程 set_pos 缓动, 2=惯性+回弹
    pub tween_start: Vec2,
    pub tween_change: Vec2,      // 目标 = start + change
    pub tween_time: Vec2,        // 已 tween 时长（per-axis 独立）
    pub tween_duration: Vec2,
    pub content_size_dirty: bool,// content 变化标记（下帧重算 overlap 并按状态补偿）
}
```

`Vec2` = `(f32, f32)` 元组（core 无 Vec2 类型，与 `transform::apply_point` 同款）；实现用元组，下同。

## 3. offset 机制（关键，复用 v1d.3 world matrix + 现有 clip）

**`scroll_pos` 作虚拟平移注入 `compute_world_transforms` DFS**——零 DOM 改、零新节点类型。

### 3.1 注入点（`scene/transform.rs` rec）

现状（v1d.3 §2.21）：`local = T(rel) ∘ T(pivot) ∘ transform.matrix ∘ T(-pivot)`（rel = layout_rect.xy − 父.xy，pivot = box center），`world[N] = world[父] ∘ local[N]`，anim override `transform.matrix`。

加 scroll：处理节点 N 时，**父的 scroll offset 前乘**：

```
parent_scroll_off = if 父是滚动容器 { 父.scroll_pos } else { (0,0) }
world[N] = world[父] ∘ T(-parent_scroll_off) ∘ local[N]
```

- 滚动容器**自身** world 不含自己 scroll_pos（其 world 用它**父**的 offset）——viewport/bg/clip 固定。
- 滚动容器的**后代**前乘该容器 `T(-scroll_pos)` → 内容平移；嵌套滚动自然累积（每层父贡献自己 offset）。
- DFS 已知父 → 查 `scene.scroll.get(父_id)` 取 offset（与 anim.get 同模式）。

### 3.2 符号

`scroll_pos.y` 增大 = 向下滚（看更底部内容）→ 内容 `T(-scroll_pos.y)` 往 y 负移（上推）。`scroll_pos.x` 增大 = 向右滚 → 内容 x 负移。照 fgui `container.SetXY(-xPos, -yPos)`（ScrollPane.cs:1388）。

### 3.3 clip + 命中（零新增）

- **clip**：容器 `clip_rect`（viewport，固定）→ `assign_sort_keys` 开 MaskContext → shader `_ClipBox` 裁剪（§2.11 现有基建）。子顶点被 `-scroll_pos` 平移 → 滚出 viewport 部分被 shader 裁。**render/blob 零改**。
- **命中**：`hit.rs` `world_to_local = inverse(world).apply_point)`（v1d.3）天然含 scroll offset → 子节点命中自动跟随滚动。clip 门控用 viewport（固定）→ 点须在 viewport 内才命中可见内容。

### 3.4 blob 零改

scroll_pos 折进既有 world matrix（`m_a..m_ty`，blob v4 已有）→ **blob 保持 v4**，MirrorPool 零改。

## 4. 物理（自维护 tween，不走 GTween — §12.7 锁定）

`scroll.rs` 自维护可变 target tween，**禁 GTween 直接 tween scroll_pos**（API 层无此入口，避免双写）。fgui 实证直译，常量照搬。

### 4.1 常量

```rust
pub const SCROLL_THRESHOLD_MOUSE: f32 = 8.0;   // 手势阈值（fgui sensitivity PC）
pub const SCROLL_THRESHOLD_TOUCH: f32 = 20.0;  // fgui touchScrollSensitivity
pub const DECELERATION_RATE: f32 = 0.967;      // fgui defaultScrollDecelerationRate
pub const VELOCITY_DECAY_BASE: f32 = 0.833;    // 帧间速度衰减幂底
pub const VELOCITY_SMOOTH: f32 = 10.0;         // 速度平滑因子（Lerp 权重 = dt*此）
pub const INERTIA_THRESH_PC: f32 = 500.0;      // PC 惯性速度阈值 v²
pub const INERTIA_THRESH_TOUCH: f32 = 1000.0;
pub const INERTIA_DIST_COEFF: f32 = 0.4;       // 惯性距离经验系数
pub const TWEEN_TIME_DEFAULT: f32 = 0.3;       // 最小 tween 时长
pub const PULL_RATIO: f32 = 0.5;               // 拖拽越界打折 + 最大越界=viewport*此
pub const BOUNCE_THRESHOLD: f32 = 20.0;        // 超界 >此 触发回弹 tween
pub const SCROLL_STEP: f32 = 25.0;             // 滚轮每格像素（fgui defaultScrollStep）
pub const EASE: fn = cubic_out;                // (t/d-1)^3+1，所有 tween 共用
```

### 4.2 状态机

`tweening` 三态：`0=空闲` / `1=编程 set_pos 缓动` / `2=惯性+回弹`。

**拖拽跟手**（process Move，slot.scrolling_pane 已定 + 非 grip）：
- `delta = cur_pos - last_pos`；`new_pos = scroll_pos(at down) + 累计_delta`。
- 越界（new_pos < 0 或 > overlap）：`new_pos = boundary + (new_pos - boundary) * PULL_RATIO`，且越界量 ≤ `viewport * PULL_RATIO`。
- 记速度：`velocity = Lerp(velocity, delta/dt, dt*VELOCITY_SMOOTH)`；帧间 elapsed>1 先 `velocity *= VELOCITY_DECAY_BASE^elapsed`。

**Up → 惯性**（process Up，tweening 0→2）：
- `v² = velocity.x² + velocity.y²`；`v² < 阈值`(PC/触摸) → 不惯性，直接回弹（若越界）。
- 否则：`duration = log(60/v², DECELERATION_RATE) / 60`（每轴用各速度），min `TWEEN_TIME_DEFAULT`；`change = velocity * duration * INERTIA_DIST_COEFF`；`tween_start = scroll_pos`，`tween_time = 0`。

**advance**（scroll.update dt，tweening≠0）：
- `tween_time += dt`（per-axis）；`norm = EASE(min(tween_time, tween_duration) / tween_duration)`。
- `scroll_pos = tween_start + tween_change * norm`。
- **回弹**（tweening=2）：若 `scroll_pos` 越界且越界量 > `BOUNCE_THRESHOLD` → 重启回弹 tween：`tween_start=scroll_pos`、`tween_change = boundary - scroll_pos`（拉回边界）、`tween_duration = TWEEN_TIME_DEFAULT`、`tween_time = 0`。
- tween 完成（`tween_time ≥ tween_duration` 且未触发新回弹）→ `scroll_pos = clamp(scroll_pos, 0, overlap)`、`tweening = 0`。

**滚轮**（§6）：`target = scroll_pos - delta * SCROLL_STEP`（delta.y>0 上滚）→ 若 target 越界直接 clamp + 触发回弹；否则启 tweening=1 缓动到 target（cubic_out, `TWEEN_TIME_DEFAULT`）。

**编程 set_scroll_pos**（FFI）：`tweening=1`，`tween_start=scroll_pos`、`tween_change=target-start`、`tween_duration=TWEEN_TIME_DEFAULT`、`animated=0` 则直接 snap（`scroll_pos=target`、tweening=0）。

### 4.3 content_size 变化补偿（§12.7"不突变"）

solve 后若 `content_size_dirty`：重算 `overlap`；若 `scroll_pos > 新 overlap` 且 tweening → 按比例缩短 `tween_change`/`tween_duration`（照 fgui `FixDuration`），避免内容尺寸变时跳变。

## 5. 手势仲裁（per-slot，照 fgui 全局互斥）

fgui 用 `static ScrollPane draggingPane` + `static GObject draggingObject` 全局互斥；LoomGUI **per-slot**（每触摸独立，slot0=鼠标）+ slot 间天然隔离（不同指不互斥）。

### 5.1 TouchSlot 新字段（input.rs，仿 drag_testing/dragging）

```rust
// TouchSlot +：
pub scroll_candidate: Option<NodeId>,  // Down 候选滚动容器（最近 capable 祖先）
pub scroll_testing: bool,              // Down 命中候选，待阈值判定
pub scrolling_pane: Option<NodeId>,    // 已判定：本 slot 正在滚动的容器
pub scroll_gesture: u8,                // 轴标志 bit0=垂直手势 bit1=水平手势（fgui _gestureFlag per-slot）
pub grip_dragging: bool,               // grip 拖拽模式（§7）
// 拖拽跟手用的 down 时刻 scroll_pos / down_pos 复用既有 slot 指针状态（last_pos/down_pos）
```

### 5.2 Down 臂

1. **grip 优先**：`hit_scrollbar_grip(scene, point)`（§7）命中 → `grip_dragging=true`、`scrolling_pane=容器`、CaptureTouch、`click_cancelled=true`、return（不走 drag/scroll 候选）。
2. 否则 hit 沿祖先链找最近 **有 ≥1 `effective` 轴**（overflow:scroll，或 auto 且该轴 overlap>0）滚动容器 → `scroll_candidate`。
3. v1d.1 drag 候选照旧（`drag_target = down_targets.iter().find(draggable)`）。
4. `scroll_testing = scroll_candidate.is_some()`；`scroll_gesture = 0`。

### 5.3 Move 臂（阈值赛跑 + 轴锁）

`scroll_testing && !scrolling_pane && !dragging`（未判定）时：
- 算 per-axis 位移 `dx = |cur.x - down.x|`、`dy = |cur.y - down.y|`。
- **轴标志**：容器 `effective_y` 且 `dy>0` → `scroll_gesture |= 1`；`effective_x` 且 `dx>0` → `scroll_gesture |= 2`（两轴 effective 直接 `=3`）。
- **drag 先达**（v1d.1 阈值 mouse2/touch10）：drag 候选存在且位移超 drag 阈值 → 走 v1d.1 DragStart（`scroll_candidate=None`、`scroll_testing=false`）。
- **scroll 先达**（mouse8/touch20）：`scroll_candidate` 存在且位移超 scroll 阈值 → 判轴锁：
  - V-only 容器：若 `dx > dy`（水平位移更大）→ **让出**（`scroll_candidate` 提升到下一可滚祖先或清空，本 slot 不滚；fgui `diff < diff2` 放弃）。
  - H-only 容器：`dy > dx` → 让出。
  - Both：两轴都跟。
  - 锁定成功 → `scrolling_pane = scroll_candidate`、`click_cancelled = true`、CaptureTouch、记 down 时刻 `scroll_pos`、`velocity = 0`。
- **已判定 scrolling_pane**：拖拽跟手（§4.2），写 scroll_pos。

> scroll-start 必 `click_cancelled = true`（滚动非点击，照 fgui）。drag 与 scroll 在同 slot 互斥（先达者赢），不同 slot 独立。

### 5.4 Up 臂

- `scrolling_pane` 且 `!grip_dragging` → 触发惯性（§4.2 Up→惯性）；清 `scrolling_pane`/`scroll_testing`/`scroll_gesture`。
- `grip_dragging` → 清 grip（不惯性，直接停）。

### 5.5 嵌套（scroll-vs-scroll）

- Down 沿祖先链取**最近**（最内层）可滚容器为候选 → 最内层优先（fgui 静态互斥等价）。
- 内层**让出**（轴锁不符）→ 候选提升到下一可滚祖先（沿链向上找下一个 capable）；都让出 → 本 slot 不滚（手势交还 click/无操作）。
- 无"方向偷取"高级形态（fgui 亦无，v1.x）。

## 6. 滚轮输入（新输入通道）

### 6.1 WheelEvent（新 FFI 输入 struct）

### 6.3 core 处理（不产事件）

tick 消费 wheel：`hit_test(x,y)` → 沿祖先链找最近 **effective**（某轴 overlap>0）滚动容器 → 滚它（§4.2 滚轮：瞬移 + 回弹/tween）。无 effective 祖先 → 丢弃（不滚）。

- **不产 EVT_SCROLL/Wheel**（v1 无消费者；defer v1.x，照 fgui onMouseWheel 冒泡）。
- **嵌套滚轮透传 defer**（v1 只滚最内层 effective；滚到边界不自动交外层——v1.x）。
- **无 shift+水平**（fgui 亦无；水平靠内容仅水平溢出自动走 x 轴）。

## 7. 滚动条 + grip 拖拽（合成节点 + 专用命中）

LoomGUI 无合成（非 Scene）节点先例——本节是新路径。

### 7.1 合成 RenderNode（build_render_nodes 末尾追加）

对每个 **effective**（overlap>0）滚动容器，按轴追加 thumb（+ 可选 track）合成 RenderNode：

- **thumb 尺寸**：`thumb_h = viewport_h * (viewport_h / content_h)`（垂直），clamp `[min_thumb, track_h]`（min_thumb 如 20px 防极小）；水平同理。
- **thumb 位置**：`thumb_y = track_top + (track_h - thumb_h) * (scroll_pos.y / overlap.y)`。
- **node_id sentinel**：`v_thumb_id = container_id | 0x4000_0000`；`h_thumb_id = container_id | 0x2000_0000`（真实 NodeId 小，高位空闲；稳定 → MirrorPool 复用零抖动，坑 24 同理）。
- **渲染属性**：Mesh quad（半透明灰默认色 `[0.6,0.6,0.6,0.6]`），`sort_key` 在容器内容之后（在上层），`mask_context = 0`（不裁剪，贴 viewport 边缘常显）。payload 走既有 Mesh 路径 → **MirrorPool/blob 零特殊**（当普通 quad）。
- 容器无 overlap（不 effective）→ 不追加（scroll 条随 effective 显隐，兑现 `auto` 语义）。
- v1 砍：track 背景、箭头、点轨道跳页、fade 动画、CSS 定制外观（v1.x）。

### 7.2 grip 命中（hit 前置）

新 `hit_scrollbar_grip(scene, point) -> Option<(NodeId /*container*/, u8 /*axis*/)>`：
- 遍历 effective 滚动容器，算各 thumb design-rect（§7.1 位置/尺寸），point 命中 → 返回。
- `hit_test`（hit.rs）**前置**调它（scrollbar 在最上层）；命中 → 返 sentinel thumb_id 给 process（§5.2 Down grip 分支）。
- 不命中 → 走既有 scene hit_test。

### 7.3 grip 拖拽（§5.2 Down grip 分支 → Move/Up）

- Down on grip → `grip_dragging=true`、`scrolling_pane=container`、CaptureTouch、`click_cancelled=true`。
- Move：`perc = (cur - track_top) / (track_h - thumb_h)`；`scroll_pos = perc * overlap`（clamp [0, overlap]，grip 不越界）。
- Up：清 grip（不惯性）。

## 8. tick 管线（偏离 §12.7(2b')，spec 记 + session-summary 改主文档）

**顺序**：

```
1. tween.update(dt, scene, out)            // anim 写（v1d.4）
2. 消费 pending_focus_request              // focus（v1d.2）
3. solve(layout) → 写 layout_rect/clip_rect + 算 content_size → scene.scroll
4. process(pointer events)                 // 仲裁 + 拖拽跟手写 scroll_pos（§5）+ Down/Up
5. scroll.update(dt, scene, slots)         // 惯性/回弹 advance（§4.2）+ 消费 pending_wheel（§6）
6. process_keys                            // 键盘（v1d.2）
7. compute_world_transforms                // 读 anim.transform + scene.scroll offset（§3）
8. last_events = out
9. rematch_pseudo_classes
10. build_render_nodes                     // scene 节点 + 合成 scrollbar（§7）
```

### 8.1 偏离 §12.7(2b') 的理由

§12.7 文字："tick 时机 = §15 (2b')，紧随 GTween、在 style/layout 前"。LoomGUI **偏离**：scroll 物理（§5 step5）放 solve **后**、compute_world_transforms **前**。理由：

1. scroll 物理需 `content_size`（solve 产物）——无法在 solve 前。
2. 拖拽跟手是事件驱动（Move delta），必须在 process（step4）——其写的 scroll_pos 须在 compute_world_transforms（step7）前被读 → 拖拽**零视觉延迟**。
3. §12.7 真意"本帧 scroll 偏移进 transform 与命中"**满足**：step4+5 写 scroll_pos，step7 读，同帧生效。

### 8.2 命中 1 帧差（可接受）

step4 process 的 `hit_test` 用 step7 **上一帧**的 `world_transforms`（含上帧 scroll offset）——scroll offset 1 帧差。**不影响**：仲裁在 Down（本手势尚未滚动，scroll_pos 是滚动前值）判定；拖拽中不重仲裁；click 用 down_targets[0]（Down 定）。clip 门控用 viewport（固定）主导。照 fgui lazy（`_needRefresh` 下帧）同款。

> **改主文档 §12.7 + §15**：session-summary 阶段把 (2b') "在 style/layout 前" 改为 "在 solve 后、compute_world_transforms 前"，加 LoomGUI 偏离理由。

## 12. 不变量 / 风险

**不变量**：
1. `overflow:Visible`（默认）→ 无 clip_rect、无 scroll → 与 v1d.4 末态逐节点等价（零回归基线）。
2. `scroll_pos = (0,0)` → world transform 与无 scroll 等价（offset 注入对 0 是 no-op）。
3. scroll 物理只在 `capable` 容器作用；非 scroll 容器 ScrollTable 无项 → 无副作用。
4. scroll 不进 blob/pkg（transient）；blob v4 / MirrorPool / shader 零改。
5. scroll-start 必 cancel click；drag/scroll 同 slot 互斥（先达者赢）。

**风险**：
1. **pkg formatVersion 6→7 级联**（坑 17）：ResolvedStyle overflow 改 → 全仓 fixture/构造点同步（grep `overflow_hidden` / `Scene {` / pkg builder）。
2. **tick 顺序偏离 §12.7**：须实证命中 1 帧差不破仲裁；改主文档 §12.7+§15。
3. **合成 scrollbar 节点**（新概念）：MirrorPool 高位 sentinel id + 专用 `hit_scrollbar_grip` 是新路径；验稳定性（增删/嵌套/sort_key）。
4. **仲裁三重复杂度**（scroll-vs-drag + 嵌套 + 轴锁）：fgui 全局静态 → per-slot 移植易丢门控（坑 37 同族：命令式→recompute 丢 disabled 门控；此处 drag/scroll 互斥门控逐条复刻）。
5. **offset 注入 borrow**（坑 49 同族）：`compute_world_transforms` DFS 读 `scene.scroll` + 写 `worlds:Vec` 末尾赋值，避 `&mut/&` 冲突（v1d.3 已立模式，照搬）。
