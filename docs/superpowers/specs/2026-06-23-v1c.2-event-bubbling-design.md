# v1c.2 — 事件冒泡 / DOM 三阶段路由 设计

> 主设计文档 §10 定事件契约；本 spec 定 v1c.2 实现。v1c.1 建交互闭环（命中 + 单点事件 + 伪类 + is_pointer_on_ui），v1c.2 补**事件路由完整化**：bubble/capture 两阶段 + stop_propagation + 多 callback + RollOver/Out 祖先链 diff。
> **对照 fgui**：`temp/FairyGUI-unity/Assets/Scripts/Event/{EventDispatcher,EventContext,EventBridge,EventListener}.cs` + `Scripts/Core/Stage.cs`（HandleRollOver/BubbleEvent/ClickTest）。实现前先读这些，避免走歪（知识库 §0 工作准则）。

## 0. 决策记录（brainstorming 拍板）

- **路由位置 = 方向 A（C# 业务侧）**：核心产 target 事件，C# 沿 `node_parent` 链 bubble/capture 路由。理由：fgui 证据（`EventDispatcher`/`Stage` 整个事件管线在 C#，非"引擎无关核心"）+ stop 是回调副作用（必须在 C#）+ 核心最小改动。代价：主设计 §10.2/line 251 修订。
- **capture 阶段**：照 fgui 完整两阶段（capture 根→target 反向 + bubble target→根 正向）。
- **listener remove**：委托引用 remove（同 fgui），**不引入 ListenerId**（YAGNI，偏离主设计 §10.2 措辞）。
- **点 1（RollOver/Out 祖先链 diff）**：修，落核心 `hover_diff`（命中 diff 非路由，留核心）。
- **点 2（hovered 状态归属）**：不修，状态留核心（rematch 在核心，必须核心可读）。
- **Move 事件**：bubble（照 fgui onTouchMove，为 v1d drag 服务；标注高频慎监听）。
- **PreventDefault**：EventContext 加字段（v1c.2 无消费者，API 对称零成本）。
- **主设计修订**：spec 定稿时同步改 `00-main-design.md`（设计-实现一致）。

## 2. 职责切分（核心 ↔ C# 边界）

方向 A 的关键区分：**"命中 diff"是核心职责，"bubble 路由"是 C# 职责**。RollOver/Out 的"哪些节点进/出 hover"属命中 diff（核心），不 bubble（fgui 也 `DispatchEvent` 直派）。

| 职责 | 归属 | 说明 |
|---|---|---|
| hit_test（几何命中） | **核心** | §10.1，每事件算（v1c.1 现状） |
| PointerState 状态机（click 阈值 / down_node） | **核心** | 紧耦合 hit_test，留核心（v1c.1 不变） |
| hover diff → RollOver/Out 产出（**祖先链 diff**，点 1） | **核心** | 命中 diff，产多条 EventRecord |
| hovered/active 状态 + set_*_chain | **核心** | 供 rematch（点 2，不动） |
| 伪类 rematch | **核心** | 改 Node.style |
| **bubble/capture 路由 + stop + EventContext + listener 表** | **C#** | fgui 业务侧；stop 是回调副作用 |
| listener 注册/移除（多 callback，委托 remove） | **C#** | 对齐 fgui EventBridge |

一句话：核心决定"**产什么事件、给哪个 target**"（含 RollOver/Out 多目标 diff），C# 决定"**事件沿链怎么派发到 listener**"。

## 4. 核心侧改动

### 4.1 hover_diff 改祖先链 diff（点 1）

v1c.1 单点 diff（旧 target RollOut + 新 target RollOver）→ 改 fgui `HandleRollOver`（Stage.cs:1315-1380）等价两链 diff：

```rust
// PointerState（input.rs）：last_hovered: Option<NodeId> → 改
pub last_hovered_chain: Vec<NodeId>,   // 上帧 hovered 链（target→root）

fn hover_diff(&mut self, scene: &mut Scene, out: &mut Vec<EventRecord>) {
    let cur = hit_test(scene, self.last_pos);
    let new_chain: Vec<NodeId> = ancestor_chain(scene, cur);  // cur + parent 链至 root；None → 空
    if new_chain == self.last_hovered_chain { return; }        // 幂等
    // 集合 diff：共同祖先段不产事件（鼠标从父进子 → 父不 RollOut）
    for n in &self.last_hovered_chain {
        if !new_chain.contains(n) { out.push(RollOut(n)); }
    }
    for n in &new_chain {
        if !self.last_hovered_chain.contains(n) { out.push(RollOver(n)); }
    }
    set_hovered_chain(scene, cur);          // hovered 状态仍设新链（rematch 用，不变）
    self.last_hovered_chain = new_chain;
}
```

`ancestor_chain(scene, target)`: `target` 起沿 `Node.parent` 至 root 收集（核心 `Node.parent: Option<NodeId>` 现成）；`target=None`（命中根外）→ 空链（→ 整条旧链 RollOut）。

**时序不变**：`hover_diff` 仍在 `process` 每个事件后跑（input.rs:201），保证 RollOver 在同帧后续事件前 + 幂等。

### 4.2 FFI 加 node_parent

```rust
/// 返 parent node_id；根/越界 → 0xFFFFFFFF（sentinel）。C# 路由沿链用。
/// 常驻不 gate（runtime 入口，同 load_package 纪律，坑 21）。
#[no_mangle]
pub extern "C" fn loomgui_node_parent(h: *const StageHandle, node_id: u32) -> u32
```

实现：`h` 非 null + `node_id < scene.nodes.len()` → `scene.nodes[node_id].parent.map(|p| p.0 as u32).unwrap_or(0xFFFFFFFF)`；越界/无 scene → sentinel。

### 4.3 EventRecord 零改（关键洞察）

`EventRecord { node_id: u32, event_type: u8, x: f32, y: f32 }` **一字段不加**：

- bubble 类（Down/Up/Move/Click）：`node_id` = target，C# 收到后**自己沿 node_parent 链**展开 capture+bubble。currentTarget/phase 由 C# `EventContext` 维护。
- RollOver/Out：核心已 diff 出**多条** EventRecord（每条 node_id = 进/出 hover 的具体节点），C# 直派。
- C# 按 `event_type` 硬分流：`{Down,Up,Move,Click}` → bubble；`{RollOver,RollOut}` → 直派（同 fgui 按类型选 BubbleEvent/DispatchEvent）。

**event blob 零改 = FFI 契约稳定**，只新增 `node_parent`。无 version bump 级联（坑 17 不触发）。

### 4.4 其余零改

Down/Up/Move/Click 产出逻辑、PointerState 状态机（click 阈值/down_node/disabled 抑制）、`set_hovered_chain`/`set_active_chain`、`rematch_pseudo_classes`、`.pkg.bin` v3、frame blob、MirrorPool、shader——**全零改**。version 串 `v1c.1 → v1c.2`。

## 5. C# 路由层（fgui EventDispatcher 重实现）

### 5.1 EventBridge（替代 v1c.1 单 callback）

```csharp
class EventBridge {
    EventCallback _bubble;    // bubble 阶段（多播委托，Add 内 -= 去重）
    EventCallback _capture;   // capture 阶段
    public void Add(EventCallback cb)        { _bubble -= cb; _bubble += cb; }
    public void AddCapture(EventCallback cb) { _capture -= cb; _capture += cb; }
    public void Remove(EventCallback cb) => _bubble -= cb;
    public void RemoveCapture(EventCallback cb) => _capture -= cb;
    public bool isEmpty => _bubble == null && _capture == null;
    public void CallBubble(EventContext ctx) { /* sender=owner, 调 _bubble */ }
    public void CallCapture(EventContext ctx) { /* 仅 _capture 非 null 时调 */ }
}
```
照 fgui EventBridge.cs（多播 + capture/bubble 两组）。v1c.1 单 callback 覆盖 → 多播（breaking，v1c.1 刚 merged 无外部消费者，sample 一起改）。

### 5.2 EventContext（照 fgui，对象池）

```csharp
class EventContext {
    public uint target;            // 原始命中（EventRecord.node_id）
    public uint currentTarget;     // 路由当前节点
    public Phase phase;            // Capture / Target / Bubble
    public EventType type; public float x, y;
    internal bool _stopsPropagation, _defaultPrevented;
    public void StopPropagation()  => _stopsPropagation = true;
    public void PreventDefault()   => _defaultPrevented = true;
    static Stack<EventContext> pool;   // Get/Return；[RuntimeInitializeOnLoadMethod(SubsystemRegistration)] 清池（照 fgui EventContext.cs:96-102）
}
public enum Phase : byte { Capture = 0, Target = 1, Bubble = 2 }
public delegate void EventCallback(EventContext ctx);
```

### 5.4 DispatchPending 分流

```csharp
public void DispatchPending(IntPtr ptr, int count) {
    if (ptr == IntPtr.Zero || count <= 0) return;
    int recSize = Marshal.SizeOf<LoomEvent>();
    for (int i = 0; i < count; i++) {
        var evt = Marshal.PtrToStructure<LoomEvent>(ptr + i * recSize);
        switch (evt.type) {
            case EventType.Down: case EventType.Up:
            case EventType.Move: case EventType.Click: BubbleRoute(evt); break;
            case EventType.RollOver: case EventType.RollOut: DirectDispatch(evt); break;
        }
    }
}
```

### 5.5 BubbleRoute（照 fgui BubbleEvent, EventDispatcher.cs:286-350）

```csharp
void BubbleRoute(LoomEvent evt) {
    var chain = AncestorChain(evt.nodeId);          // [target,...,root]
    var ctx = EventContext.Get();
    ctx.target = evt.nodeId; ctx.type = evt.type; ctx.x = evt.x; ctx.y = evt.y;
    // capture: 根→target 反向；不检查 stop（照 fgui line 302-311 全跑）
    for (int i = chain.Count - 1; i >= 0; i--) {
        ctx.currentTarget = chain[i]; ctx.phase = Phase.Capture;
        FireCapture(chain[i], ctx);
    }
    // bubble: target→root 正向；stop break（照 fgui line 315-328）
    if (!ctx._stopsPropagation) {
        for (int i = 0; i < chain.Count; i++) {
            ctx.currentTarget = chain[i];
            ctx.phase = (chain[i] == ctx.target) ? Phase.Target : Phase.Bubble;
            FireBubble(chain[i], ctx);
            if (ctx._stopsPropagation) break;
        }
    }
    EventContext.Return(ctx);
}

void FireBubble(uint nodeId, EventContext ctx)
    => TryGetBridge(nodeId, ctx.type)?.CallBubble(ctx);
void FireCapture(uint nodeId, EventContext ctx)
    => TryGetBridge(nodeId, ctx.type)?.CallCapture(ctx);
```

### 5.6 DirectDispatch（照 fgui DispatchEvent→InternalDispatchEvent:226-229）

RollOver/Out：核心已 diff 多目标，每条**单节点**直派（不沿链），但仍跑该节点 capture+bubble 回调（照 fgui DispatchEvent）：

```csharp
void DirectDispatch(LoomEvent evt) {
    var ctx = EventContext.Get();
    ctx.target = ctx.currentTarget = evt.nodeId; ctx.phase = Phase.Target;
    ctx.type = evt.type; ctx.x = evt.x; ctx.y = evt.y;
    var bridge = TryGetBridge(evt.nodeId, evt.type);
    if (bridge != null) { bridge.CallCapture(ctx); bridge.CallBubble(ctx); }
    EventContext.Return(ctx);
}
```

### 5.7 AncestorChain + node_parent 缓存

```csharp
List<uint> AncestorChain(uint target) {
    var chain = new List<uint>();
    uint c = target;
    while (c != 0xFFFFFFFF) { chain.Add(c); c = ParentOf(c); }
    return chain;   // [target, parent, ..., root]
}
uint ParentOf(uint nodeId) {
    if (!_parentCache.TryGetValue(nodeId, out var p))
        _parentCache[nodeId] = p = Native.loomgui_node_parent(_handle, nodeId);
    return p;
}
```

### 9.3 零改契约

- EventRecord / event blob / frame blob / MirrorPool / MaterialManager / TextRasterizer / shader / `.pkg.bin` v3：**零改**。
- 核心 Down/Up/Move/Click 产出 / PointerState 状态机 / rematch / set_*_chain：**零改**。

### 9.4 AI 可预测性约束

- 路由语义照 fgui/DOM（capture 反向 + bubble 正向 + stop），AI 对 DOM 事件模型强先验。
- listener 在 C# 显式注册（AddListener/AddCapture），谁监听谁收，无核心侧隐式派发。
- RollOver/Out 祖先链 diff 对齐 CSS :hover 祖先语义（鼠标在元素或后代上 → 元素 hover）。
- 事件产出确定性：同输入序列产同事件序列（状态机无随机/时间依赖）。

### 9.5 已知简化（围栏文档/sample 须告知）

- RollOver/Out 是**单节点直派**（核心 diff 出多目标，每条在该节点直派，不 bubble 沿链）——对齐 fgui `DispatchEvent`（RollOver/Out 本就不冒泡）；祖先链 diff 已保证父不误 RollOut。
- capture 阶段不检查 stop（照 fgui 实际行为，非 DOM 严格语义）——围栏文档标注。
- Move bubble 高频（每鼠标移动触发 + 沿链派发），业务慎监听 Move；hover 反馈用 RollOver/Out 或 :hover 伪类。
