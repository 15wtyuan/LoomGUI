# v1d.4 设计：GTween 时钟 + 基础 tween

> **范围来源**：`docs/roadmap/v1d-plan.md` §1 #8 / §3 v1d.4 / §4 v1d.4。
> **依赖**：无硬依赖；建议 v1d.3 后（transform 已支持，tween 能动 translate/scale 更值）—— v1d.3 已完成。
> **不在 v1 验收 6 项内**：纯基础设施 + demo 打磨；v1d.5 ScrollPane 自带 tween，**不依赖本引擎**。

---

## 0. 目标

一个最小可用的 tween 引擎：`TweenManager` 驱动 + `tween(node, prop, start, end, dur, ease)` + delay + onComplete（事件通道）。可动 opacity / transform(translate·scale·rotate) / 颜色(bg·text)。easing 子集 10 个。

## 1. 决策日志（brainstorming 期定）

| 决策 | 选择 | 理由 |
|---|---|---|
| 动什么 | opacity + transform(translate/scale/rotate) + 颜色(bg/text) | 覆盖弹窗动画全档；transform 是 v1d.4 值所在 |
| transform 落点 | **replace-override**（动画覆盖 css_matrix 读取点） | 不重开 v1d.3 pivot/world compose 坑；单 transform 源对 AI 最可预测；混用嵌套 div |
| dt 时钟 | 单 unscaled dt（零 FFI 改动） | 菜单暂停照播动画；advance_time 已收 unscaled dt |
| onComplete | 事件通道 EVT_TWEEN_COMPLETE（复用 borrow_events） | 零新 FFI 回调机制；对齐 core 产事件模式 |
| easing 子集 | Linear + Quad/Cubic/Back 的 In/Out/InOut（10） | 覆盖 overshoot；照 fgui EaseManager 数学直译 |
| 生命周期 | delay + onComplete | 弹窗基本两件套；repeat/yoyo 无消费者 |

## 2. 不做（YAGNI，推 v1.x）

repeat/yoyo/breakpoint/snapping/path tween；per-tween `timeScale`/`ignoreEngineTimeScale`（含双 dt 参数）；skew/matrix tween；颜色以外属性（position/size——layout 动画）；Transition/Controller/Gear 编排；路径 tween；`ITweenListener`/双参回调；tween 池预分配容量配置。

---

## 3. 架构总览

新增 `loomgui_core/src/tween.rs`：`TweenManager` + `Tween` + `Ease` + `TweenProp`。`Stage` 持有 `pub tweens: TweenManager`。

**Scene 新增 transient 字段** `anim: AnimTable`（同 `world_transforms`，运行时态、不进 pkg、`#[derive(Debug, Clone)]` Scene 无需 serde skip）。每 tick 由 `TweenManager.update` 写入，由 `compute_world_transforms` / `build_render_nodes` 读。

### tick 顺序（`Stage::tick_and_render` 顶部插入 tween update）

```
0. tween.update(dt, scene, &mut out)   ← 写 scene.anim + 产 EVT_TWEEN_COMPLETE 进 out
1. solve(scene, ...)
2. compute_world_transforms(scene)     ← 读 anim.transform override
3. process 输入（pointer + key）
4. rematch_pseudo_classes
5. build_render_nodes(...)             ← 读 anim.opacity/bg_color/text_color override
```

**零回归保证**：`anim` 全 None 时，所有 override 退回 CSS 值，渲染/命中与 v1d.3 逐字节一致。现有 core 测试不经 tween → dt=0 → 不推进 → 输出不变。

## 4. 数据模型

### 4.1 `scene::anim`（`scene/node.rs` Scene 新字段 + `scene/anim.rs` 或并入 node.rs）

```rust
/// 每节点动画 override 表（index = NodeId.0，懒增长到 scene.nodes.len()）。
/// 全 None = 无动画 → 渲染退回 CSS ResolvedStyle 值。
#[derive(Debug, Clone, Default)]
pub struct AnimTable(pub Vec<NodeAnim>);

#[derive(Debug, Clone, Copy, Default, PartialEq)]
pub struct NodeAnim {
    pub opacity: Option<f32>,
    pub transform: Option<crate::transform::Affine2>,  // replace-override css_matrix
    pub bg_color: Option<[f32; 4]>,
    pub text_color: Option<[f32; 4]>,
}
```

`AnimTable::get(node_id) -> Option<&NodeAnim>` / `ensure(&mut self, n)` 增长到 n 并返回 `&mut [NodeAnim]`。`NodeAnim::is_empty()`（全 None）→ render 跳过 override 查表（热路径快路径）。

### 4.2 `tween.rs`

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
#[repr(u8)]
pub enum TweenProp { Opacity = 0, Translate = 1, Scale = 2, Rotation = 3, BgColor = 4, TextColor = 5 }

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
#[repr(u8)]
pub enum Ease {
    Linear = 0, QuadIn, QuadOut, QuadInOut,
    CubicIn, CubicOut, CubicInOut, BackIn, BackOut, BackInOut,
}
impl Ease {
    /// t∈[0,dur] → [0,1]。dur<=0 直接返 1。
    pub fn evaluate(self, t: f32, dur: f32) -> f32 { ... }
}

#[derive(Debug, Clone)]
struct Tween {
    node: crate::scene::NodeId,
    prop: TweenProp,
    start: [f32; 4], end: [f32; 4], value_size: u8,  // size 由 prop 定
    ease: Ease,
    delay: f32,
    duration: f32,
    elapsed: f32,
    tag: u32,
    started: bool,   // 首次越过 delay 时置 true
    killed: bool,
}

#[derive(Debug, Default)]
pub struct TweenManager { tweens: Vec<Tween> }  // active 列表；v1 不做 pool（YAGNI）

impl TweenManager {
    pub fn new() -> Self;
    /// 注册一个 tween。start/end 取前 value_size 个分量。返回 ()（v1 无 handle）。
    pub fn tween(&mut self, node, prop, start: [f32;4], end: [f32;4], ease, delay, dur, tag);
    /// 停止该节点该 prop 的 tween（killed=true，override 保留末值）。
    pub fn kill(&mut self, node, prop);
    /// 清该节点所有 override（回 CSS）。
    pub fn clear_anim(&mut self, anim: &mut AnimTable, node);
    /// 清该节点某 prop 的 override。
    pub fn clear_anim_prop(&mut self, anim: &mut AnimTable, node, prop);
    /// 每 tick 调：推进所有 active tween，写 anim，产 complete 事件。dt 由 Stage 传。
    pub fn update(&mut self, dt: f32, scene: &mut Scene, out: &mut Vec<EventRecord>);
}
```

**value_size 映射**：`Opacity`=1、`Rotation`=1、`Translate`=2、`Scale`=2、`BgColor`=4、`TextColor`=4。

### 4.3 `update` 算法（核心）

```
ensure anim 到 scene.nodes.len()
for t in tweens (索引 i，记录 free slot 用于就地压缩):
    if t.killed → 标记删除，continue
    t.elapsed += dt
    if t.elapsed < t.delay → continue            // delay 门控，不写 override
    if !t.started → t.started = true              // （v1 无 onStart 回调，仅标记）
    tt = t.elapsed - t.delay
    n  = t.ease.evaluate(min(tt, t.duration), t.duration)   // 钳到 [0,1]
    cur = lerp(t.start, t.end, n, t.value_size)              // 逐分量
    apply(t.node, t.prop, cur, &mut anim)
    if tt >= t.duration:
        apply 末值（n=1，已含）   // 确保停在 end
        out.push(EVT_TWEEN_COMPLETE { node=t.node, prop, tag=t.tag })
        t.killed = true
压缩 tweens：移除 killed（swap_remove 或 retain，保序用 retain）
```

**lerp**：`cur[k] = start[k] + (end[k]-start[k]) * n`，k∈0..value_size。

**apply(node, prop, cur, anim)**：
- `Opacity` → `anim[node].opacity = Some(cur[0])`
- `Translate` → `anim[node].transform = Some(transform::from_translate(cur[0], cur[1]))`
- `Scale` → `anim[node].transform = Some(transform::from_scale(cur[0], cur[1]))`
- `Rotation` → `anim[node].transform = Some(transform::from_rotate(cur[0]))`  // rad
- `BgColor` → `anim[node].bg_color = Some([cur[0],cur[1],cur[2],cur[3]])`
- `TextColor` → `anim[node].text_color = Some([cur[0],cur[1],cur[2],cur[3]])`

**一节点一 transform tween 规则**：`anim[node].transform` 是单一 `Option<Affine2>`。同节点并发多个 transform tween → 后 apply 者覆盖（last-write-wins per tick）。文档写明：要 scale+translate 同节点 → 嵌套 div（HTML 惯例）。Opacity/Color 是独立通道，与 transform tween 并发无冲突。

**持久语义**：tween killed（自然完成或 `kill`）→ 其写入的 override **保留末值**（fgui/CSS `fill:forwards`）。回 CSS 须显式 `clear_anim(_prop)`。

## 5. 属性应用点（replace-override）

| 属性 | 读取处（现有代码） | 改成 |
|---|---|---|
| Opacity | `render/mod.rs` build_render_nodes `alpha = style.opacity` | `anim.get(id).opacity.unwrap_or(style.opacity)` |
| Transform | `scene/transform.rs` compute_world_transforms 读 `style.transform.matrix` | `anim.get(id).transform.unwrap_or(style.transform.matrix)` |
| BgColor | `render/mod.rs` Mesh color ← `style.background_color` | `anim.get(id).bg_color.or(style.background_color)` |
| TextColor | `render/mod.rs` Text color ← `style.color` | `anim.get(id).text_color.unwrap_or(style.color)` |

> `anim.get(id)` 返回 `Option<&NodeAnim>`；None 或 `NodeAnim::is_empty()` → 直接退回 CSS（热路径零开销）。

**hit / world_to_local**：命中用 `scene.world_transforms`，而 world_transforms 已读 anim.transform override（见上）→ 命中自动跟随动画，无需额外改 hit.rs。

## 6. 时钟

单 unscaled dt，零 FFI 改动：

- `Stage` 新增字段 `pending_dt: f32`（默认 0）。
- `Stage::advance_time(dt)`（FFI `loomgui_stage_tick` 已先调）末尾加 `self.pending_dt = dt;`。
- `Stage::tick_and_render` 顶部：`let dt = self.pending_dt; self.pending_dt = 0.0; let mut out = Vec::new(); self.tweens.update(dt, scene, &mut out);`（out 续接后续 process 产的事件，统一成 last_events）。

**为什么 stash 而非给 tick_and_render 加 dt 参数**：`tick_and_render()` 现被 ~10 处 core 测试无参调用；加参要逐个改 `0.0`。stash 法 1 字段 + 2 行，零测试 ripple；且现有测试不走 advance_time → pending_dt=0 → tween 不推进 → 零回归。

per-tween `timeScale`/`ignoreEngineTimeScale` defer v1.x（无消费者）。

## 7. easing（10 个，照 fgui EaseManager 直译）

BSD 兼容（Robert Penner / Daniele Giardini）。`Ease::evaluate(self, t: f32, dur: f32) -> f32`：

```rust
const OVERSHOOT: f32 = 1.70158;  // Back 默认
if dur <= 0.0 { return 1.0; }
match self {
    Linear      => t / dur,
    QuadIn      => { let t=t/dur; t*t },
    QuadOut     => { let t=t/dur; -t*(t-2.0) },
    QuadInOut   => { let t=t/(dur*0.5);
                     if t<1.0 { 0.5*t*t } else { let t=t-1.0; -0.5*(t*(t-2.0)-1.0) } },
    CubicIn     => { let t=t/dur; t*t*t },
    CubicOut    => { let t=t/dur-1.0; t*t*t+1.0 },
    CubicInOut  => { let t=t/(dur*0.5);
                     if t<1.0 { 0.5*t*t*t } else { let t=t-2.0; 0.5*(t*t*t+2.0) } },
    BackIn      => { let t=t/dur; t*t*((OVERSHOOT+1.0)*t-OVERSHOOT) },
    BackOut     => { let t=t/dur-1.0; t*t*((OVERSHOOT+1.0)*t+OVERSHOOT)+1.0 },
    BackInOut   => { let s=OVERSHOOT*1.525; let t=t/(dur*0.5);
                     if t<1.0 { 0.5*(t*t*((s+1.0)*t-s)) }
                     else { let t=t-2.0; 0.5*(t*t*((s+1.0)*t+s)+2.0) } },
}
```

测试已知值：各 ease 在 t=0 → 0、t=dur → 1；BackOut 中段 >1（overshoot）；CubicIn 中段 < Linear。

## 8. 生命周期 + 事件

- **delay**：`elapsed < delay` 时跳过 apply（不写 override）；越过 delay 首帧 n 从 0 起。
- **onComplete = `EVT_TWEEN_COMPLETE = 16`**（`input.rs` 加 const）。**复用 EventRecord 现有字段，零结构改动**：
  ```rust
  EventRecord { node_id: node.0 as u32, event_type: EVT_TWEEN_COMPLETE,
                click_count: prop as u8,  // 复用：prop 枚举值 0..5
                pad: [0,0], touch_id: tag as i32,  // 复用：调用方 tag
                x: 0.0, y: 0.0 }
  ```
  C# 事件循环按 `event_type==16` 分发，读 `click_count`=prop、`touch_id`=tag，靠 tag 识别链式动画。
- **kill/clear**：
  - `kill_tween(node, prop)` → 该 tween `killed=true`（停 update），**override 保留末值**。
  - `clear_anim(node)` → `anim[node]` 全通道置 None → 回 CSS。
  - `clear_anim_prop(node, prop)` → 仅清该 prop 对应通道（Opacity/Translate/Scale/Rotation/BgColor/TextColor → opacity/transform/transform/transform/bg_color/text_color）。

## 13. 测试

### Rust unit（`tween.rs`，`#[cfg(test)]`）
- `ease_*`：10 个 ease 各自在 t=0 → 0、t=dur → 1；BackOut/BackIn/BackInOut 中段 overshoot 方向正确；CubicIn(0.5) < Linear(0.5) < CubicOut(0.5)。

## 14. 关键不变量（实现期守住）

1. **零回归**：anim 全 None → 渲染/命中 == v1d.3。
2. **replace-override**：动画 transform 覆盖 css_matrix 读取点，不 compose；一节点一 transform tween（并发 last-write-wins，文档+测试覆盖）。
3. **持久**：tween killed → override 保留末值；回 CSS 须显式 clear。
4. **单 unscaled dt**：advance_time stash pending_dt；tick_and_render 消费。
5. **EVT 16 零结构改动**：复用 click_count=prop、touch_id=tag。
6. **blob/MirrorPool 零改动**：anim 在 core fold 进既有字段，blob 仍 v4。

## 15. 风险 / 已知边角

- **一节点一 transform tween**：scale+translate 同节点需嵌套 div。文档写明，测试覆盖 last-write-wins。
- **dt stash 与 FFI tick 顺序**：`loomgui_stage_tick` 必须先 `advance_time(dt)` 再 `tick_and_render`（现状即如此），否则 tween 拿不到 dt。测试覆盖。
- **`duration<=0`**：立即 complete（n=1），产事件——Stage::tween 校验。
- **tween 跨 load 重置**：load_inline/load_package 重建 scene → `anim` 重置为空，但 `Stage.tweens`（独立字段）残留旧 tween（指向失效 node_id）。**处理**：load 时 `self.tweens.clear()`（TweenManager 加 `clear()`）。测试覆盖。
