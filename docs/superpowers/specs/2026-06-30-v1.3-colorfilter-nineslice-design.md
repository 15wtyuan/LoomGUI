# v1.3 ColorFilter + 九宫格 slice 设计

> **定位**：v1.x 第 3 项（编号统一 2026-06-30）。绘制质量层——ColorFilter 替代 v1 简化 grayed 灰化、支持标准 CSS `filter` 色调统一；九宫格 `border-image-slice` 让 UI 皮肤缩放不变形（配合 v1.1 bg-image）。profiling 作验收门槛。摸 fgui `ColorFilter.cs` + `Image.cs SliceFill` 过河。
> **依据**：roadmap.md §1 v1.3 + fgui 对照（`temp/FairyGUI-unity/Assets/Scripts/Filter/ColorFilter.cs`、`Core/Image.cs:286`）+ design §4.4 border-image-slice canonical。

---

## 0. 目标 + 非目标

**目标**：
- ColorFilter：4×5 颜色矩阵，支持标准 CSS `filter` 函数（grayscale/brightness/contrast/saturate/hue-rotate/invert/sepia），替代 v1 简化 `grayed` 顶点色灰化。disabled 灰化 = `filter:grayscale(1)`。
- 九宫格：`border-image-slice`（标准 CSS，4 值上右下左）让背景图/`<img>` 四角不缩放、四边单轴拉伸、中心自由拉伸。配合 bg-image 做按钮/面板皮肤。
- slice + border-radius 共存：四角圆角扇形 + 中间切片（nine_slice_rounded）。
- profiling 验收门槛：draw call/GC/内存实机达标，零回归。

**非目标**（YAGNI）：
- 任意 4×5 矩阵的自定义 CSS 语法（`-l-color-matrix: a,b,...`）——留后补，v1.3 只做标准 filter 函数。
- ColorFilter 动画 tween 通道（用 opacity 淡入足够）。
- `border-image-source`（标准 CSS 另一属性）——LoomGUI 收窄为 slice 配 `background-image` 单源，不支持双源共存。
- 九宫格平铺（`border-image-repeat:round/repeat`）——v1.3 只做 slice 默认 stretch。
- ColorFilter 对 Text 节点（program=1）——Text 走 ALPHA_MASK，v1.3 ColorFilter 只对 Mesh（program=0/2→3）。Text 灰化用 `color` 顶点色近似。
- 九宫格对 Container 无 bg-image 的情形（无图无切片意义）。

---

## 1. 方案选择

### ColorFilter（独立 program 不批合，对齐 fgui）

**采用**：加 `program=3`（ColorFilter），4×5 矩阵走 MaterialPropertyBlock（per-renderer 覆盖，不拆 Material）。Material key `(texture, program, mask_ctx)` 不变——program=3 天然与 program=0/2 不合批 = fgui"独立 DrawCall"语义。

**否决**：
- filter_id 索引 + CBUFFER 保批合（方案 2）——过度工程。ColorFilter 节点数量少（disabled 按钮几个），批合省不出 draw call，却要横跨 Rust/C#/HLSL 四层搭索引表。YAGNI。
- 只做预设不开放矩阵（方案 3）——违背 roadmap"4×5 颜色矩阵"承诺，色调统一是刚需。

### 九宫格（照搬 fgui SliceFill 几何）

**采用**：`mesh::nine_slice` 产 16 顶点（4×4）+ 9 quad（`TRIANGLES_9_GRID` 固定索引照搬 fgui）。CSS `border-image-slice` 4 值 → 源图像素 → fgui gridRect 转换。slice+radius 共存自设计 `mesh::nine_slice_rounded`（四角圆角扇形 + 中间切片）。

**否决**：
- 九宫格 defer 到后号——你定（"后面也是要改成共存的，不如现在一次到位"）。
- fgui `scale9Grid` 命名——LoomGUI 用标准 CSS `border-image-slice`（AI 可预测 + 浏览器原生渲染）。

## 2. CSS 子集

### ColorFilter（标准 CSS `filter`）

| 属性 | 值 | 语义 | 对应矩阵（fgui 预设） |
|---|---|---|---|
| `filter` | `grayscale(1)` | 灰化（disabled） | `AdjustSaturation(-1)` |
| | `brightness(n)` | 亮度 n∈[0,∞) | `AdjustBrightness(n-1)` |
| | `contrast(n)` | 对比度 | `AdjustContrast(n-1)` |
| | `saturate(n)` | 饱和度 | `AdjustSaturation(n-1)` |
| | `hue-rotate(deg)` | 色相旋转 | `AdjustHue(deg/180)` |
| | `invert(1)` | 反相 | `Invert()` |
| | `sepia(1)` | 棕褐 | Tint 预设组合 |
| | 多函数 `filter:grayscale(1) brightness(1.2)` | 矩阵相乘叠加 | `ConcatValues` 链 |
| | `none` / 未设 | 无 filter | identity |

> **约定**：v1.3 每个函数 → fgui 预设矩阵；多函数 → 矩阵相乘（fgui `_ConcatValues`/`ConcatValues` 数学照搬）。任意 4×5 矩阵自定义语法留后补。

### 九宫格（标准 CSS `border-image-slice`）

| 属性 | 值 | 语义 |
|---|---|---|
| `border-image-slice` | `<top> <right> <bottom> <left>` | 4 值切片线距各边距离；px 或 %（% 乘源图对应边） |
| | `<a>` / `<a> <b>` / `<a> <b> <c>` | CSS 4 值缩写规则（同 margin） |
| | `fill` 关键字 | v1.3 暂不支持（中心填充与否）——默认 fill |

> **约定**：slice 配 `background-image`（Container）或 `<img src>`（Image）单源。不支持 `border-image-source` 双源。源图来自打包器图集（同 v1.1 bg-image atlas 链）。

## 3. 数据模型 + 管线

### 3.1 ResolvedStyle 字段

`loomgui_core/src/style/resolved.rs`：
```rust
pub struct ResolvedStyle {
    // ... 现有字段 ...
    pub color_filter: Option<[f32; 20]>,   // None=无 filter；Some=4×5 矩阵（行主序）
    pub border_image_slice: Option<SliceInsets>,  // None=无切片
}
pub struct SliceInsets { pub top: f32, pub right: f32, pub bottom: f32, pub left: f32 }  // 源图像素
```

### 3.2 program 号语义（更新）

| program | 含义 | shader keyword | frag |
|---|---|---|---|
| 0 | img / 无图 Container | — | `tex * vcol` |
| 1 | Text | ALPHA_MASK | `half4(vcol.rgb, vcol.a*tex.a)` |
| 2 | Container+bg-image | BG_COMPOSITE | `tex.rgb*tex.a + vcol.rgb*(1-tex.a)` |
| **3** | **ColorFilter（任意 program 0/2 叠 filter）** | **COLOR_FILTER** | 上述后 + `mul(M,col)+off` |

> **关键**：program=3 是"叠加 filter"标志，**不是独立几何路径**。ColorFilter 节点的 mesh 仍由 quad/rounded_rect/nine_slice 产出，只是 program 标 3 + 矩阵走 MPB。即：无图+filter → program=3（mesh=quad）；bg-image+filter → program=3（mesh=quad，走 BG_COMPOSITE+COLOR_FILTER 双 keyword）；slice+filter → program=3（mesh=nine_slice）。

### 3.3 ColorFilter 矩阵传递（MPB，照搬 fgui UpdateMatrix）

- 矩阵 = `Matrix4x4`（前 4 列 16 float）+ `Vector4 offset`（第 5 列）= 20 float = 5 Vector4。
- shader Properties 加 `_CF0..3`（矩阵 4 行）+ `_CFOff`（offset），进 `UnityPerMaterial` CBUFFER + Properties 对应（MPB SetVector 覆盖，同 `_ObjM` 模式）。
- MirrorPool：program=3 节点 SetVector ×5（`_CF0..3`+`_CFOff`）。
- Material key 不加 color_filter 维度——同 `(texture,3,mask_ctx)` 共享 Material，矩阵 MPB per-renderer 覆盖。

### 3.4 grayed 替代

- v1 `grayed: bool` → v1.3 `filter:grayscale(1)` → 灰化矩阵 → program=3。
- `color_tint`（style.color）**保留**走顶点色——per-vertex tint，与 ColorFilter 矩阵正交。
- DrawState key 不变（filter 走 program=3 + MPB，不打断 program=0/2 批合）。

### 3.5 九宫格 mesh 分流（Container/Image 分支）

```
match (has_slice, has_radius) {
    (false, false) => quad,
    (false, true)  => rounded_rect,           // 现状
    (true,  false) => nine_slice,             // 新：方角 16 顶点
    (true,  true)  => nine_slice_rounded,     // 新：四角圆角扇形 + 中间切片
}
```

### 3.6 nine_slice 几何（照搬 fgui SliceFill）

- 4×4 顶点：gridX[0..3] = rect.x、x+left、xMax-right、xMax；gridY 同理（v 翻转同 quad）。
- **clamp**（照搬 fgui）：容器比源图小 → `contentRect.width < sourceW - gridRect.width` 时四角重叠，用 `max(0, ...)` clamp 防越界。
- 9 quad：`TRIANGLES_9_GRID` 固定索引（照搬 fgui `Image.cs:267`）。
- UV：gridTexX/Y 按切片像素比例从源图 UV 切。

### 3.7 nine_slice_rounded 几何（LoomGUI 自设计，fgui 无现成）

- 四角区域：用 `rounded_rect` 的角扇形顶点（三角扇，照搬 v1.2 `mesh::rounded_rect`），半径取 border-radius resolve 值。
- 边/中心：用 nine_slice 的拉伸顶点，但四角的 2×2 顶点替换为圆角扇形的起止顶点。
- **不变量**：四角圆角区不拉伸（扇形顶点 UV 取源图对应角区），四边单轴拉伸，中心双轴拉伸。
- 实现期定具体顶点布局（设计层只定不变量 + 四角走圆角扇形、中间走切片）。

## 4. shader 改动（LoomGUI-Unlit.shader）

```hlsl
#pragma multi_compile _ COLOR_FILTER   // 新增（program=3）

Properties 加：
_CF0 ("CF0", Vector) = (1,0,0,0)
_CF1 ("CF1", Vector) = (0,1,0,0)
_CF2 ("CF2", Vector) = (0,0,1,0)
_CF3 ("CF3", Vector) = (0,0,0,1)
_CFOff ("CFOff", Vector) = (0,0,0,0)

CBUFFER(UnityPerMaterial) 加 _CF0..3 + _CFOff。

frag 末尾（现有 ALPHA_MASK/BG_COMPOSITE/默认分支之后，CLIPPED 之前）：
#if defined(COLOR_FILTER)
    float4x4 cfM = float4x4(_CF0, _CF1, _CF2, _CF3);
    col = half4(mul(cfM, float4(col.rgb, col.a)).rgb + _CFOff.rgb, col.a);
    // alpha 行 fgui 固定 (0,0,0,1,0)——col.a 不变，仅 rgb 受矩阵。
#endif
```

> **矩阵数学**：照搬 fgui `ColorFilter.cs`——`Matrix4x4`（4 行，每行 4 float = 前 16）+ offset（第 5 列 4 float）。fgui 的 alpha 行恒 `(0,0,0,1,0)` → alpha 不变。LoomGUI 同。

## 5. blob / pkg 影响

- **ColorFilter**：program 列已在 v1 坑79 加（第 19 列 u8）。矩阵不进 blob SOA——走 MPB，**blob 零改**。但 ResolvedStyle 加 `color_filter` 字段 → PKG_FORMAT_VERSION bump（v9→v10，scene/style 序列化加字段）。dirty hash 含 color_filter。
- **九宫格**：ResolvedStyle 加 `border_image_slice` 字段 → 同一 v9→v10 bump（合并一次 bump）。mesh 顶点数变（16 vs 4）→ dirty hash 含 slice + 顶点（v1.2 已验 dirty hash 捕获全顶点）。
- **blob VERSION 不变**（仍 v5）——program=3 复用现有 program 列，矩阵走 MPB 不入 blob。
- **PKG_FORMAT_VERSION 9→10**：color_filter + border_image_slice 两字段进 ResolvedStyle 序列化。

## 7. 验收（家里机 PlayMode + FrameDebugger）

### 功能验收
- **ColorFilter**：showcase 卡演示 grayscale/brightness/contrast/saturate/hue-rotate/invert 各效果 + 多函数叠加。disabled 按钮 `filter:grayscale(1)` 灰化正确（替代 v1 顶点色灰化）。
- **九宫格**：showcase 卡演示 bg-image + border-image-slice 皮肤缩放（小/大两种尺寸，四角不变形）。
- **slice+radius 共存**：圆角皮肤按钮——四角圆角 + 切片拉伸共存正确。
- **img 节点九宫格**：`<img>` + slice 回归。

### profiling 验收门槛
- draw call 回归：v1.3 前后同场景 draw call 数对比。ColorFilter（program=3）独立 draw call——disabled 按钮数量翻倍时涨幅可控。九宫格不增 draw call（纯几何）。
- GC 零回归：MirrorPool buffer 复用现状零 per-frame alloc；ColorFilter MPB SetVector ×5 不引入新 alloc。
- 500 节点静态压测（v1e-perf 已有）：加 ColorFilter + 九宫格后仍无卡顿。

### 零回归
- 无 filter 节点 program 走原 0/2（img/text/Container+bg-image）零改。
- 无 slice 节点 mesh 走原 quad/rounded_rect 零改。
- color_tint（style.color）顶点色路径保留。

## 9. 风险

- **nine_slice_rounded 自设计**：fgui 无现成，顶点布局实现期定。风险：四角扇形与边切片顶点拼接错位。缓解：TDD 先验不变量（四角不拉伸 + 边单轴 + 中心双轴），再验视觉。
- **ColorFilter draw call 涨**：disabled 按钮多时。缓解：profiling 验收门槛暴露，必要时升 filter_id 索引（方案 2）。
- **PKG_FORMAT_VERSION 10 bump**：color_filter + border_image_slice 两字段，老 pkg.bin 不兼容。缓解：重打 showcase pkg.bin（本机编码机职责）。
- **filter 多函数矩阵相乘顺序**：CSS 规范 filter 函数顺序 = 从左到右应用。fgui ConcatValues 是矩阵相乘。需确认顺序一致（实现期测试覆盖）。
