# v1c.3 多触摸设计（spec）

> 范围：C = 多触摸槽 + CaptureTouch/touch monitor + Move 对齐 fgui。click 增强（双击/downTargets 兜底/缩放容忍）defer v1c.4。
> 参考实现：`temp/FairyGUI-unity/`（fgui）。所有机制对照 fgui Stage.cs/EventDispatcher.cs 实证。
> 前序：v1c.2 事件冒泡（路由在 C# 业务侧，方向 A）。本 spec 修订 v1c.2 的 Move 路由（无 monitor 不产）。

## 0. 决策（§0）

| 项 | 决策 | 依据 |
|---|---|---|
| 范围 | C：多触摸槽 + CaptureTouch + Move 对齐 fgui | v1c.2 §9.2 defer；click 正交 defer v1c.4 |
| 多触摸槽位置 | 核心（`PointerState` 多槽） | capture 是输入状态，属核心；方向 A「核心产事件、C# 路由」 |
| 触摸槽数 | 固定 5（slots[0] 鼠标 + slots[1..4] 触摸 4 指） | 照 fgui 5 槽；鼠标 touch_id=-1 绕开 fgui fingerId 冲突 |
| 鼠标+触摸 | **同帧共存**（偏离 fgui 互斥） | fgui 互斥因 touchId=0 给鼠标撞 fingerId；LoomGUI 鼠标 -1 绕开，共存更通用 |
| touch_id 哨兵 | 鼠标 = -1（slots[0] 具体值）；**不实现 fgui -1 广播** | fgui 自身 3 处 AddTouchMonitor 调用全传具体值，-1 仅 API 预留 |
| CaptureTouch 加 monitor | capture 阶段 + bubble 阶段**各加一个**（照 fgui，标志消费即清） | fgui `_touchCapture` 消费即清零，cap/bub 两阶段独立 |
| Move 语义 | 有 monitor 产 Move@monitor 直派；无 monitor 不产 | fgui Move 无 monitor 只发 Stage 自身（业务不注册=无回调） |
| hovered（伪类源） | 全局 union（任一活跃槽命中链） | CSS :hover「任一指在元素或后代上 → 元素 hover」 |
| RollOver/Out 事件 | per-touch（每槽独立 hover_diff） | fgui lastRollOver 是 TouchInfo 实例字段，per-touch 正确 |
| active | 全局 union recompute（基于 down_node 命中链） | 任一指按下即 :active；多指不互相清 |
| 双语义 | RollOver/Out（per-touch「该指进出」）≠ hovered（全局「任一指在其上」） | 描述不同事实，可能不一致，业务各取所需 |
| click 判定 | 沿用 v1c.2（down_node==hit && <10px），不改 | 增强 defer v1c.4 |

## 1. 契约改动（§1）

### 1.1 PointerEvent / EventRecord 加 touch_id

```rust
// loomgui_core/src/input.rs
#[repr(C)]
#[derive(Debug, Clone, Copy)]
pub struct PointerEvent {
    pub kind: PointerKind,   // @0 (1B)
    pub button: u8,          // @1
    pub pad: [u8; 2],        // @2-3
    pub touch_id: i32,       // @4  (-1 = 鼠标主指 slots[0]；>=0 = 触摸 fingerId)
    pub x: f32,              // @8
    pub y: f32,              // @12
}   // 16 字节

#[repr(C)]
#[derive(Debug, Clone, Copy)]
pub struct EventRecord {
    pub node_id: u32,        // @0
    pub event_type: u8,      // @4
    pub pad: [u8; 3],        // @5-7
    pub touch_id: i32,       // @8  ← 新增（破 v1c.2 零改契约）
    pub x: f32,              // @12
    pub y: f32,              // @16
}   // 20 字节（v1c.2 是 16）
```

**波及面（破 v1c.2 零改契约，全列）**：
1. C# `LoomEvent` struct 手动同步：新增 `touch_id @8`，`y` 从 @12→@16（`StructLayout.Sequential` 自动排，但字段序必须加 touch_id）。
2. `borrow_events` 消费侧 stride：`Marshal.SizeOf<LoomEvent>()` 自动跟（当前用此），但若任何处按硬编码 16 步进须改 20。spec 标注。
3. `loomgui_ffi_c` abi_tests 手解 `rec_size`：16→20，且新增 sizeof 断言（16/20）。
4. `.pkg.bin` v3 **不动**（EventRecord 不进包，只跨 FFI）。

**鼠标兼容**：鼠标输入 `touch_id = -1`，核心当 slots[0]（主指），行为与 v1c.2 等价。v1c.1/v1c.2 鼠标 PlayMode 验收不破（除 Move 语义，见 §2.3）。

### 1.2 PointerState → 多槽

```rust
#[derive(Debug, Clone)]
pub struct TouchSlot {
    pub touch_id: i32,                  // -1 = 鼠标主指 / 触摸 fingerId；空闲槽 = -1（slots[1..4] 初始）
    pub last_pos: (f32, f32),
    pub is_down: bool,
    pub down_node: Option<NodeId>,
    pub down_pos: (f32, f32),
    pub last_hit: Option<NodeId>,       // 本帧命中（hover_diff + is_pointer_on_ui 用）
    pub last_hovered_chain: Vec<NodeId>,
    pub touch_monitors: Vec<NodeId>,    // v1c.3：capture 的节点（move/up 直派）
}

pub struct PointerState {
    pub slots: Vec<TouchSlot>,          // 固定 5 槽
    // slots[0] = 鼠标主指（touch_id=-1 常驻）
    // slots[1..4] = 触摸（touch_id=-1 空闲，Down 时分配 fingerId）
}
```

**槽分配**（照 fgui GetHitTarget 分配逻辑）：
- `process` 收到 PointerEvent，按 `touch_id` 线性遍历找已占槽（slots[i].touch_id == ev.touch_id）。
- 鼠标（touch_id=-1）恒用 slots[0]。
- 触摸找不到已占槽且 `kind==Down` → 取 slots[1..4] 首个空闲（touch_id==-1）分配。
- 触摸找不到且非 Down → 丢弃（无主触摸的 move/up，照 fgui）。
- 触摸 Down 时 slots[1..4] 全占（4 指已下）→ 丢弃该 Down（照 fgui 满 5 槽丢弃）。
- Up 处理后，触摸槽 `touch_id` 重置为 -1（释放，照 fgui line 1309）。

## 2. CaptureTouch + Move 对齐 fgui（§2）

### 2.1 CaptureTouch 数据流

capture 是**输入状态**（核心持 touch_monitors），不是路由策略。

1. **Down 产事件**：核心对 down 产 `EventRecord{node_id=hit, event_type=Down, touch_id}`（走正常 hit target → C# BubbleRoute）。
2. **C# 回调里 CaptureTouch**：listener 在 Down 回调（capture 或 bubble 阶段）调 `ctx.CaptureTouch()` → EventContext 设 `_touchCapture=true`。
3. **C# 消费标志，记 monitor 节点**（照 fgui 消费即清）：BubbleRoute 的 capture 阶段循环检测 `_touchCapture`→清零→记 `_capNode=currentTarget`；bubble 阶段循环检测→清零→记 `_bubNode=currentTarget`（cap/bub 各最多一个，照 fgui EventDispatcher.cs:305-324）。
4. **C# 通知核心**：DispatchPending Down 分支 BubbleRoute 跑完后，对 `_capNode`（若 Some）和 `_bubNode`（若 Some）各调 `loomgui_stage_add_touch_monitor(handle, touch_id, node_id)` → 核心加进该槽 `touch_monitors`（去重）。
5. **核心 Move/Up 派发看 monitor**（见 2.3）。
6. **Up 后清 monitor**：核心 Up 处理完，清该槽 `touch_monitors`（照 fgui `End` 后 `touchMonitors.Clear()`）。
7. **C# 主动释放**：listener 可调 RemoveTouchMonitor → `loomgui_stage_remove_touch_monitor(handle, node_id)` → 核心从所有槽移除该 node（置 null 占位非 RemoveAt，照 fgui，避免遍历索引偏移）。

### 2.2 EventContext.CaptureTouch（C#）

- `CaptureTouch()` 无参（照 fgui EventContext.cs:61），隐式 capture 当前 EventContext 的 currentTarget + touchId。
- EventContext 加字段 `public int touchId`（DispatchPending 每事件首行设 `ctx.touchId = rec.touch_id`，池对象复用必须重置）。
- `EventContext._touchCapture` 消费即清零（照 fgui）。

### 2.3 Move 语义改造（核心改，对齐 fgui）

v1c.2 Move 走 BubbleRoute（命中链冒泡）。**改为 fgui 语义**（核心产事件层决定）：

```
process 收到 Move 事件:
  更新该槽 last_pos
  跑该槽 hover_diff（更新 last_hit + 产 RollOver/Out，仍每事件后跑）
  let slot = 该 touch_id 的槽
  if slot.touch_monitors 非空:
      for monitor in &slot.touch_monitors:
          产 EventRecord{node_id=monitor, event_type=Move, touch_id, x, y}  // 每 monitor 一条，直派
  else:
      不产 Move 事件   // 对齐 fgui「无 monitor 只发 Stage」；LoomGUI 无 Stage listener = 不产
```

**C# DispatchPending 分流调整**：
```csharp
switch (evt.type) {
    case Down: case Click:  BubbleRoute(evt); break;          // 不变
    case Up:
        BubbleRoute(evt);                                    // hit target 的 Up
        // 核心已为 monitor 额外产 Up（去重，见下）
        break;
    case Move:                                               // 核心只在有 monitor 时产
        DirectDispatch(evt);   // ← 改：Move 改直派（核心算好的 monitor 目标）
        break;
    case RollOver: case RollOut: DirectDispatch(evt); break;  // 不变
}
```

**Up 去重**（修审核 Critical-5）：核心产 Up 时，hit target 的 Up（node_id=hit）+ 各 monitor 的 Up（node_id=monitor）。**若某 monitor == hit target，不为该 monitor 重复产**（避免同节点收两次 Up）。实现：产 monitor Up 前 `if monitor != hit { push }`。

**Down/Click 不变**：仍走 hit target → C# BubbleRoute。

### 2.4 鼠标/触摸统一（实证支持）

fgui 鼠标 Move 无 monitor 也不产业务事件（DispatchEvent 只发 Stage 自身）。LoomGUI 对齐：**鼠标/触摸统一「Move 产事件当且仅当有 monitor」**，不学 fgui 的 began 不对称（鼠标悬停移动也 Move 到 Stage），因 LoomGUI 无 Stage listener，该差异不可观测。

**v1c.2 行为变化**：v1c.2 鼠标 Move 在命中元素上产 Move 事件（沿链 bubble）。v1c.3 改后鼠标 Move **不产**（除非 capture）。核实：v1c.2 interact sample 无 Move listener（仅 Click/RollOver/RollOut），**无现存业务破坏**。此为有意识的对齐 fgui，spec 明示。

### 3.1 FFI 接口

| FFI | 改动 | 说明 |

### 3.2 LoomInputCollector 多指采集

每帧 Collect：先采鼠标（touch_id=-1，slots[0]），再采所有活跃触摸（touch_id=fingerId，slots[1..4]）。

```csharp
public void Collect(IntPtr stage, Vector2 rootSize) {
    if (stage == IntPtr.Zero) return;
    var events = new List<Bindings.PointerEvent>();
    var screenSize = new Vector2Int(Screen.width, Screen.height);
#if ENABLE_INPUT_SYSTEM
    if (Mouse.current != null) { /* 鼠标 touch_id=-1，Down/Up/Move 边沿 */ }
    if (Touchscreen.current != null) {
        foreach (var touch in Touchscreen.current.touches) {
            if (touch == null) continue;
            var phase = touch.phase.ReadValue();
            if (phase == Stationary) continue;        // 跳过静止
            byte kind = 2;
            if (phase == Began) kind = 0;
            else if (phase == Ended || phase == Canceled) kind = 1;  // Ended+Canceled 都 Up
            events.Add(... touch_id = touch.touchId.ReadValue() ...);
        }
    }
#else
    foreach (var t in Input.touches) { ... touch_id = t.fingerId ... }
#endif
    Native.loomgui_stage_set_input((StageHandle*)stage, events.ToArray(), events.Count);
}
```

- 鼠标 + 触摸**同帧共存**（带触摸屏桌面）：鼠标 slot0，触摸 slot1-4。
- EditMode 无 Touchscreen → 跳过触摸（同 Mouse.current null）。
- Ended + Canceled 都映射 Up（Canceled 不特判 click，defer）。
- 此 C# 本机不编译（无 Unity），家里机验。touch_id 字段名/绑定以家里机 csbindgen 重生成 + 编译为准。

## 4. 核心状态机 + is_pointer_on_ui 多指化（§4）

### 4.1 process 多槽主循环

```rust
pub fn process(&mut self, scene: &mut Scene, events: &[PointerEvent]) -> Vec<EventRecord> {
    let mut out = Vec::new();
    if events.is_empty() {
        for slot in active_slots_mut() { slot.hover_diff(scene, &mut out); }
        recompute_hovered(scene);   // 全局 union
        return out;
    }
    for ev in events {
        let slot_idx = find_or_alloc_slot(ev);   // 按 touch_id；非 Down 无槽→跳过；满→丢
        let Some(slot) = slots.get_mut(slot_idx) else { continue };
        slot.last_pos = (ev.x, ev.y);
        let hit = hit_test(scene, slot.last_pos);
        slot.last_hit = hit;
        match ev.kind {
            Move => {
                slot.hover_diff(scene, &mut out);
                for m in &slot.touch_monitors {
                    out.push(EventRecord { node_id: m.0 as u32, event_type: EVT_MOVE, touch_id: ev.touch_id, x, y });
                }
            }
            Down => {
                slot.is_down = true; slot.down_pos = (x,y); slot.down_node = hit;
                if let Some(n) = hit { if !scene.nodes[n.0].disabled {
                    out.push(Down@hit);   // active 由末尾 recompute_active 统一（不调 set_active_chain）
                }}
                slot.hover_diff(scene, &mut out);
            }
            Up => {
                slot.is_down = false;
                if let Some(n) = hit { if !scene.nodes[n.0].disabled {
                    out.push(Up@hit);
                    if click 判定(down_node==hit && <10px) { out.push(Click@hit); }
                }}
                for m in &slot.touch_monitors {
                    if Some(m) != hit { out.push(Up@m, touch_id); }   // 去重（#5）
                }
                slot.touch_monitors.clear();
                slot.down_node = None;
                slot.hover_diff(scene, &mut out);
                if 触摸槽 { slot.touch_id = -1; }   // 释放（照 fgui line 1309）
            }
        }
    }
    recompute_hovered(scene);   // 全局 union（任一活跃槽命中链）
    recompute_active(scene);    // 全局 union（所有 is_down 槽 down_node 命中链）
    out
}
```

### 4.2 hover_diff 拆分（修审核 #11）

- per-slot `hover_diff`：只产 RollOver/Out（diff 该槽 last_hovered_chain），**不调 set_hovered_chain**。
- 全局 hovered 在 process 末尾 `recompute_hovered`：清所有 → 所有活跃槽（last_hit 非 None）命中链 union 置 true。
- **Down/Up 分支不调 set_active_chain**（修审核 Critical-11），active 纯由 `recompute_active` 驱动。

### 4.3 active 全局合并

```rust
fn recompute_active(scene, slots) {
    清所有 active;
    for slot in slots { if slot.is_down {
        沿 slot.down_node 祖先链置 active;   // 基于 down_node（Down 时命中），非当前 hit
    }}
}
```

`:active` = 任一指按下时命中该元素或祖先。多指按多按钮都 active；松一指重算剩余。

### 4.4 is_pointer_on_ui 多指化

```rust
pub fn is_pointer_on_ui(&self) -> bool {
    self.pointer_state.slots.iter().any(|slot| {
        match slot.last_hit { None => false, Some(id) => id != root_id }
    })
}
```

每槽缓存 `last_hit`（process 时 hit_test 存）。tick_and_render 改为遍历活跃槽各算 last_hit（不单算 last_pos）。

### 4.5 Stationary 不刷新（照 fgui 局限，defer）

手指不动（无 Move 事件）时，hover 不随元素移动刷新（照 fgui GetHitTarget Stationary 分支保留旧 target）。元素动后 hover 跟随 defer v1c.4+。

### 5.3 零改契约（v1c.3 破/不破明细）

**破**（v1c.2 零改契约例外，spec 明示）：
- `EventRecord` 16→20 字节（+touch_id @8）— 多触摸刚需
- `PointerEvent` +touch_id（FFI 内部，C# struct 跟）
- C# `LoomEvent` struct 手动 +touch_id（同步 ABI）
- `borrow_events` 消费侧 stride 16→20（若硬编码）

**不破**：
- frame blob / MirrorPool / MaterialManager / TextRasterizer / shader / `.pkg.bin` v3：**零改**
- RollOver/Out 祖先链 diff 机制：**保留**（per-slot 化但逻辑同 v1c.2）
- EventBridge capture+bubble 两组 / 委托 remove / EventContext 池：**保留**
- Down/Up/Click 的 BubbleRoute 路由：**保留**（Click 仍 down_node==hit 简化）
- node_parent FFI：**保留**

### 5.4 范围 / defer

**v1c.3 做**：多触摸槽（固定 5：1 鼠标+4 触摸）+ CaptureTouch/touch monitor（cap+bub 各一）+ Move 对齐 fgui（无 monitor 不产）+ EventRecord/PointerEvent 加 touch_id + active/hovered/cur_hit 多指合并 + FFI add/remove_touch_monitor + LoomInputCollector 多指采集 + Up 去重。

**defer v1c.4**（click 增强，正交可独立做）：双击（350ms 窗口）/ downTargets 链兜底 / 缩放容忍 / Move 中超阈值取消 click / Canceled 跳过 click / Stationary hover 跟随。

**defer v1d+**：拖拽/ScrollPane/手势仲裁（§10.4，消费 capture 但本身新机制）/ onKeyDown 键盘 / transform 命中。

### 5.5 AI 可预测性

- 多指语义照 fgui TouchInfo（每指独立 + capture monitor），AI 对「手指移出仍收事件」有 fgui 先验。
- Move 无 monitor 不产 — 对齐 fgui，AI 可预测（「想收 Move 必须 capture」）。
- touch_id 贯穿 EventRecord/PointerEvent/EventContext — 事件来源确定可追踪。
- 鼠标 touch_id=-1 单指等价 v1c.2（除 Move 语义对齐）— 鼠标行为基本不破。

### 5.6 已知偏离/简化（围栏文档/sample 须告知）

- **鼠标+触摸同帧共存**（偏离 fgui 互斥）：fgui 因 touchId=0 给鼠标撞 fingerId 才互斥，LoomGUI 鼠标 -1 绕开，共存更通用。
- **RollOver/Out per-touch 事件 + hovered 全局 union 伪类双语义**：可能不一致（A 指移出 X 但 B 指还在 X → X 收 RollOut 但 :hover 仍 true），各描述不同事实。
- **CaptureTouch 加 cap+bub 各一**（照 fgui，非整条链）：典型拖拽只 1 个。
- **add_touch_monitor 不实现 -1 广播**：fgui 自身不用，YAGNI。
- **Move 无 monitor 不产**（含鼠标）：v1c.2 鼠标 Move 产事件行为变化，无现存业务破坏（sample 无 Move listener）。
- **click 判定沿用 v1c.2 简化**：capture 下 hit 变导致 click 丢失是已知简化，v1c.4 downTargets 兜底修。
- **Stationary 不刷新 hover**：照 fgui 局限，元素动后 hover 不跟随。
