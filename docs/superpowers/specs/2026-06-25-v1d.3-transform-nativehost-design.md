# v1d.3 transform 命中渲染 + NativeHost-lite 设计契约

> 范围来源：`docs/roadmap/v1d-plan.md` §3 v1d.3（defer #7 transform）+ 本轮新增 NativeHost-lite（从 v1.x 提前）+ 调试通路（id 查询 + 整树 dump）。
> 模式：transform 检测/矩阵全在 core（loomgui_core）；后端 Unity **扁平**——identity/merge 节点走现有 TRS 路径，**非 identity 节点走 matrix shader**（顶点 × world matrix，支持剪切）。NativeHost + 调试 dump 是后端 C# + 少量查询 FFI。
> ★ 已对照 fgui 源码（`temp/FairyGUI-unity/`）：见 §1 对照表。

## 0. 决策（已拍）

| # | 决策 | 选定 |
|---|---|---|
| 1 | transform 渲染/命中模型 | **A**：core 算累计世界矩阵（DFS，含父链）。**identity/merge 节点走现有 TRS 路径不动**；**非 identity 节点走 matrix shader**（顶点 × world matrix，GO transform=identity），支持任意仿射含剪切 |
| 2 | transform-origin / pivot | **A**：固定 box **center**（w/2, h/2），不支持自定义 transform-origin（YAGNI） |
| 3 | NativeHost 范围 | **GoWrapper-lite**：跟随 world transform + 显隐 + 排序 + break batch。不做 layout 测量 / intrinsic size push / hit / clip / 所有权 / Godot |
| 4 | NativeHost 表达 | **A**：普通 `<div>` 占位（core 不认新 kind）+ 后端 `BindNativeHost(id 字符串或 nodeId, go)`/`Unbind` |
| 5 | clip + transform | **保守**：clip rect 仍是 layout 轴对齐 design rect，**不随 transform 旋转**（留 v1.x） |
| 6 | 动态 UI / panel | **下一个独立子项目**，v1d.3 不做。world matrix 每帧重算 → 天然支持将来动态增删（接口预留） |
| 7 | 调试通路 | **FFI `node_id_by_id`**（id→nodeId 查询，NativeHost + 调试用）+ **FFI `dump_scene`**（整树 JSON dump，缓解「看不到 UI 树」痛点，为可视化铺路） |

### 0.1 不范围（明确 defer）

- **transform-origin 自定义**：固定 center（决策 2）。
- **旋转 clip box / shape mask**：clip 不随 transform 旋转（决策 5）。
- **CSS `skew()` / `matrix()` 函数解析**：v1d.3 只解析 `translate/rotate/scale`。剪切由「非均匀缩放 ∘ 旋转」复合产生（world matrix 表达 + matrix shader 渲染，已支持）；`skew()` 函数本身留 v1.x。
- **3D / VertexMatrix / 透视**：v1.x。
- **translate `%` 单位**：v1d.3 只 px。
- **动态 UI / panel / 实例化 / 运行时增删节点 API**：独立子项目（决策 6）。
- **NativeHost 完整版**：size push（§15 drain 步）/ clip / hit / 所有权 / Godot → v1.x。
- **UI 树可视化（editor inspector / overlay）**：v1d.3 只 dump 文本/JSON（决策 7），可视化 UI 后续。

| 机制 | fgui | LoomGUI v1d.3 | 异同 |

## 2. 数据结构变更

### 2.1 LocalTransform（新，style/resolved.rs）

```rust
/// CSS transform 解析产物（渲染/命中层，不进 taffy）。默认 = identity。
#[derive(Debug, Clone, Copy, PartialEq, Serialize, Deserialize)]
pub struct LocalTransform {
    pub translate_x: f32,  // px（CSS translate；v1d.3 拒 %）
    pub translate_y: f32,  // px
    pub rotation: f32,     // 弧度（CSS rotate(deg) → rad）
    pub scale_x: f32,      // 无量纲（默认 1）
    pub scale_y: f32,      // 无量纲（默认 1）
}
impl Default for LocalTransform {
    fn default() -> Self { Self { translate_x:0.0, translate_y:0.0, rotation:0.0, scale_x:1.0, scale_y:1.0 } }
}
impl LocalTransform {
    /// identity → 走现有 TRS 路径 + 参与 merge。
    pub fn is_identity(&self) -> bool {
        self.translate_x == 0.0 && self.translate_y == 0.0
            && self.rotation == 0.0 && self.scale_x == 1.0 && self.scale_y == 1.0
    }
}
```

## 3. 机制

### 3.1 CSS transform 解析（parse/css.rs `apply_decl`）

`"transform"` 声明值 = 函数序列（空格分隔）。v1d.3 解析：
- `translate(<len>,<len>)` → translate_x/y（px；拒 `%`）
- `rotate(<angle>)` → rotation（deg→rad）
- `scale(<num>[,<num>])` → scale_x/y（单值 = 双轴同）
- 多函数复合：`translate(10px,20px) rotate(45deg) scale(1.5)` → §3.2 矩阵里按 T ∘ R ∘ S 复合
- 拒 `skew()`/`matrix()`/3D/`calc()`（围栏外，静默忽略 + 验证器报错）

### 3.2 world matrix 累计（新函数，tick 内调）

每帧 tick 在 solve 后、hit/render 前，DFS 整树算 `world_transforms`：

```
对每节点 n（DFS 先序，parent 已算）：
  pivot = (n.layout_rect.w / 2, n.layout_rect.h / 2)
  rel = (n.layout_rect.x - parent.layout_rect.x,   // 相对父位置（layout_rect 存绝对，反推）
         n.layout_rect.y - parent.layout_rect.y)
  t = n.style.transform
  // local = T(rel + t.translate) ∘ T(pivot) ∘ R(t.rotation) ∘ S(t.scale) ∘ T(-pivot)
  local = translate(rel.x + t.translate_x, rel.y + t.translate_y)
        ∘ translate(pivot) ∘ rotate(t.rotation) ∘ scale(t.scale_x, t.scale_y) ∘ translate(-pivot)
  n.world = parent.world ∘ local
根节点：parent.world = identity
```

> pivot=center 固定（决策 2），matrix 构造含 pivot → shader × matrix 自动处理旋转中心，后端零特殊处理。translate 叠 layout rel（CSS transform translate 是渲染层额外平移，不置 layout_dirty）。

### 3.3 world matrix 序列化（写 blob）

world matrix `Affine2 [a,b,c,d,tx,ty]` **直接**进 blob 6 列（§2.4），不分解 TRS。matrix 天然表达任意仿射含剪切。

后端从 matrix 取所需：
- TRS 路径（identity/merge）：用 tx,ty（a=1,b=0,c=0,d=1 时 rotation=0/scale=1）
- matrix shader 路径（非 identity）：整 matrix 作 `_ObjectMatrix`

### 3.4 命中 world_to_local（hit.rs 改写）

替代裸 `layout_rect` AABB（hit.rs:50）：

```
hit_test(scene, world_point):
  DFS 逆等效绘制序，对节点 n：
    if n.clip_rect 且 world_point 不在 clip_rect（世界 AABB，不旋转） → 跳整子树
    local_point = inverse(n.world_transforms[i]) · world_point   // 仿射逆，含剪切可逆
    if n.touchable 且 local_point 在 (0,0, layout_rect.w, layout_rect.h) → 命中
```

> 仿射矩阵（含剪切）总有逆（det≠0 时）。剪切节点的本地 box 仍轴对齐 (0,0,w,h)（box 在本地系轴对齐，剪切是 world matrix 应用后的视觉剪切），命中点逆变换回本地判轴对齐 box → 剪切节点命中正确。

### 3.5 batch break（render/batch.rs）

非 identity transform 节点（`!n.style.transform.is_identity()`）**不参与跨节点 merge**（merge 断点）。identity 节点 merge 行为不变（merged 顶点 world + matrix=identity）。照 fgui breakBatch。

### 3.5b mesh 顶点 + 渲染路径（render/mod.rs + blob.rs + shader，关键正确性）

两条渲染路径，按 world matrix 是否纯平移（`a==1 && b==0 && c==0 && d==1`）分：

- **TRS 路径**（identity 节点 + merge 节点）：render 产**绝对 layout 顶点**（现状）；blob re-base 减 `tx,ty`（=layout top-left）→ top-local；GO `localPosition`=tx,ty、rotation=0/scale=1；现有 material（无 OBJECT_MATRIX）。**merge 节点**：merged 顶点 world + matrix=identity（tx=ty=0），现状不动。
- **matrix 路径**（非 identity 节点）：render 产顶点时**减 `layout_rect.xy` → box 本地 (0..w)**；blob **不 re-base**；GO `transform=identity`；material 带 `_ObjectMatrix`（world matrix → float4x4）；shader vertex × `_ObjectMatrix` → world。break merge。

> 渲染层 `build_render_nodes` 读 `style.transform.is_identity()` 决定顶点基 + 路径标记；blob.rs 按标记决定是否 re-base。TRS 路径完全保留现状（所有现有测不变）。

### 3.6 shader + 后端（Unity）

**shader**（现有 unlit 加 OBJECT_MATRIX keyword）：
```glsl
#pragma multi_compile _ OBJECT_MATRIX
#pragma multi_compile _ CLIPPED
// vertex:
#if OBJECT_MATRIX
    pos = mul(_ObjectMatrix, float4(pos.xy, 0, 1));  // world matrix（含 pivot/剪切）
#endif
// 之后正常 MVP + CLIPPED discard
```
OBJECT_MATRIX × CLIPPED = 4 variant。material key 加 `matrix_flag`。

**MirrorPool.Sync**：
- 纯平移 matrix（identity/merge 节点）→ TRS 路径（localPosition=tx,ty，现有 material）
- 非纯平移 → matrix 路径（GO transform=identity + `_ObjectMatrix` uniform + OBJECT_MATRIX material）
- 根 GO `(1,-1,1)` y-flip + `Cull Off` 不变；matrix 路径下 y-flip 由根统一（_ObjectMatrix 是 design→world 前的本地变换，根翻转在外层）

### 3.7 NativeHost-lite（后端 C# + id 查询 FFI）

- HTML：`<div id="model-slot"></div>` 占位（core 当 Container，照常算 world matrix）。
- 后端 API：
  ```csharp
  public void BindNativeHost(uint nodeId, GameObject go);
  public void BindNativeHost(string id, GameObject go);  // 内部 node_id_by_id 查 nodeId
  public void UnbindNativeHost(uint nodeId);
  ```
- `NativeHostManager`：每帧 MirrorPool.Sync 后遍历 `_bindings`，从 blob 读 nodeId 的 world matrix/visible/sort_key，设外部 GO（matrix 路径规则同 §3.6——外部 GO 用 world matrix 设 TRS 或直接 localToWorldMatrix；sort_key → Renderer.sortingOrder，多 renderer 保相对序；visible=false → SetActive(false)）。host 不进 UI 批合。node 消失 → 自动 Unbind。

### 3.8 调试 dump（core + FFI）

`dump_scene` 遍历 `Scene.nodes` 输出 JSON（Rust 拥有，下 tick 失效，C# 拷贝）：
```json
[{"node_id":0,"parent":-1,"tag":"div","id":"root","classes":["main"],"kind":"Container",
  "layout":{"x":0,"y":0,"w":1080,"h":1920},"transform":{"tx":0,"ty":0,...},"visible":true}, ...]
```
C# 可 `Debug.Log` 或 parse 做 editor inspector。为 UI 树可视化（后续）铺路。

## 4. tick 管线插入点（stage.rs `tick_and_render`）

现有（v1d.2）：solve → process(hit+状态) → rematch → render。
v1d.3：solve → **`compute_world_transforms`（§3.2）** → process（hit 用 world_to_local §3.4）→ rematch → render（transform 从 `world_transforms` §3.3）。

> world matrix 必须在 hit 前。事件回调改 transform 延下帧（§15 反馈环规则）。

## 5. FFI（loomgui_ffi_c/src/lib.rs）

- **transform**：零新渲染 FFI。world matrix 走 blob 6 列（§2.4）。
- **`loomgui_node_id_by_id(h, id_ptr, id_len) -> i32`**（新）：查 `Node.id_attr == id`，返 NodeId.0 或 -1。NativeHost + 调试用。
- **`loomgui_dump_scene(h, *mut usize) -> *const u8`**（新）：返整树 JSON（§3.8），Rust 拥有下 tick 失效。
- NativeHost 本体零新 FFI（NativeHostManager 读 blob）。
- `loomgui_version()` → `"v1d.3"`；blob.rs `VERSION` 3→4 + 所有手搓 C# fixture 同步（坑 17）+ 本机重编 commit .dll（坑 10）。

## 6. C# mirror（loomgui_unity）

- **FrameBlob**：transform 解析从 local_x/y(2) → world matrix 6 列（按 §2.4 列序）。`IsValid` 校验 version=4（拒 v3）。
- **MirrorPool.Sync**：纯平移 matrix → TRS 路径（现状）；非纯平移 → matrix 路径（GO transform=identity + `_ObjectMatrix` + OBJECT_MATRIX material，§3.6）。
- **NativeHostManager**（新）：`Dictionary<uint, GameObject> _bindings` + `Dictionary<string,uint> _idCache`；每帧 Sync 后同步外部 GO（§3.7）。
- **LoomStage**：`BindNativeHost(uint/string, GameObject)` / `Unbind(uint)` 转发；`DumpScene()` 调 `loomgui_dump_scene` 返 JSON（调试/日志）。
- **shader**：加 OBJECT_MATRIX keyword + material key matrix_flag。

## 7. 风险

| # | 风险 | 对策 |
|---|---|---|
| R1 | matrix shader 路径 + CLIPPED 组合 4 variant 编译/性能 | multi_compile 4 variant（OBJECT_MATRIX × CLIPPED）；非 identity 节点占少数，variant 冷启动一次性编译 |
| R2 | matrix 路径 GO transform=identity 与 sortingOrder/camera 交互 | sortingOrder 在 Renderer（GO 上），GO transform=identity 不影响；UI 相机 cullingMask 不变 |
| R3 | clip 不随 transform 旋转 → 旋转容器溢出内容不裁 | §0.1 defer；sample 避免「旋转 + overflow:hidden」；knowledge 告知 |
| R4 | blob version bump → 家里机忘换 .dll → 静默拒帧（坑 10） | v4 + IsValid 拒 v3；本机重编 commit .dll push |
| R5 | world matrix 每帧 DFS 全树算 → 性能 | O(n) 矩阵乘，500 节点无压力；invalidation set 留 v1e |
| R6 | 顶点 re-base 两路径（§3.5b）实现错 → 非 identity 位置飞 | blob round-trip + hit 单测覆盖非 identity；PlayMode demo 验证 |
| R7 | 剪切 matrix 的逆（命中）数值稳定性 | 仿射逆 det≠0 即可；极端 scale（→0）加 epsilon 保护 |
| R8 | dump_scene JSON 大 → FFI 拷贝开销 | 仅调试按需调（非每帧）；500 节点 JSON ~几十 KB 可接受 |
