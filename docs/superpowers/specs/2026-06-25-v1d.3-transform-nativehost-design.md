# v1d.3 transform 命中渲染 + NativeHost-lite 设计契约

> 范围来源：`docs/roadmap/v1d-plan.md` §3 v1d.3（defer #7 transform）+ 本轮新增 NativeHost-lite（从 v1.x 提前，§0.1 列剩余仍 defer）。
> 模式：transform 检测/矩阵全在 core（loomgui_core），后端 Unity **扁平**应用（不嵌套 GO）；NativeHost 是后端纯 C# 逻辑（零新 FFI）。
> ★ 已对照 fgui 源码（`temp/FairyGUI-unity/`）：见 §1 对照表。

## 0. 决策（已拍）

| # | 决策 | 选定 |
|---|---|---|
| 1 | transform 渲染/命中模型 | **A**：core 算累计世界矩阵（DFS，含父链），后端 Unity 继续**扁平**（不嵌套 GO）。命中用世界逆矩阵，渲染把世界 TRS 烤进 blob，flat GO 设 TRS |
| 2 | transform-origin / pivot | **A**：固定 box **center**（w/2, h/2），不支持自定义 transform-origin（YAGNI） |
| 3 | NativeHost 范围 | **GoWrapper-lite**：跟随 world transform + 显隐 + 排序 + break batch。不做 layout 测量 / intrinsic size push / hit / clip / 所有权 / Godot |
| 4 | NativeHost 表达 | **A**：普通 `<div>` 占位（core 不认新 kind）+ 后端 `BindNativeHost(nodeId, go)`/`Unbind`。core 零改动（world transform 本就要算） |
| 5 | clip + transform | **保守**：clip rect 仍是 layout 轴对齐 design rect，**不随 transform 旋转**。旋转容器溢出裁剪不精确，留 v1.x |
| 6 | 动态 UI / panel | **下一个独立子项目**，v1d.3 不做。world matrix 每帧重算 → 天然支持将来动态增删节点（接口预留） |

### 0.1 不范围（明确 defer）

- **transform-origin 自定义**：固定 center（决策 2）。
- **旋转 clip box / shape mask**：clip 不随 transform 旋转（决策 5），shape mask v1.x。
- **skew / matrix() / 3D / VertexMatrix**：v1.x。
- **translate `%` 单位**：v1d.3 只 px。
- **非均匀缩放叠加旋转的剪切**：blob 传 TRS（core 从 world matrix 分解），剪切复合 TRS 不可分解 → 不保证正确（§7 R1）。常见「旋转 / 缩放 / 旋转+均匀缩放 / 嵌套旋转」可分解，够用。
- **动态 UI / panel / 实例化 / 运行时增删节点 API**：独立子项目（决策 6）。v1d.3 只保证 world matrix 每帧重算。
- **NativeHost 完整版**：size push（§15 drain 步）/ clip / hit / 所有权 / Godot → v1.x。

## 2. 数据结构变更

### 2.1 LocalTransform（新，style/resolved.rs）

```rust
/// CSS transform 解析产物（渲染/命中层，不进 taffy）。默认 = identity。
#[derive(Debug, Clone, Copy, PartialEq, Serialize, Deserialize)]
pub struct LocalTransform {
    pub translate_x: f32,  // px（CSS translate；v1d.3 拒 %）
    pub translate_y: f32,  // px
    pub rotation: f32,     // 弧度（CSS rotate(deg) → rad）
    pub scale_x: f32,      // 无量纲（默认 1）
    pub scale_y: f32,      // 无量纲（默认 1）
}
impl Default for LocalTransform {
    fn default() -> Self { Self { translate_x:0.0, translate_y:0.0, rotation:0.0, scale_x:1.0, scale_y:1.0 } }
}
impl LocalTransform {
    /// identity → 后端零特殊处理，参与 merge（现状路径）。
    pub fn is_identity(&self) -> bool {
        self.translate_x == 0.0 && self.translate_y == 0.0
            && self.rotation == 0.0 && self.scale_x == 1.0 && self.scale_y == 1.0
    }
}
```

## 3. 机制

### 3.1 CSS transform 解析（parse/css.rs `apply_decl`）

`"transform"` 声明值 = 函数序列（空格分隔）。解析（拒绝围栏外函数 = 静默忽略 + 围栏验证器报错）：
- `translate(<len>,<len>)` → translate_x/y（px；v1d.3 拒 `%`）
- `rotate(<angle>)` → rotation（deg→rad）
- `scale(<num>[,<num>])` → scale_x/y（单值 = 双轴同）
- 多函数复合：`translate(10px,20px) rotate(45deg) scale(1.5)` → 顺序在 §3.2 矩阵里体现（T ∘ R ∘ S）

> v1d.3 单函数直接填字段；多函数按矩阵乘复合（LocalTransform 字段是「最终合并值」，非逐函数记录）。

### 3.2 world matrix 累计（新模块/函数，tick 内调）

每帧 tick 在 solve 后、hit/render 前，DFS 整树算 `world_transforms`：

```
对每节点 n（DFS 先序，parent 已算）：
  pivot = (n.layout_rect.w / 2, n.layout_rect.h / 2)
  rel = (n.layout_rect.x - parent.layout_rect.x,   // 相对父位置（layout_rect 存绝对，反推）
         n.layout_rect.y - parent.layout_rect.y)
  t = n.style.transform
  // local = T(rel + t.translate) ∘ T(pivot) ∘ R(t.rotation) ∘ S(t.scale) ∘ T(-pivot)
  local = translate(rel.x + t.translate_x, rel.y + t.translate_y)
        ∘ translate(pivot) ∘ rotate(t.rotation) ∘ scale(t.scale_x, t.scale_y) ∘ translate(-pivot)
  n.world = parent.world ∘ local
根节点：parent.world = identity（rel = root.layout_rect.xy）
```

> pivot=center 固定（决策 2）。translate 叠在 layout rel 上（CSS transform translate 是渲染层额外平移，不影响 layout——layout_dirty 不置，只算 world matrix）。

### 3.3 world TRS 分解（core，写 blob 前）

从 world matrix `Affine2 [a,b,c,d,tx,ty]` 分解后端 TRS：

```
position (box top-left 世界) = (tx, ty)        // = world · (0,0)
scale_x = sqrt(a² + b²)
scale_y = sqrt(c² + d²)
rotation = atan2(b, a)
```

写进 `RenderNode.transform` 的 x/y/scale_x/scale_y/rotation（render/mod.rs:72-73 改为从 `world_transforms[i]` 分解取，而非 layout_rect 直填）。

> **数学正确性**：GO position=world top-left、rotation=θ、scale=s，mesh 顶点 top-local（0..w，相对 GO origin=world top-left）。box center 世界 = position + R·S·(w/2,h/2) = tx + R·S·(w/2,h/2)；绕 center 旋转等价（§1 pivot 补偿行）。identity（θ=0,s=1）：position=layout top-left，rotation=0，scale=1 → 现状。
>
> **剪切限制**：world matrix 含「非均匀缩放 ∘ 旋转」（列向量非正交）时 TRS 分解有歧义。v1d.3 不保证正确（§0.1/§7 R1）。

### 3.4 命中 world_to_local（hit.rs 改写）

替代裸 `layout_rect` AABB（hit.rs:50）：

```
hit_test(scene, world_point):
  DFS 逆等效绘制序，对节点 n：
    // clip 门控（世界系 AABB，不旋转——决策 5）
    if n.clip_rect 且 world_point 不在 clip_rect → 跳整子树
    local_point = inverse(n.world_transforms[i]) · world_point
    // 本地 box = (0,0,w,h)（top-left 原点；local_point 已在本地系，不用 layout_rect.x/y）
    if n.touchable 且 local_point 在 (0,0, layout_rect.w, layout_rect.h) → 命中
```

> clip_rect 仍是 layout 绝对 design rect（世界系 AABB 判），不随 transform 旋转。本地 box 用 `(0,0,w,h)`（非 layout_rect.x/y）。

### 3.5 batch break（render/batch.rs）

`reorder_for_batching` / `merge_meshes`：**非 identity transform 节点**（`!n.style.transform.is_identity()`）**不参与跨节点 merge**（作 merge 断点）。identity 节点 merge 行为不变（merged transform=0、绝对顶点）。照 fgui breakBatch（§1）。

### 3.5b mesh 顶点 re-base 规则（render/mod.rs + blob.rs，关键正确性）

旋转后 **world top-left ≠ layout top-left**，故 identity 与非 identity 节点顶点处理分两路径：

- **identity 节点**（现状不动）：render 产**绝对 layout 顶点**（`mesh::quad(layout_rect,…)`）；blob re-base 减 `transform.x/y`（=layout top-left=world top-left）→ top-local。参与 merge（绝对顶点 + merged transform=0）。
- **非 identity 节点**：render 产顶点时**减 `layout_rect.xy` → box 本地 (0..w)**；blob **不 re-base**（顶点已本地）。break merge。GO 用 `local_x/y`(=world top-left) + rotation + scale 定位，box 本地顶点经 GO TRS → world。

> 渲染层 `build_render_nodes` 需知节点 identity（读 `style.transform.is_identity()`）决定顶点基；blob.rs 同理决定是否 re-base。两条路径，identity 路径完全保留现状（所有现有测不变）。

### 3.6 后端扁平应用（Unity MirrorPool）

- **identity 节点**：local_x/y=layout top-left，rotation=0/scale=1 → GO `localPosition`=绝对 top-left、`localRotation`=0、`localScale`=1（**现状零改动**）。
- **非 identity 节点**：GO `localPosition`=local_x/y（world top-left）、`localRotation.z`=rotation、`localScale`=(scale_x, scale_y, scale_x)（z 取 scale_x，2D 无深度；y-flip 由根 GO `(1,-1,1)` 统一处理，节点自身不带负号）。rotation 符号以 PlayMode 实测为准（根 y-flip 下方向可能需取反——plan 调）。
- 根 GO 的 `(1,-1,1)` y-flip + shader `Cull Off` 不变。

### 3.7 NativeHost-lite（纯后端 C#，零 core 改动 + 零新 FFI）

- HTML：`<div id="model-slot"></div>` 普通占位（core 当 Container 节点，照常算其 world transform）。
- 后端 API（LoomStage）：
  ```csharp
  public void BindNativeHost(uint nodeId, GameObject go);   // 挂外部 GO 到占位节点
  public void UnbindNativeHost(uint nodeId);
  ```
- `NativeHostManager`（新 C# 类）：每帧 MirrorPool.Sync 后，遍历 `_bindings`：
  - 从 blob 读该 nodeId 的 local_x/local_y/scale_x/scale_y/rotation/visible/sort_key
  - 设外部 GO：position/rotation/scale = world TRS（同 §3.6 flat GO 规则）；`visible=false` → `SetActive(false)`
  - sort_key → `Renderer.sortingOrder`（多 renderer 保相对序，照 GoWrapper.cs:271-281）
  - host 不进 UI 批合（天然——它不是 UI 节点）
- node 从 scene 消失（blob 无该 nodeId）→ 自动 Unbind（`SetActive(false)` + 移除，防 GO 悬空）

## 4. tick 管线插入点（stage.rs `tick_and_render`）

现有（v1d.2）：solve → process(hit+状态) → rematch → render。
v1d.3：solve → **`compute_world_transforms`（新，§3.2）** → process（hit 用 world_to_local §3.4）→ rematch → render（transform 从 `world_transforms` 分解 §3.3）。

> world matrix 必须在 hit 前（命中用本帧刚 solve 布局 + world matrix）。事件回调改 transform 延下帧（同布局反馈环规则，§15）。

## 5. FFI（loomgui_ffi_c/src/lib.rs）

- **transform**：零新 FFI 函数。world TRS 走既有 blob transform 列（扩展，§2.4）。
- **NativeHost**：零新 FFI。后端 C# NativeHostManager 直接读 blob（同 MirrorPool 读法）。
- `loomgui_version()` 字符串 `"v1d.2"` → `"v1d.3"`。
- blob.rs `VERSION` 3→4（§2.4）+ 所有手搓 C# fixture 同步（坑 17）+ 两台机工作流：本机重编 + commit .dll（坑 10）。

## 6. C# mirror（loomgui_unity）

- **FrameBlob**：transform 列解析 2→5（+scale_x/scale_y/rotation 读位，按 §2.4 列序）。`IsValid` 校验 version=4（拒 v3，坑 10）。
- **MirrorPool.Sync**：identity 节点零改动；非 identity 节点设 `localRotation`/`localScale`（§3.6）。
- **NativeHostManager**（新）：`Dictionary<uint, GameObject> _bindings`；每帧 Sync 后遍历，读 blob 同步外部 GO（§3.7）。
- **LoomStage**：`public void BindNativeHost(uint, GameObject)` / `UnbindNativeHost(uint)` 转发 NativeHostManager。
