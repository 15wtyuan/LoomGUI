# v1.2 border-radius 设计

> **定位**：v1.x 批 1 第 2 项。补 v1 围栏外静默丢弃的 `border-radius`（mapping.rs:477 测试断言 `!apply_decl`，AI 写了被静默丢，坑 57 同类），违背 AI 可预测性首要判据。
> **依据**：FairyGUI `RoundedRectMesh.cs` 成熟实现对照 + roadmap.md §1 批 1 第 2 项 + v1.1 background-image 共存基础。
> **fgui 对照**：`temp/FairyGUI-unity/Assets/Scripts/Core/Mesh/RoundedRectMesh.cs`。

---

## 0. 目标 + 非目标

**目标**：
- Container/Button 支持 `border-radius`（1~4 值展开四角，正圆角 + 椭圆角 `h / v` 语法）。
- 支持 px + %（`50%` 画圆/椭圆，AI 高频）。
- 圆角与 background-color + background-image（cover/contain/stretch）三共存。
- radius=0 退回旧 quad 路径（零回归 + dirty hash 不变）。

**非目标**（YAGNI）：
- 圆角描边（lineWidth/lineColor）——border 边框线 v1 都没画，圆角描边更不该进。
- 圆角裁子节点（`overflow:hidden`+radius 裁溢出子节点）——留批 3 soft-clip。
- `border-top-left-radius` 等 longhand。
- 椭圆**形状**（fgui `EllipseMesh`，整节点画椭圆）——无 CSS 先验，围栏无对应属性。
- 圆角精确命中（命中仍用 layout 矩形，圆角外角几像素不命中——可接受，fgui `HitTest` 也用 rect.contains 简化）。
- border-radius 动画 tween 通道——动画改圆角用节点重建（YAGNI，非高频）。

---

## 1. 方案选择

**采用**：`render/mesh.rs` 加 `rounded_rect`，三角扇拓扑（中心点 + 4 角圆弧顶点）。Container/Button 分支：四角 (rx,ry) 全 0 → 旧 `quad`；否则 `rounded_rect`。UV 线性映射，与 v1.1 `fit_uv` 共存。**零新 payload 变体 / 零新 shader / 零新 blob 列 / 零 mask 改动**。

**否决**：
- SDF shader 圆角（仍画 quad，shader 像素 SDF 裁剪）——必改 shader + MaterialManager 新 variant + blob 新 uniform 列，违背"几何在核心 + 零 shader 改动"。
- 内嵌 mask 纹理——新纹理资源 + shader 采样 + 分辨率受限，最重最不值。

## 2. CSS 子集

```
border-radius: <len>{1,4} [ / <len>{1,4} ]?
  <len> = <px> | <%>
```

| 写法 | 语义 |
|---|---|
| `border-radius:8px` | 四角正圆角 r=8 |
| `border-radius:4px 8px` | TL/BR=4, TR/BL=8（CSS 双值：TL/BR vs TR/BL） |
| `border-radius:4px 8px 12px 16px` | TL TR BR BL |
| `border-radius:50%` | 椭圆角：水平半径=50%·w，垂直=50%·h → w=h 时正圆，否则椭圆 |
| `border-radius:10px / 5px` | 椭圆角：水平半径四角 10，垂直半径四角 5 |
| `border-radius:10px 20px / 5px 15px` | 水平四角展开 [10,20,10,20]，垂直 [5,15,5,15] |

**1~4 值展开**（CSS 标准，序 [TL,TR,BR,BL]，与 mapping.rs `parse_four` [top,right,bottom,left] 同序）：
- 1 值：`[v,v,v,v]`
- 2 值：`[v0,v1,v0,v1]`（TL/BR=v0, TR/BL=v1）
- 3 值：`[v0,v1,v2,v1]`
- 4 值：`[v0,v1,v2,v3]`

**`/` 语义**：`/` 前 = 水平半径四角，`/` 后 = 垂直半径四角。省略 `/` → 垂直 = 水平（正圆角）。

**% resolve（渲染期）**：水平半径 = `width × pct/100`，垂直半径 = `height × pct/100`。width/height = layout rect 的 w/h。**不在打包期 resolve**（ResolvedStyle 存 CSS 原始值，layout 尺寸运行时才定）。

**半径钳制（改进 1，vs fgui per-corner min）**：CSS 标准按边缩放——
```
scale = min(1, w/(rx_TL+rx_TR), w/(rx_BR+rx_BL), h/(ry_TL+ry_BL), h/(ry_TR+ry_BR))
所有半径 × scale（只缩不放，防负再 max(0.0))
```
比 fgui `min(w/2,h/2)` per-corner 更准：不对称四角/椭圆角下 fgui 会过度钳制（如 `border-radius:40px 10px` 宽 60，fgui 砍 TL 到 30，实际 40+10=50≤60 不溢不该砍）。

**无效值**：`auto`/`inherit`/`initial`/负数（负数 resolve 后钳 0）/非 px-% 数字 → 整条 `border-radius` 落 `false` 静默忽略（不改默认），与 v1 现有围栏外 CSS 一致。

**围栏同步**：v1-scope §2 把 `border-radius` 从"砍"移到"支持"；主文档 §4.1 把 border-radius 从"装饰属性静默忽略"移除。**触发 docs 改动**（属性新增到围栏 → 同步 v1-scope §2 + 主文档 §4.1/§4.4）。

## 3. 数据结构

### 3.1 LengthPercentage 复用

border-radius 的 px/% 与 taffy `LengthPercentage` 同形（`Length(v)` / `Percent(v)`）。直接复用 `taffy::style::LengthPercentage`，不新造类型。

### 3.2 BorderRadius / CornerRadius（`loomgui_core/src/style/resolved.rs`）

```rust
/// CSS border-radius 单角半径（v1.2）。
/// (h, v) = (水平, 垂直) 半径，存 CSS 原始值（px/%），渲染期 resolve 成像素。
/// `/` 省略时 v = h（正圆角）。
#[derive(Debug, Clone, Copy, PartialEq, Default, Serialize, Deserialize)]
pub struct CornerRadius {
    pub h: LengthPercentage,  // 水平半径
    pub v: LengthPercentage,  // 垂直半径
}

#[derive(Debug, Clone, Copy, PartialEq, Default, Serialize, Deserialize)]
pub struct BorderRadius {
    /// [TL, TR, BR, BL]，与 CSS 1~4 值展开序一致。
    pub corners: [CornerRadius; 4],
}
```

- `Default` = 四角 `Length(0)`（直角，零回归）。
- `Copy`：4×2×2 f32 = 64 字节，可 Copy，调用点免 `&`。
- 走现有 bincode 序列化（ResolvedStyle 已 bincode 进 pkg.bin）。

### 3.3 ResolvedStyle 新字段


  - `read_rejects_unsupported_version` 测试：基线升 v9（TooNew=10, TooOld=8）。

## 4. 解析（`loomgui_core/src/style/mapping.rs`）

### 4.1 `parse_radius_group` helper（新增，不复用 parse_four）

`parse_four` 丢 % 信息（剥 px 后只 parse f32）。border-radius 要保留 px/%，新写：

```rust
/// 解析 border-radius 1~4 值（每值 px 或 %）→ [LengthPercentage;4]（TL,TR,BR,BL）。
/// 与 parse_four 同序，但保留 %。任一值非法 → None（CSS：整条声明无效）。
fn parse_radius_group(s: &str) -> Option<[LengthPercentage; 4]> {
    let parts: Vec<&str> = s.split_whitespace().collect();
    let p = |i: usize| -> Option<LengthPercentage> {
        let s = parts.get(i)?.trim();
        if s == "auto" || s == "inherit" || s == "initial" { return None; }  // §4.3 陷阱
        Some(parse_lp(s))  // parse_lp 现有：px→Length，%→Percent，裸数字→Length
    };
    let v0 = p(0)?;
    Some(match parts.len() {
        1 => [v0, v0, v0, v0],
        2 => [v0, p(1)?, v0, p(1)?],
        3 => [v0, p(1)?, p(2)?, p(1)?],
        _ => [v0, p(1)?, p(2)?, p(3)?],
    })
}
```

- 复用现有 `parse_lp`（line 14），零新解析逻辑。
- `auto`/`inherit`/`initial` 显式判 None（`parse_lp` 默认返 `Length(0)` 会误当直角解析成功，不符 CSS）。
- 任一值解析失败 → `None` → 整条 border-radius 落 false。

### 4.2 `apply_decl` 新 arm

```rust
"border-radius" => {
    // 语法：<len>{1,4} [ / <len>{1,4} ]?  —— / 前水平半径，/ 后垂直半径（省略=同水平）
    let (h_group, v_group) = match value.split_once('/') {
        Some((h, v)) => (h, v),
        None => (value, value),  // 无 / → 垂直 = 水平（正圆角）
    };
    let h = match parse_radius_group(h_group) {
        Some(g) => g,
        None => return false,  // 水平组解析失败 → 整条无效
    };
    let v = match parse_radius_group(v_group) {
        Some(g) => g,
        None => return false,  // 垂直组解析失败 → 整条无效
    };
    style.border_radius = BorderRadius {
        corners: [
            CornerRadius { h: h[0], v: v[0] },  // TL
            CornerRadius { h: h[1], v: v[1] },  // TR
            CornerRadius { h: h[2], v: v[2] },  // BR
            CornerRadius { h: h[3], v: v[3] },  // BL
        ],
    };
    true
}
```

- `split_once('/')`：CSS `/` 两侧允许空格（`10px / 5px`），`split_whitespace` 在 helper 内消化。
- 无 `/` → `v_group = h_group`（同一字符串解析两次，正圆角）。
- 现有测试 `mapping.rs:477` `assert!(!apply_decl(&mut s, "border-radius", "4px"))` **翻转**为 `assert!(...)`——v1.2 兑现的体现。

### 4.3 无效值处理

| 输入 | 结果 |
|---|---|
| `4px` / `50%` / `4px 8px` / `10px / 5px` | ✅ 解析 |
| `auto` / `inherit` / `initial` | `parse_radius_group` p() 返 None → 整条 false |
| `4px abc` | p(1) None → 整条 false |
| `-5px` | `Length(-5.0)` 解析通过，渲染期钳制为 0（§5 改进 1 的 `max(0.0)`） |
| `4px /` / `/ 5px` | 空组 `parse_radius_group("")` → p(0) None → false |

## 5. 渲染（`loomgui_core/src/render/mesh.rs` + `render/mod.rs`）

### 5.1 `rounded_rect` 函数（mesh.rs 新增，与 `quad` 并列）

三角扇拓扑，照搬 fgui `RoundedRectMesh` 核心算法 + 两个改进：

```rust
/// 生成圆角矩形的 verts/uvs/colors/indices（SOA 四表，与 quad 同形）。
///
/// - 三角扇：中心点 + 4 角圆弧顶点，三角形 (0, i, i+1) 连接，末尾回 1 闭合。
/// - radii = [TL, TR, BR, BL]，每角 (h, v) 像素半径（调用前已 resolve，本函数内钳制）。
/// - 全 0 时调用方应走 quad（本函数不特判，仍产正确直角扇但浪费顶点）。
/// - UV 线性映射：顶点归一化位置 (pos-rect.xy)/rect.size × (uv_max-uv_min) + uv_min。
///   与 fit_uv 共存：uv_min/uv_max 即 fit_uv 算出的子区。
/// - 产 design y-down 坐标；v 翻转由调用点交换 uv v 处理（同 quad）。
#[allow(clippy::type_complexity)]
pub fn rounded_rect(
    rect: &Rect,
    color: [f32; 4],
    radii: &[(f32, f32); 4],  // [TL, TR, BR, BL] (h, v) 像素
    uv_min: [f32; 2],
    uv_max: [f32; 2],
) -> (Vec<[f32; 2]>, Vec<[f32; 2]>, Vec<[f32; 4]>, Vec<u32>) {
    let (w, h) = (rect.w, rect.h);
    if w <= 0.0 || h <= 0.0 {
        return quad(rect, color, uv_min, uv_max);  // 退化：空 rect 走 quad
    }
    // 改进 1：CSS 标准按边缩放钳制（vs fgui per-corner min）。
    // 两邻角半径和不超过边长，等比缩放；只缩不放（min(1.0) 兜底）；防负 max(0.0)。
    let (tl, tr, br, bl) = (radii[0], radii[1], radii[2], radii[3]);
    let scale = 1.0_f32
        .min(w / (tl.0 + tr.0).max(1e-6))
        .min(w / (bl.0 + br.0).max(1e-6))
        .min(h / (tl.1 + bl.1).max(1e-6))
        .min(h / (tr.1 + br.1).max(1e-6))
        .min(1.0);
    let clamp = |r: (f32, f32)| -> (f32, f32) {
        ((r.0 * scale).max(0.0), (r.1 * scale).max(0.0))
    };
    let tl = clamp(tl); let tr = clamp(tr); let br = clamp(br); let bl = clamp(bl);

    let lerp = |a: f32, b: f32, t: f32| a + (b - a) * t;
    let push_vert = |verts: &mut Vec<[f32;2]>, uvs: &mut Vec<[f32;2]>,
                     colors: &mut Vec<[f32;4]>, px: [f32;2]| {
        verts.push(px);
        uvs.push([lerp(uv_min[0], uv_max[0], (px[0]-rect.x)/w),
                  lerp(uv_min[1], uv_max[1], (px[1]-rect.y)/h)]);
        colors.push(color);
    };

    let mut verts = Vec::new();
    let mut uvs = Vec::new();
    let mut colors = Vec::new();
    // 中心点（索引 0）
    push_vert(&mut verts, &mut uvs, &mut colors, [rect.x + w/2.0, rect.y + h/2.0]);

    // 改进 2：角序 TL→TR→BR→BL（CSS 视觉直觉序，vs fgui BR→BL→TL→TR）。
    // 起始角：TL=π, TR=-π/2, BR=0, BL=π/2（逆时针 design y-down）。
    // 圆心 = 角顶点内缩 (rx, ry)。
    let corners = [
        (tl, (rect.x + tl.0,         rect.y + tl.1),         std::f32::consts::PI),
        (tr, (rect.x + w - tr.0,     rect.y + tr.1),         -std::f32::consts::FRAC_PI_2),
        (br, (rect.x + w - br.0,     rect.y + h - br.1),     0.0),
        (bl, (rect.x + bl.0,         rect.y + h - bl.1),     std::f32::consts::FRAC_PI_2),
    ];
    for &(r, center, start) in &corners {
        let (rx, ry) = r;
        if rx <= 0.0 || ry <= 0.0 {
            // 直角：单顶点 = 角顶点（照 fgui radius==0 分支，砍 lineWidth）
            let corner_pt = [center.0 + if start == 0.0 || start == -std::f32::consts::FRAC_PI_2 { rx } else { -rx },
                             center.1 + if start == 0.0 || start == std::f32::consts::FRAC_PI_2 { ry } else { -ry }];
            // ↑ 直角时 rx/ry 已被钳到 0，corner_pt 即角顶点；简化直接用 center±(rx,ry) 方向
            push_vert(&mut verts, &mut uvs, &mut colors, corner_pt);
            continue;
        }
        // 自适应分段（照搬 fgui：ceil(π·max(rx,ry)/8)+1，每 ~8px 弧长一段）
        let sides = ((std::f32::consts::PI * rx.max(ry) / 8.0).ceil() as i32 + 1).max(2);
        let delta = std::f32::consts::FRAC_PI_2 / sides as f32;
        for j in 0..=sides {
            let mut a = start + delta * j as f32;
            if j == sides { a = start + std::f32::consts::FRAC_PI_2; }  // 末段精度锁（照 fgui）
            push_vert(&mut verts, &mut uvs, &mut colors,
                      [center.0 + a.cos() * rx, center.1 + a.sin() * ry]);
        }
    }
    // 三角扇：(0, i, i+1)，末尾回 1 闭合
    let n = verts.len() as u32;
    let mut indices = Vec::new();
    for i in 1..n {
        let next = if i + 1 < n { i + 1 } else { 1 };
        indices.extend_from_slice(&[0, i, next]);
    }
    (verts, uvs, colors, indices)
}
```

**照搬 fgui**：三角扇拓扑、自适应分段 `ceil(π·max(rx,ry)/8)+1`、末段精度锁 `start+π/2`、直角分支（`rx<=0||ry<=0` → 单顶点）。

**改进 1（留）**：CSS 标准按边缩放钳制（vs fgui `min(w/2,h/2)` per-corner）——不对称四角/椭圆角下不过度钳制。

**改进 2（留）**：角序 TL→TR→BR→BL（CSS 视觉直觉序）——三角扇闭合与角序无关，视觉等价。

**注**：上方直角分支的 `corner_pt` 表达为示意（用 start 方向判断 ±）；实现时可更直白地按角序硬编码四角的角顶点坐标（TL=(rect.x,rect.y) 等），避免 start 方向判断的绕。plan 期定具体写法。

### 5.2 resolve + 分流（render/mod.rs Container/Button 分支）

现有 Container/Button 分支（render/mod.rs:125-145）的 background-image + fit_uv 逻辑**不变**，在其后追加 border-radius resolve + 分流：

```rust
NodeKind::Container | NodeKind::Button => {
    let color = anim.and_then(|a| a.bg_color)
        .unwrap_or(n.style.background_color.unwrap_or([0.0, 0.0, 0.0, 0.0]));
    let (texture, u_min, u_max) = match &n.style.background_image {
        Some(url) => match textures.get(url) {
            Some(m) => { let (a,b) = fit_uv(n.style.background_size, rect, &m); (m.tex_id, a, b) }
            None => (0u32, [0.0, 0.0], [1.0, 1.0]),
        },
        None => (0u32, [0.0, 0.0], [1.0, 1.0]),
    };  // ↑ 现有逻辑零改

    // border-radius resolve：CSS 原始值 → 像素（% 乘 rect 对应边）
    let resolve = |lp: LengthPercentage, side: f32| -> f32 {
        match lp {
            LengthPercentage::Length(v) => v,
            LengthPercentage::Percent(p) => side * p,
        }
    };
    let (rw, rh) = (rect.w, rect.h);
    let bc = &n.style.border_radius.corners;  // [TL,TR,BR,BL]
    let radii = [
        (resolve(bc[0].h, rw), resolve(bc[0].v, rh)),  // TL
        (resolve(bc[1].h, rw), resolve(bc[1].v, rh)),  // TR
        (resolve(bc[2].h, rw), resolve(bc[2].v, rh)),  // BR
        (resolve(bc[3].h, rw), resolve(bc[3].v, rh)),  // BL
    ];
    let all_zero = radii.iter().all(|&(rx, ry)| rx <= 0.0 || ry <= 0.0);

    // v 翻转（同现有 quad/Image 分支：交换 uv v 传给 mesh 函数）
    let (v, uvc, col, idx) = if all_zero {
        crate::render::mesh::quad(rect, color, [u_min[0], u_max[1]], [u_max[0], u_min[1]])
    } else {
        crate::render::mesh::rounded_rect(
            rect, color, &radii,
            [u_min[0], u_max[1]], [u_max[0], u_min[1]],
        )
    };
    rn.payload = NodePayload::Mesh {
        verts: v, uvs: uvc, colors: col, indices: idx, texture, program: 0,
    };
}
```

**关键点**：
- `%` resolve 在 `build_render_nodes`（有 layout rect），**不在打包期**——ResolvedStyle 存 CSS 原始值，跨 pkg.bin 序列化无损。
- `all_zero` 走旧 `quad`——radius=0 时 mesh 与 v1.1 完全一致（4 顶点 quad），dirty hash 不变（坑 75 对齐）。
- v 翻转：rounded_rect 产 design y-down，调用点传 `[u_min[0], u_max[1]], [u_max[0], u_min[1]]`（交换 v）——与现有 quad/Image 分支同手法，rounded_rect 的 UV 线性映射用这组翻转后的 uv_min/uv_max，v 自然随顶点 y 翻。
- Image 分支（NodeKind::Image）**不加 border-radius**——`<img>` 是贴图节点，圆角由其容器或自身 background-image 路径处理；img 直接贴图走 quad（§0 非目标未列但属 YAGNI：img 圆角非高频，AI 用 div+background-image+radius 替代）。

## 6. 共存语义

### 6.1 background-color + background-image + border-radius 三共存

| 层 | 来源 | 进 Mesh 哪个字段 |
|---|---|---|
| background-color | `style.background_color` / `anim.bg_color` | `colors`（每顶点 [f32;4]） |
| background-image | `style.background_image` + `fit_uv` | `texture` + `uvs`（uv_min/uv_max 子区） |
| border-radius | `style.border_radius` | `verts`（圆角顶点位置）+ `uvs`（随顶点线性映射） |

- **圆角 + 纯色**：rounded_rect 顶点色 = background-color，圆角外无顶点 = 不画 = 透明透出底层。✅
- **圆角 + 背景图**：rounded_rect UV 按顶点归一化位置线性映射到 `fit_uv` 子区。
  - cover：子区是图中央可见区，圆角顶点落 rect 内 → UV 落子区内 → 图对应区域贴圆角面。✅
  - contain：子区外扩（留白 clamp），圆角顶点 UV 仍在子区线性范围 → 图完整贴入，圆角外透 background-color。✅
  - stretch：子区 = 整图，圆角顶点 UV = 归一化位置 → 图拉伸贴满圆角面。✅
- **shader 零改**：`LoomGUI/Unlit` `mainTexture × vertexColor`——圆角顶点同走，图透明区透顶点色。圆角外像素无顶点覆盖，不进光栅，无需 shader 裁剪。

### 6.2 border-radius + overflow:hidden（子节点）

- **仅自身圆角**：Container 自身画圆角背景，`overflow:hidden` 裁子节点仍走**矩形** `_ClipBox`（现有 mask 机制）。
- 后果：圆角容器里溢出子节点，圆角外角区域被矩形裁（直角裁），不随圆角裁——子节点可能"探出"圆角背景圆弧。
- **决策**：v1.2 接受（圆角裁子节点留批 3 soft-clip）。showcase §3.7 观察；若明显，批 3 解决。
- **不阻塞**：圆角按钮/卡片背景是主用例（按钮文字在圆角内不溢出），子节点溢出圆角是 edge case。

### 6.3 border-radius + border-width（边框线）

- v1 **不画 border 边框线**（border_width 仅盒模型占位）。border-radius 不与边框线交互——圆角是背景几何，边框线是另一层（v1 未实现）。
- 未来画 border 线需 border 也走圆角路径（fgui lineWidth 分支）——v1.2 不预留（YAGNI）。

## 7. FFI / Unity 后端（零改）

- **blob**：Mesh arena 已是 `Vec` 四表（verts/uvs/colors/indices），圆角 mesh 只是更多顶点/索引，**无新列、无新字段**。FrameBlob.cs 上传对 4 顶点 quad 和 N 顶点圆角扇一视同仁。
- **MirrorPool**：`tex_id → Texture2D` 查找、GO 镜像 diff 不关心顶点数——payload 变即重建 mesh 上传。圆角 mesh 的 `texture` 沿用 v1.1 background-image 机制。
- **MaterialManager**：DrawState key = `(program, texture, mask_context, matrixFlag)`——圆角 mesh 不引入新 key 维度。同 texture 圆角按钮与直角按钮仍可同批合。
- **shader**：`LoomGUI/Unlit` `mainTexture × vertexColor` 零改——圆角外无顶点 = 不光栅 = 自然透明，无需 SDF/stencil。
- **裁剪**：`mask_context` + `_ClipBox` 矩形硬裁剪零改——border-radius 不进裁剪链（§6.2）。
- **dirty hash**（坑 75 对齐）：Mesh payload hash 已含 verts/uvs/colors/texture。
  - radius>0：verts 4→N → hash 变 → 视觉 dirty（正确）。
  - radius 全 0：走旧 quad，verts/uvs 与 v1.1 同 → hash 不变 → Unchanged（零回归）。
  - radius 值改（hover 切 4px→8px）：verts 变 → hash 变 → 视觉 dirty。✅
  - **无需额外改 dirty.rs**——现有 hash 已捕获 verts 变化。

**唯一 FFI 侧动作**：重建 `loomgui_ffi_c.dll`（核心改了 ResolvedStyle + mesh.rs + mapping.rs）。按双机工作流，本机 build + commit dll，家里机 pull 后 PlayMode 验。

## 9. 测试

### 9.1 Rust 单测（core）

**mapping（style/mapping.rs）**：
- `parse_radius_group`：`8px`→`[8,8,8,8]`；`4px 12px`→`[4,12,4,12]`；`50%`→`[0.5,0.5,0.5,0.5]`（Percent）；3 值/4 值展开序正确。
- `auto`/`inherit`/`initial` → `None`（§4.3 陷阱）。
- 非法 `4px abc` → `None`。
- `apply_decl("border-radius","8px")` → `true` + `corners` 全 `Length(8)`。
- `/` 语法：`"10px / 5px"` → h 全 `Length(10)`，v 全 `Length(5)`。
- `/` 省略：`"8px"` → v = h（正圆角）。
- `/` 一侧非法：`"8px / abc"` → `false`。
- 现有测试 `mapping.rs:477` 翻转：`assert!(apply_decl(&mut s, "border-radius", "4px"))`。

**rounded_rect（render/mesh.rs）**：
- 全 0 直角：`radii=[(0,0);4]` → 产三角扇但视觉=直角矩形（调用点走 quad 分流，测 all_zero 分流）。
- 正圆角 `[(8,8);4]` × 80×80 rect：顶点数 = 1(中心) + 4×(sides+1)，sides=ceil(π·8/8)+1=4 → 1+4×5=21 顶点；三角形 = 20。
- 半径钳制（改进 1）：`[(40,40);4]` × 60×40 rect——TL+TR 水平和=80>60 → scale=60/80=0.75 → 实际半径 30；验证顶点落点 = 30 而非 40。
- 椭圆角 `[(20,10);4]`：顶点弧 `cos·20, sin·10`，验证某角顶点坐标 = center + (cos·20, sin·10)。
- 末段精度锁：末顶点 angle 恰为 `start+π/2`（与首顶点对齐无缝隙）。
- UV 线性映射：rect=(0,0,80,80)，uv_min=[0,0] uv_max=[1,1]，中心顶点 UV=[0.5,0.5]；角顶点 UV = 归一化位置。
- 退化：`rect.w=0` → 走 quad fallback（§5.1）。

**resolve + 共存（render/mod.rs Container 分支）**：
- `%` resolve：`border-radius:50%` × 80×80 rect → radii 全 (40,40) → rounded_rect 调用。
- `all_zero` 走 quad：`border-radius:0` / 未设 → payload verts.len()==4（quad）。
- 圆角 + 背景图：`background-image:url(x)` + `border-radius:8px` → payload `texture` 非 0 + verts.len()>4 + uvs 随顶点。
- 圆角 + 纯色：无 background-image → `texture`==0 + verts.len()>4。

**dirty（style/dynamic.rs）**：
- `border_radius` 是视觉字段——改值不触发 layout dirty（同 background_color/background_image，机制 `layout_changed = taffy_style != old || order != old`，新视觉字段自动归视觉）。验证测试：改 border_radius → 视觉 dirty 非 layout dirty。

**dirty hash（render/dirty.rs）**：
- 圆角 mesh verts 进 hash（坑 75 后 uvs 也进）——改 radius 值 → verts 变 → hash 变 → 视觉 dirty。
- radius=0 走 quad → hash 与 v1.1 同 → Unchanged（零回归）。
- 回归测试：同节点 radius 0→8 hash 必变。
- **无需新增 dirty.rs 实现**（verts 已在 hash）。

### 9.2 打包器测试（loomgui_pkg）

- border-radius 是 ResolvedStyle 字段，`pack_inner` 已序列化整 ResolvedStyle → **打包器零改**，无新收集链（不像 background-image 要进 atlas srcs）。
- 回归：含 border-radius 的包能正确打包 + 反序列化（bincode 往返 `border_radius` 字段无损）。
- PKG_FORMAT_VERSION 9 拒绝测试：旧 v8 pkg 被 `read_rejects_unsupported_version` 拒（基线升 v9，TooNew=10/TooOld=8）。

### 9.3 Showcase 验收（家里机 PlayMode）

见 §8 R1-R6。

### 9.4 跨引擎一致（设计契约）

- 圆角 mesh 在 Rust 核心生（§3.3）→ Unity/Godot 后端零几何差异（主文档 G1）。无 Unity 专属圆角逻辑（不像 SDF shader 方案会引擎分歧）。

## 10. 已知限制 / 后续

- **圆角裁子节点未实现**（§6.2）：`overflow:hidden`+border-radius 的子节点溢出仍矩形裁，不随圆角裁。留批 3 soft-clip。
- **border 边框线未画**（§6.3）：border-radius 不与 border 描边交互。未来画 border 线需 border 也走圆角路径。
- **img 不支持 border-radius**（§5.2）：`<img>` 走 quad，圆角用 div+background-image+radius 替代。img 圆角非高频。
- **圆角命中仍矩形**（§0）：圆角外角几像素不命中。可接受，fgui 同简化。
- **椭圆形状未支持**：`EllipseMesh` 整节点画椭圆，无 CSS 先验，围栏无属性。v2 几何扩展。
- **border-radius 动画**：用节点重建；独立 tween 通道留 v2（YAGNI）。
