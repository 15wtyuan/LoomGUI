# v1d.2 键盘 + 焦点 + Tab + `:focus` 设计契约

> 范围来源：`docs/roadmap/v1d-plan.md` §3 v1d.2（defer #3 键盘 + #6 焦点/Tab/`:focus`）。
> 模式：检测全在 core（loomgui_core），C#（LoomEventHandler）只路由/dispatch。照 v1d.1。
> ★ 已对照 fgui 源码（`temp/FairyGUI-unity/`）：见 §1 对照表。

## 0. 决策（已拍）

| # | 决策 | 选定 |
|---|---|---|
| 1 | key 事件 ABI 通道 | **A**：复用既有 EventRecord 流（EVT_KEY_DOWN=12/UP=13，key_code 复用 `touch_id`、modifiers 复用 `pad[0]`，零新输出 ABI、单流） |
| 2 | focus opt-in 模型 | **A**：HTML `tabindex` 属性（0=DOM序/N=显式序/-1=仅编程；无属性=不可聚焦） |
| 3 | click-to-focus | **A**：pointer-down 命中 `tabindex>=0` 节点 → 自动聚焦（照 fgui+DOM）；不可聚焦/`-1` 不夺焦 |
| 4 | Tab 导航 | **A**：Tab/Shift+Tab 按 tabindex 序遍历，链尾 wrap（照 DOM）；core 内算导航 |
| 5 | modifiers | **A**：`KeyEvent.modifiers: u8` 位掩码（bit0=shift/bit1=ctrl/bit2=alt，照 fgui InputEvent.cs:27/32/37） |

### 0.1 不范围（明确 defer，不进 v1d.2）

- **IME / 字符输入**：keydown/up 只给 `key_code`（KeyCode 枚举值），不给 `character`。IME 合成字符随 TextInput defer v1.x。fgui `InputEvent.character` 有是因为有 InputTextField；LoomGUI v1d.2 无文本输入控件，character 无消费者。
- **TextInput 控件**：v1.x。
- **`:focus-visible`**：v1d.2 只做 `:focus`（聚焦即匹配）。`:focus-visible`（区分键盘/鼠标聚焦成因）需 core 记焦点成因，YAGNI，留 v1.x。

## 1. fgui 对照（已核实源码）

| 机制 | fgui | LoomGUI v1d.2 | 异同 |
|---|---|---|---|
| 键盘事件 bubble 源 | `Stage.focus`（无焦点→Stage 全局，Stage.cs:958-962） | `Scene.focused_node`（无焦点→不发 key 事件，§3.2） | LoomGUI 无 Stage 全局兜底——无焦点即丢弃 key（core 不持有"全局"概念，焦点是节点级） |
| key 字段 | keyCode + character + modifiers（InputEvent.cs:27/32/37） | key_code + modifiers（无 character，§0.1） | 砍 character |
| 焦点模型 | 单一全局 `Stage.focus`（Stage.cs:35/326） | `Scene.focused_node: Option<NodeId>`（单一全局） | 同 |
| 可聚焦 opt-in | `focusable` bool 默认 true（DisplayObject.cs:919） | `tabindex` 属性（默认不可聚焦） | **异**：LoomGUI 无原生可聚焦元素（无 `<input>`），默认 off 更合理；tabindex 兼 DOM |
| click-to-focus | pointer-down → SetFocus(target)（Stage.cs:1127/1181/1282） | pointer-down 命中 `tabindex>=0` → focus | 同 |
| FocusIn/FocusOut | 派发到焦点链祖先（Stage.cs:412-441） | FocusIn/FocusOut 事件进 EventRecord 流（node_id=焦点节点） | 同语义，通道异（fgui 回调 vs LoomGUI EventRecord） |
| `:focus` 伪类 | **无**（fgui 无 CSS） | `pseudo_focus` flag + selector `:focus` + compound 门控 | LoomGUI 补 fgui 没有的 |
| Tab 导航 | `DoKeyNavigate(backward)` 按 `tabStop` bool + DFS 序（Stage.cs:487-537），无数值 tabIndex | Tab/Shift+Tab 按 tabindex 数值序遍历 | **异**：tabindex 数值序是 fgui 没有的能力 |
| modifiers | ctrl/shift/alt/cmd（InputEvent.cs:27/32/37） | shift/ctrl/alt（无 cmd，Unity 无 cmd 概念，bit0-2） | 砍 cmd |

## 2. 数据结构变更

### 2.1 input.rs

```rust
/// 键盘输入事件（FFI POD）。C# set_key_input 推一组；core process 产 keydown/up EventRecord。
#[repr(C)]
#[derive(Debug, Clone, Copy)]
pub struct KeyEvent {
    pub key_code: u32,   // KeyCode 枚举值（Unity KeyCode 转 u32，core 不解释语义，只透传+Tab 判定）
    pub modifiers: u8,   // bit0=shift / bit1=ctrl / bit2=alt
    pub is_down: bool,   // true=按下→keydown；false=松开→keyup
    pub pad: [u8; 3],    // 对齐（KeyEvent 8B）
}
// KeyEvent 8B：key_code@0(4) + modifiers@4(1) + is_down@5(1) + pad@6(2)。
```

新 EVT 常量（复用既有 EventRecord event_type 空位，10/11 留给未来，本批用 12-15）：

```rust
pub const EVT_KEY_DOWN: u8 = 12;
pub const EVT_KEY_UP: u8 = 13;
pub const EVT_FOCUS_IN: u8 = 14;
pub const EVT_FOCUS_OUT: u8 = 15;
```

> **ABI 不变**：EventRecord 仍 20B、PointerEvent 仍 16B。key 事件复用 EventRecord 字段：
> - `node_id` = 当前焦点节点（keydown/up）或焦点变化相关节点（FocusIn/Out，见 §3.3）
> - `event_type` = 12/13/14/15
> - `click_count` = 0（key 事件无意义）
> - `pad[0]` = modifiers（key 事件；FocusIn/Out=0）
> - `pad[1]` = 0
> - `touch_id` = key_code（i32 重解释 u32 bit pattern；key 事件；FocusIn/Out=0）
> - `x`/`y` = 0.0（key 事件无指针位置）

## 3. 检测逻辑（全在 core）

### 3.1 Tab 链构造

core 每次需要导航时（收到 Tab keydown）即时构造 Tab 链（不缓存——节点树可能变，缓存失效成本 > 即时构造）：

```rust
/// 收集所有 tabindex>=0 节点，按 (tabindex 升序, DFS DOM 序) 排序。
/// tabindex=0 组排在所有正整数组之后（照 DOM：正整数显式序先于 0 组）。
fn build_tab_chain(scene: &Scene) -> Vec<NodeId> {
    let mut positive: Vec<(i32, NodeId)> = Vec::new();   // tabindex>0
    let mut zero: Vec<NodeId> = Vec::new();              // tabindex==0（DFS 序）
    // DFS 先序遍历 roots（与 Scene::build 填 children 序一致 = DOM 序）
    for root in &scene.roots {
        dfs_collect(scene, *root, &mut positive, &mut zero);
    }
    positive.sort_by_key(|(t, _)| *t);   // 正整数按 tabindex 升序（同值保持 DFS 序，stable）
    positive.iter().map(|(_, n)| *n).chain(zero.iter().copied()).collect()
}
```

- `tabindex=-1` / `None` → 不进链（-1 仅编程聚焦，None 不可聚焦）。
- `disabled` 节点跳过（聚焦 disabled 无意义，照 fgui focusable+disabled 不获焦）。
- `dfs_collect` 签名：`fn dfs_collect(scene: &Scene, id: NodeId, positive: &mut Vec<(i32,NodeId)>, zero: &mut Vec<NodeId>)`——递归 children（先序=DOM 序），按 tabindex 分桶（`Some(t) if t>0` → positive push `(t,id)`；`Some(0)` → zero push；其余跳过），disabled 跳过。

### 3.2 keydown/up 处理

Stage tick 管线（§4）process 指针事件后，处理 pending_keys：

```rust
for ke in &pending_keys {
    let focused = scene.focused_node;
    match (ke.is_down, ke.key_code) {
        // Tab 导航：仅 keydown 触发（keyup 不导航）
        (true, TAB) => {
            let chain = build_tab_chain(scene);
            if chain.is_empty() { continue; }   // 无可聚焦节点 → Tab 无操作（不发 keydown）
            let backward = (ke.modifiers & MOD_SHIFT) != 0;
            let next = next_focus(&chain, focused, backward);
            focus_node(scene, next, &mut out);   // 发 FocusOut(旧) + FocusIn(新)，见 §3.3
        }
        // 普通 keydown/up：有焦点才发（无焦点丢弃，§1）
        (true, _) => {
            if let Some(n) = focused {
                out.push(key_event_record(n, EVT_KEY_DOWN, ke));
            }
        }
        (false, _) => {
            if let Some(n) = focused {
                out.push(key_event_record(n, EVT_KEY_UP, ke));
            }
        }
    }
}
```

- `next_focus(chain, current, backward)`：current 在链中 → 取前/后一个；current 不在链（或 None）→ 链首（forward）/链尾（backward）；链尾 forward → wrap 链首；链首 backward → wrap 链尾。
- Tab/Shift+Tab **不发 keydown EventRecord**（导航消费了 Tab——照 DOM：Tab 默认动作是移焦，不冒泡 keydown 给业务。业务要拦 Tab 须 v1.x preventDefault，本批不做）。
- 其他键（Enter/Space/Esc/字母…）有焦点即发 keydown/up，业务 listener 自行处理（如 Enter 触发 click 是业务逻辑，core 不自动合成）。

### 3.3 焦点变化 + FocusIn/FocusOut

```rust
/// 设焦点为 new（None=清除焦点）。发 FocusOut@旧焦点 + FocusIn@新焦点。
/// input.rs 模块级自由函数（非 PointerState 方法）——process（click-to-focus）+ process_keys
/// （Tab 导航）+ Stage::request_focus（pending_focus_request 消费）三处共用。
fn focus_node(scene: &mut Scene, new: Option<NodeId>, out: &mut Vec<EventRecord>) {
    let old = scene.focused_node;
    if old == new { return; }   // 无变化不发
    // 清旧焦点 focused 标志
    if let Some(o) = old {
        if o.0 < scene.nodes.len() { scene.nodes[o.0].focused = false; }
        out.push(EventRecord { node_id: o.0 as u32, event_type: EVT_FOCUS_OUT, click_count:0, pad:[0,0], touch_id:0, x:0.0, y:0.0 });
    }
    // 设新焦点
    if let Some(n) = new {
        if n.0 < scene.nodes.len() { scene.nodes[n.0].focused = true; }
        out.push(EventRecord { node_id: n.0 as u32, event_type: EVT_FOCUS_IN, click_count:0, pad:[0,0], touch_id:0, x:0.0, y:0.0 });
    }
    scene.focused_node = new;
}
```

- FocusIn/Out 的 `node_id` = 焦点节点本身（不是祖先链——照 fgui onFocusIn/Out 派发对象）。
- `:focus` 伪类匹配靠 `node.focused`，rematch_pseudo_classes 每帧跑（既有机制，§4）自动吃焦点变化。

### 3.4 click-to-focus（pointer-down arm）

input.rs `PointerKind::Down` arm 加（hit 算出后）：

```rust
// v1d.2：click-to-focus——pointer-down 命中 tabindex>=0 节点 → 聚焦（照 fgui+DOM）。
// 沿命中链找最近可聚焦节点（叶子优先，同 drag_target 模式）。
// 不可聚焦/`-1` → 不夺焦（照 DOM：点空白不 blur）。
let focus_target = slot.down_targets.iter()
    .find(|&&n| n.0 < scene.nodes.len()
        && !scene.nodes[n.0].disabled
        && matches!(scene.nodes[n.0].tabindex, Some(t) if t >= 0))
    .copied();
if let Some(t) = focus_target {
    focus_node(scene, Some(t), &mut out);
}
```

- 但 `focus_node` 改 `scene.focused_node` + 发 FocusIn/Out——input.rs `process` 借 `&mut scene`，需 `focus_node` 也借 `&mut scene`。同函数内顺序借用 OK（process 已持 `&mut scene`，直接调）。
- **注意借用**：`process(&mut self, &mut scene, ...)`——focus_node 需 `&mut scene`，process 已持。`out` 是局部 Vec。可行。
- click-to-focus 产 FocusOut(旧) + FocusIn(新) 进同一 `out`，与 pointer 事件同流返回。

### 3.5 disabled 节点不聚焦

- click-to-focus：`disabled` 跳过（§3.4 find 条件含 `!disabled`）。
- Tab 链：`disabled` 跳过（§3.1）。
- `request_focus` API（§5）：disabled 节点 request → no-op（照 fgui）。

## 4. Stage tick 管线（§15 增量）

```rust
pub fn tick_and_render(&mut self) -> FrameData {
    let scene = self.scene.as_mut().expect("load first");
    let mut out = Vec::new();
    // ⓪ v1d.2：消费 pending_focus_request（编程聚焦/清焦点，tick 外 request_focus/blur 记）
    if let Some(req) = self.pending_focus_request.take() {
        focus_node(scene, req, &mut out);   // FocusOut(旧)+FocusIn(新)
    }
    solve(scene, &self.font, self.root_size, &self.textures);
    let input = std::mem::take(&mut self.pending_input);
    let mut ptr_out = self.pointer_state.process(scene, &input);   // ① 指针事件（含 click-to-focus 的 FocusIn/Out）
    out.append(&mut ptr_out);
    // ② v1d.2：键盘事件（keydown/up + Tab 导航 + FocusIn/Out）
    let keys = std::mem::take(&mut self.pending_keys);
    process_keys(scene, &keys, &mut out);
    self.last_events = out;
    rematch_pseudo_classes(scene);   // ③ :focus/:hover/:active 重匹配（吃 focused/hovered/active 变化）
    build_render_nodes(scene, &self.font, &self.textures)   // ④
}
```

- `process_keys` 是 input.rs 模块级自由函数（pub(crate)），见 §3.2。
- FocusIn/Out 由三处产生（pending_focus_request 消费 + click-to-focus in process + Tab 导航 in process_keys），都进同一 `out`。
- rematch 在 process + process_keys 之后——`:focus` 伪类吃到本帧焦点变化，本帧 render 立即生效（视觉变）。
- **pending_focus_request 在 solve 前**：编程聚焦可能在 tick 外调（C# 任意时刻），须在下 tick 最前消费，确保该 tick 的 hit/rematch 都用新焦点。

## 5. FFI（loomgui_ffi_c/src/lib.rs）

新增 3 个常驻函数（不 gate——输入是 runtime 稳定入口）：

```rust
/// v1d.2：注入本帧键盘事件（扁平 KeyEvent 数组）。tick 前调。null/len=0 = 无键盘输入。
#[no_mangle]
pub extern "C" fn loomgui_stage_set_key_input(h: *mut StageHandle, keys: *const KeyEvent, len: usize) {
    if h.is_null() { return; }
    let sh = unsafe { &mut *h };
    if keys.is_null() || len == 0 { sh.stage.set_key_input(&[]); return; }
    let ks = unsafe { std::slice::from_raw_parts(keys, len) };
    sh.stage.set_key_input(ks);
}

/// v1d.2：编程聚焦节点（照 fgui RequestFocus）。tabindex=-1/None/disabled 也接受（编程可强制聚焦
/// tabindex=-1 节点；disabled 仍拒——§3.5）。NodeId 越界静默跳过。null 句柄 → no-op。
#[no_mangle]
pub extern "C" fn loomgui_stage_request_focus(h: *mut StageHandle, node_id: u32) {
    if h.is_null() { return; }
    let sh = unsafe { &mut *h };
    sh.stage.request_focus(NodeId(node_id as usize));
}

/// v1d.2：读当前焦点节点。无焦点/无 scene → 0xFFFF_FFFF（sentinel，同 node_parent）。
#[no_mangle]
pub extern "C" fn loomgui_stage_focused_node(h: *const StageHandle) -> u32 {
    const NONE: u32 = 0xFFFF_FFFF;
    if h.is_null() { return NONE; }
    let sh = unsafe { &*h };
    match &sh.stage.scene {
        Some(scene) => scene.focused_node.map(|n| n.0 as u32).unwrap_or(NONE),
        None => NONE,
    }
}
```

Stage 加（`pending_keys: Vec<KeyEvent>` 字段 + `pending_focus_request: Option<Option<NodeId>>` 字段——外层 Some=有请求，内层 Some(id)=聚焦某节点 / None=清焦点）：

```rust
pub fn set_key_input(&mut self, keys: &[KeyEvent]) {
    self.pending_keys.clear();
    self.pending_keys.extend_from_slice(keys);
}
/// v1d.2：编程聚焦（照 fgui RequestFocus）。强制聚焦任意非 disabled 节点
/// （含 tabindex=None/-1——request_focus 是编程 API，不查 tabindex，照 fgui 不查 focusable）。
/// disabled 拒（§3.5）。NodeId 越界静默跳过。不直接写 last_events——记 pending_focus_request，
/// 下 tick_and_render 开头消费（避免 R3：tick 覆盖 last_events 丢事件）。
pub fn request_focus(&mut self, node_id: NodeId) {
    if let Some(scene) = self.scene.as_ref() {
        if node_id.0 >= scene.nodes.len() { return; }
        if scene.nodes[node_id.0].disabled { return; }   // §3.5 disabled 拒
    } else { return; }
    self.pending_focus_request = Some(Some(node_id));
}
/// v1d.2：编程清焦点。同 request_focus，pending_focus_request = Some(None)。
pub fn blur(&mut self) {
    self.pending_focus_request = Some(None);
}
```

## 6. C# 镜像（loomgui_unity）

### 6.1 LoomInputCollector

- 加键盘采集：`Update` 读 `Input.GetKeyDown`/`GetKeyUp`（KeyCode 遍历或 Event.current），组 `KeyEvent[]` 调 `set_key_input`。

### 6.2 LoomEventHandler

`EventType` 加 4 个（:byte，照既有）：

```csharp
public enum EventType : byte {
    // 既有 0-9...
    DragStart = 6, DragMove = 7, DragEnd = 8, LongPress = 9,
    KeyDown = 12, KeyUp = 13, FocusIn = 14, FocusOut = 15,
```

`DispatchPending` switch +4 case → BubbleRoute（keydown/up/focus/blur 都 bubble，照 fgui）。

### 6.3 EventType 字段扩展

既有 EventContext 读 `EventRecord` 字段。key 事件需读 key_code + modifiers：

```csharp
// EventContext 加（从 EventRecord touch_id 字段读 key_code，pad[0] 读 modifiers）
public uint KeyCode => (uint)rec.touch_id;   // i32 bit pattern → u32
public byte Modifiers => rec.pad0;            // 既有点位读取（pad[0]）
```

## 7. 风险

- **R1（借用）**：`process` 持 `&mut scene`，click-to-focus 调 `focus_node(&mut scene, ...)`——同函数内顺序调，无并发借用。`process_keys` 在 `process` 返回后调，独立持 `&mut scene`。可行，但实现须注意 `down_targets` 借用期不再持 scene 引用（find 用索引，§3.4 已用 `.copied()`）。
- **R2（Tab 不发 keydown）**：Tab 被导航消费，业务收不到 Tab keydown。若业务要拦 Tab（如自定义导航），v1.x 加 preventDefault。本批 spec 明确此行为（§3.2），不视为 bug。
- **R3（request_focus 时机，已解）**：`request_focus` 在 tick 外调（C# 任意时刻）。若直接写 `last_events`，下 tick 开头 `last_events = out` 全量覆盖会丢事件。**解法**：request_focus 不写 last_events，记 `pending_focus_request: Option<Option<NodeId>>`，tick_and_render 最前（solve 前）消费进 `out`。blur 同理（`pending_focus_request = Some(None)`）。见 §4/§5。
- **R4（pkg.bin v5 迁移）**：旧 v4 包被拒（MIN=MAX=5）。PlayMode 用新打包器重打 interact sample。无线上旧包（v1 dev 期），无迁移负担。
- **R5（Node::default 补字段）**：加 tabindex/focused 后，`Node::default()`（node.rs:72）须补 `tabindex: None, focused: false`，否则编译错。`Scene::build` 内构造 Node 也要填 tabindex（从 entry）+ focused:false。实现者注意。
- **R6（request_focus 强制语义）**：request_focus 强制聚焦任意非 disabled 节点（含 tabindex=None/-1），照 fgui RequestFocus 不查 focusable。与 click-to-focus（查 tabindex>=0）/Tab 链（查 tabindex>=0）不同——编程 API 更宽松。§5 已明确。
