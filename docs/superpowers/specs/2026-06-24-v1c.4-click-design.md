## §0 决策记录（含 fgui 调研结论）

1. **「缩放容忍」是伪需求，删除**。fgui `_clickTestThreshold` 是**固定屏幕像素**（mouse 10 / touch 50），全项目 7 处引用无一处乘 scaleFactor/DPI。`UIContentScaler.scaleFactor` 只作用 GRoot design 空间，不影响 Stage 输入坐标。LoomGUI click 判定已在 design 空间（采集层 `ScreenToDesign` 已换算），阈值固定 design px 即正确，无需额外缩放机制。v1c.4 不实现此项。
2. **双击无独立事件**（照 fgui）。无 `onDoubleClick`；消费侧在 onClick 读 `clickCount==2` / `isDoubleClick`。clickCount 累进 **1→2→1 循环**（不到 3）。
3. **时间源复用已有 FFI**：`loomgui_stage_tick(h, dt)` 已收 `Time.deltaTime`（core 忽略为 `_dt`）。C# 改传 `Time.unscaledDeltaTime`，core 累积 `time_s += dt`。**零新增 FFI 参数**。照 fgui `Time.unscaledTime`。
4. **clickCount 载体复用 pad 字节**：`EventRecord.pad:[u8;3]` 未用 → 改 `click_count:u8` + `pad:[u8;2]`，**大小不变（20B）**，C# `LoomEvent` 加 `byte clickCount`（offset 5）。零版本/layout 破坏。
5. **Canceled 表达**：`PointerKind::Canceled=3`（repr(u8)，`PointerEvent` 仍 16B）。
6. **⚠️ 唯一偏离 fgui — Canceled 语义**：fgui `End()` 对 Canceled 仍可能跑 clickCount 累加分支（仅 `ClickTest`+onClick 被跳过，clickCount 状态悄悄前进，quirk）。LoomGUI 改为：`Canceled` 直接置 `click_cancelled=true` → Up 仍发、ClickTest 返 None（不发 Click）、cancel 分支 reset `click_count=1`+`last_click_time=0`。即「Canceled = 隐式 CancelClick」，与 fgui `CancelClick()` 行为一致、更干净。**用户已确认（2026-06-24）**。
7. **阈值比较从 euclidean 改 per-axis**：fgui 用 `Mathf.Abs(x-downX) > t`，LoomGUI 现 v1c.3 用 `√(dx²+dy²)≤10`。改为 per-axis `|dx|>t || |dy|>t` 对齐（对角线更宽松）。

## §1 架构（Direction A，照 v1c.1-3 分工）

Click 判定**全部在 core `input.rs`**（照 fgui `TouchInfo` 全在 Stage）。理由：

- core 已持命中（`hit_test`）+ 状态机（`PointerState`）+ `EVT_CLICK` 产出。
- 双击时间状态（`last_click_time`）与 click 判定强耦合（同一次 Up 既判位移又判 clickCount），分置 core/C# 会重复持 down 状态。
- C# 仅：路由（`LoomEventHandler`）、`stopImmediate`、`CancelClick` 转发、监听器表——与 v1c.1-3 一致。

## §3 Click 判定（core，照 fgui `End()`+`ClickTest()`+`Move()`）

### 3.1 Down（Begin）

`down_targets = ancestor_chain(hit)`（leaf→root，复用现有 `ancestor_chain`）。`click_cancelled = false`。（其余同 v1c.3：设 `is_down`/`down_pos`/`down_node`，disabled 不产 Down。）

### 3.2 Move（Move 取消判定，新增）

若 `is_down && (|dx|>MOVE_CANCEL_PX || |dy|>MOVE_CANCEL_PX)` → `click_cancelled = true`。（`dx,dy` = `last_pos - down_pos`，per-axis。）照 fgui `Stage.cs:1705`。hover_diff + monitor Move 派发逻辑同 v1c.3 不变。

### 3.3 Up / Canceled（End）

`slot.is_down = false`。**Up 与 Canceled 都发 `EVT_UP`**（照 fgui onTouchEnd 两路径都跑），目标 = 当前 `hit`（disabled 仍不发）。

**Canceled**（`PointerKind::Canceled`）：先 `click_cancelled = true`（§0 决策 6）。

**Click 判定**（Up/Canceled 都走，但 Canceled 因 `click_cancelled` 必返 None）——`click_test()` 仿 fgui `ClickTest()`：

```rust
fn click_test(slot: &TouchSlot, scene: &Scene, current_hit: Option<NodeId>) -> Option<NodeId> {
    if slot.click_cancelled { return None; }
    let t = click_threshold(slot.touch_id);
    let dx = slot.last_pos.0 - slot.down_pos.0;
    let dy = slot.last_pos.1 - slot.down_pos.1;
    if dx.abs() > t || dy.abs() > t { return None; }   // 位移超阈值 → 无 click
    // fgui：downTargets[0] 若仍有效→它；否则沿 current_hit 祖先找首个在 down_targets 内的存活祖先
    if let Some(&leaf) = slot.down_targets.first() {
        if leaf.0 < scene.nodes.len() { return Some(leaf); }   // "still on stage"≈索引有效
    }
    let mut cur = current_hit;
    while let Some(id) = cur {
        if id.0 >= scene.nodes.len() { break; }
        if slot.down_targets.contains(&id) { return Some(id); }
        cur = scene.nodes[id.0].parent;
    }
    None
}
```

**Click 目标 = `click_test()` 结果**（非 `hit`）。若 `Some(target)` 且 `!target.disabled` → 发 `EVT_CLICK`，`click_count` 由 §3.4 算。

> ⚠️ **行为变化**（v1c.3→v1c.4）：v1c.3 Click 目标 = 当前 hit 且要求 `down_node==hit`；v1c.4 目标 = `down_targets[0]`（按下叶），即使光标漂移到相邻元素（照 fgui「点按缩放」语义）。这是 fgui 对齐的核心差异，验收须覆盖。

### 3.4 clickCount 累进（双击，照 fgui `End()` @1745-1755）

发 Click 前（位移/cancel 通过后）：

```rust
// 注：time_s 作参数传入（避免 &mut self 与 &mut slot 借用冲突；实现在 PointerState 方法里
// 先读 self.time_s 再传，或用 slot_idx 索引分两次借）。
fn bump_click_count(slot: &mut TouchSlot, button: u8, time_s: f32) -> u8 {
    let t = click_threshold(slot.touch_id);
    let within_time = time_s - slot.last_click_time < DOUBLE_CLICK_TIME;
    let within_pos = (slot.last_pos.0 - slot.last_click_pos.0).abs() < t
                  && (slot.last_pos.1 - slot.last_click_pos.1).abs() < t;
    let same_button = slot.last_click_button == button;
    let count = if within_time && within_pos && same_button {
        if slot.click_count == 2 { 1 } else { slot.click_count + 1 }   // 1→2→1 循环
    } else { 1 };
    slot.click_count = count;
    slot.last_click_time = time_s;
    slot.last_click_pos = slot.last_pos;
    slot.last_click_button = button;
    count
}
```

`EVT_CLICK` 的 `click_count` = 返回值。fgui 注：touch 路径 fgui 用 Unity `tapCount` 覆盖；**LoomGUI 统一用核心 350ms 逻辑**（跨引擎一致，不依赖 Unity tapCount）。

Cancel 分支（`click_test` 返 None）：照 fgui `lastClickTime=0; clickCount=1`——即一次失败的 click 重置双击窗口。在 `click_test` 返 None 时执行 `slot.last_click_time = 0.0; slot.click_count = 1;`。

## §6 stopImmediatePropagation（纯 C#，core 不动）

`EventBridge.CallBubble`/`CallCapture` 从整块多播委托改为逐个调：

```csharp
public void CallBubble(EventContext ctx) {
    if (_bubble == null) return;
    foreach (EventCallback cb in _bubble.GetInvocationList()) {
        cb(ctx);
        if (ctx._stopsImmediatePropagation) break;   // 当前节点剩余监听器不再触发
    }
}
```

- `StopPropagation`：仅置 `_stopsPropagation` → `BubbleRoute` 节点循环 break（当前节点剩余仍跑）。
- `StopImmediatePropagation`：置两者 → CallBubble 逐回调 break（当前节点剩余也断）+ 节点循环 break。

`BubbleRoute` 已有 `if (ctx._stopsPropagation) break;` 覆盖节点间。capture 阶段同款逐回调 break（W3C 语义；fgui capture 不查 stop，LoomGUI 此处更标准）。

## §7 Stationary hover 跟随（core，fgui 局限 LoomGUI 改进）

现状：`last_hit` 仅事件到达时刷新；静止光标 + 元素动画移入其下 → 无事件 → hover 不更新。

改 `process()`：先算本帧事件占用哪些槽（`find_or_alloc_slot`），再对**未占用**的活跃槽（slot0 鼠标常驻 + `touch_id>=0` 触摸槽）`hit_test(last_pos)` 刷新 `last_hit` → `hover_diff_slot`。事件占用的槽走各自 handler 的 diff（不重复）。

```rust
pub fn process(&mut self, scene: &mut Scene, events: &[PointerEvent]) -> Vec<EventRecord> {
    let mut out = Vec::new();
    let used: Vec<usize> = events.iter().filter_map(|e| self.find_or_alloc_slot(e)).collect();
    // Stationary hover follow：未占用的活跃槽刷新命中 + hover diff
    for i in 0..self.slots.len() {
        if (i == 0 || self.slots[i].touch_id >= 0) && !used.contains(&i) {
            self.slots[i].last_hit = hit_test(scene, self.slots[i].last_pos);
            Self::hover_diff_slot(&mut self.slots[i], scene, &mut out);
        }
    }
    // … 现有事件循环（Down/Up/Move/Canceled，handler 内 hover_diff）…
    self.recompute_hovered(scene);
    self.recompute_active(scene);
    out
}
```

`ponytail:` fgui 依赖 Move 事件（无此改进）；LoomGUI 每帧每未占用活跃槽 +1 次 hit_test，鼠标+数指可忽略。

## §12 风险 / 坑预告

1. **`time_s` f32 精度**：运行千秒后 f32 精度 ~0.1ms，对 350ms 窗口无影响。若担心可换 f64（+4B/Stage，可接受）。默认 f32（ponytail）。
2. **C# `LoomEvent` clickCount offset**：`type`(1B@4) 后 `clickCount`(1B@5)，touch_id 仍 @8。Sequential layout 自动 2B 填充 @6-7，与 Rust `click_count:u8`+`pad:[u8;2]` 对齐。改 C# 镜像后必 EditMode 测验证 sizeof=20（坑 17 同款）。
3. **Canceled 偏离 fgui quirk**（§0.6）：文档化，验收明确「Canceled 不累加 clickCount」。
4. **`dt` 改 unscaledDeltaTime**：现 `Time.deltaTime` 被忽略，改值安全；但若 v1d+ 动画需 scaled dt，届时 tick 或需双参数。本版仅 click 用时，传 unscaledDeltaTime。
5. **Click 目标从 hit→down_leaf**（§3.3）：行为变化，旧测 `down_up_same_node_within_threshold_emits_click` 仍过（同节点），但新增漂移测覆盖差异。
