# v1.1 background-image 设计

> **定位**：v1.x 批 1 第 1 项。补 v1 围栏已声明（v1-scope §2）却零解析的 `background-image` / `background-size`——AI 写了被静默忽略（坑 57），违背 AI 可预测性首要判据。
> **依据**：Explore 调研结论（Mesh 变体已支持色+图共存）+ roadmap.md §1 批 1。

---

## 0. 目标 + 非目标

**目标**：
- 容器节点（div/button）支持 `background-image:url(x)` + `background-size:cover|contain|100%`。
- background-color + background-image 共存（CSS 标准：图透明区透出色）。
- url 走打包器图集（与 `<img>` 同一条 atlas 收集链）。

**非目标**（YAGNI）：
- span 文本节点背景图（Text 节点不改）。
- `background-position` / `background-repeat`（围栏砍）。
- 九宫格 `-l-slice`（批 2 独立属性）。
- background-image tween 通道（用节点 opacity 淡入足够）。
- data URI / 绝对路径（只支持 atlas 相对路径，同 img）。

---

## 1. 方案选择

**采用**：复用现有 `NodePayload::Mesh`（已带 `texture`+`uvs`+`colors`），Container/Button 分支条件产带图 quad。**零新 payload 变体 / 零新 shader / 零新 blob 列**。

**否决**：新建 `NodePayload::BackgroundImage` 变体——冗余。Mesh 已支持色+图共存（Container 用 `texture:0`+顶点色；Image 用 `texture:tex_id`+白顶点色+UV）。新变体要扩 blob SOA + MirrorPool 新分支 + shader 路径，全多余。

---

## 2. CSS 子集（围栏内已声明，本次兑现）

| 属性 | 值 | 语义 |
|---|---|---|
| `background-image` | `url(x)` / `url("x")` / `url('x')` | 容器背景图，x 为 atlas 相对路径（同 img src） |
| `background-size` | `cover` | 铺满裁剪（scale=max(rw/iw, rh/ih)，UV 居中取可见子区） |
| | `contain` | 完整放入留白（scale=min(rw/iw, rh/ih)，UV 居中 + 边距） |
| | `100%` | Stretch 拉伸填满（UV 0..1） |
| | （未设） | 默认 `Stretch`（UI 皮肤用例；围栏文档告知 AI "v1 默认非 CSS auto"） |

砍（围栏外）：`background-position`、`background-repeat`、`auto`、多背景层。

**围栏文档同步**：v1-scope §2 已列 background-image/background-size，本次实现兑现（不触发 docs/design 改动——属性已在围栏列表）。在 v1-scope §2.1 预览可信清单补一条：background-image Chrome 预览可信（标准 CSS）。

## 4. 解析（`loomgui_core/src/style/mapping.rs`）

`apply_decl` 追加两个 arm：

```rust
"background-image" => {
    style.background_image = parse_url(value);  // 提取 url(...) 内路径，去可选引号
    style.background_image.is_some()
}
"background-size" => {
    style.background_size = match value.trim() {
        "cover" => BackgroundSize::Cover,
        "contain" => BackgroundSize::Contain,
        "100%" => BackgroundSize::Stretch,
        _ => return false,  // 围栏外值（auto/px/两值）静默忽略，落 false
    };
    true
}
```

**`parse_url`**（mapping.rs 新增 helper）：
- 输入 `url(foo.png)` / `url("foo.png")` / `url('foo.png')`。
- 提取括号内字符串，去首尾配对引号（`"` 或 `'`）。
- 空内容 / 非 `url(...)` 格式 → `None`（落 false）。
- 不做 URL 解码（v1 路径无特殊字符）。

**无效值处理**：围栏外值（`auto`/`50px`/`100% 50%`/多背景）落 `_ => false` 静默忽略——与 v1 现有围栏外 CSS 一致（坑 57 同类）。

## 5. 打包器收集（`loomgui_pkg/src/lib.rs`）

现有 `pack_inner` 只收集 `NodeKind::Image { src }`。追加：遍历 `scene.nodes` 的 `style.background_image`，非 None 的 url 进同一 `srcs`（HashSet 去重，img + bg-image 同 url 只入一次）。

```rust
// 现有 Image src 收集后追加：
for n in &scene.nodes {
    if let Some(url) = &n.style.background_image {
        if seen.insert(url.as_str()) {
            srcs.push(url.clone());
        }
    }
}
```

- 解码/shelf_pack/blit/AtlasSection 流程**零改**——srcs 统一进同一条链。
- `AtlasSprite.src` = url 字符串，runtime `textures.get(url)` 命中（与 img 同一 TextureRegistry）。
- 缺图 → `image::open` Err → build-time fail（与 img 一致）。

**注意**：`scene.nodes` 的 style 字段在打包期已 cascade resolve（`resolve_styles` 在 `pack_inner` 顶部已调）——`background_image` 字段可读。

## 6. 渲染（`loomgui_core/src/render/mod.rs` Container/Button 分支）

现有 Container/Button 分支（render/mod.rs:125-129）产 `texture:0` 纯色 quad。改为：有 `background_image` 且 `textures.get(url)` 命中 → 产带图 quad。

```rust
NodeKind::Container | NodeKind::Button => {
    let color = anim.and_then(|a| a.bg_color)
        .unwrap_or(n.style.background_color.unwrap_or([0.0, 0.0, 0.0, 0.0]));
    let (texture, u_min, u_max) = match &n.style.background_image {
        Some(url) => match textures.get(url) {
            Some(m) => {
                let (a, b) = fit_uv(n.style.background_size, rect, m);
                (m.tex_id, a, b)
            }
            None => (0u32, [0.0, 0.0], [1.0, 1.0]),  // 未注册哨兵→白占位
        },
        None => (0u32, [0.0, 0.0], [1.0, 1.0]),  // 无图：UV 无意义
    };
    // v 翻转（同 Image 分支 render/mod.rs:139）
    let (v, uvc, col, idx) = crate::render::mesh::quad(
        rect, color, [u_min[0], u_max[1]], [u_max[0], u_min[1]],
    );
    rn.payload = NodePayload::Mesh {
        verts: v, uvs: uvc, colors: col, indices: idx,
        texture, program: 0,
    };
}
```

### 6.1 fit_uv 算法（render 层算 = 核心单一真相源）

cover/contain 用**统一 span 公式**，区别只在缩放比 s 取 max（cover）/ min（contain）：

```rust
/// 按 background-size 算 rect → atlas UV 映射。
/// rect=容器布局尺寸；meta=TexMeta（width/height 原始像素 + uv_min/uv_max atlas 子区）。
fn fit_uv(size: BackgroundSize, rect: Rect, meta: &TexMeta) -> ([f32;2], [f32;2]) {
    let (rw, rh) = (rect.w, rect.h);
    let (iw, ih) = (meta.width as f32, meta.height as f32);
    let (uv_min, uv_max) = (meta.uv_min, meta.uv_max);
    if iw <= 0.0 || ih <= 0.0 { return (uv_min, uv_max); }
    let (auw, auh) = (uv_max[0]-uv_min[0], uv_max[1]-uv_min[1]);  // 子区 UV 宽高
    let (cu, cv) = ((uv_min[0]+uv_max[0])/2.0, (uv_min[1]+uv_max[1])/2.0);  // 子区中心
    match size {
        BackgroundSize::Stretch => (uv_min, uv_max),  // 整子区拉伸填满
        BackgroundSize::Cover | BackgroundSize::Contain => {
            // s = 图缩放比：Cover 取 max（图≥容器，UV 内收取子区中央可见，裁掉图边缘）
            //                 Contain 取 min（图≤容器，UV 外扩，子区外采样→clamp 透明，容器边缘留白）
            let s = match size {
                BackgroundSize::Cover => (rw/iw).max(rh/ih),
                BackgroundSize::Contain => (rw/iw).min(rh/ih),
                _ => unreachable!(),
            };
            // rect 每轴 UV 跨度 = au × (容器尺寸 / 图缩放后尺寸)。
            // Cover: 跨度 ≤ au（内收）；Contain: 跨度 ≥ au（外扩）。
            let u_span = auw * rw / (iw * s);
            let v_span = auh * rh / (ih * s);
            ([cu - u_span/2.0, cv - v_span/2.0],
             [cu + u_span/2.0, cv + v_span/2.0])
        }
    }
}
```

- **Cover 内收**：UV 跨度 < 子区，取子区中央可见部分 → 图放大铺满容器，超出部分（图边缘）被裁。
- **Contain 外扩**：UV 跨度 > 子区，rect 边缘采样落到子区外 → shader clamp 到子区边缘像素。
  - 图边缘透明（icon 常见）→ 留白透明 → 透出 background-color。✅ 共存语义成立。
  - **风险**：atlas 无 padding，外扩可能采样到**邻近 sprite**（渗色）。shelf_pack 紧排无间隙。
  - **决策**：v1.1 接受（atlas sprite 边缘通常透明；showcase §3.6 验收观察）。若渗色明显，批 2 加 atlas padding（独立项，已知限制）。

### 6.2 共存语义

- background-color（`color` 变量）进 Mesh 顶点色（每顶点 [f32;4]）。
- background-image 进 `texture` + `uvs`。
- `LoomGUI/Unlit` shader 已 `mainTexture × vertexColor`：图透明区 tex alpha=0 → 结果 = 顶点色（透出 background-color）；图不透明区 → 顶点色 tint 图。
- **零 shader 改动**。

### 6.3 fallback

- url 未注册（`textures.get` None）→ `texture:0` → MirrorPool 用 `Texture2D.whiteTexture` → 白块 × 顶点色 = 纯色块（退化成 background-color 视觉）。可接受（打包缺图本就 build-time Err，runtime 几乎不 None）。

## 7. FFI / Unity 后端（零改）

- **blob**：Mesh arena 已带 `texture`(u32) + `uvs`（FrameBlob.cs:108-128）。无新列。
- **MirrorPool**：`tex_id → Texture2D` 查找已通（MirrorPool.cs:127-129）。Container texture 非 0 时自动走 atlas 纹理。
- **MaterialManager**：`Get(program:0, texture, maskCtx, matrixFlag)`——texture 非 0 时 key 含 atlas 纹理，与 Image 同批合路径。
- **shader**：`LoomGUI/Unlit` 已支持 `mainTexture × vertexColor`。
- **DrawState/batch**：带图 Container 的 texture 从 0 变非 0，DrawState key 变——但这是现有机制（Image 已如此）， FairyBatching 重排自然处理，无新逻辑。

## 9. 测试

### 9.1 Rust 单测（core）

- **mapping**（style/mapping.rs）：
  - `parse_url`：`url(x)` / `url("x")` / `url('x')` / `url( x )`（空格）→ `Some("x")`；`not-url` / `url()` / 空 → `None`。
  - background-size：`cover`/`contain`/`100%` → 对应枚举；`auto`/`50px`/`100% 50%` → `false`（忽略，不改默认）。
- **fit_uv**（render/mod.rs）：
  - cover：正方图 × 长方容器 → UV 居中裁剪，可见区 = 容器比例。
  - contain：正方图 × 长方容器 → UV 收缩，留白对称。
  - stretch：UV = (uv_min, uv_max) 恒等。
  - 退化：iw/ih=0 → 返回原 UV。
- **共存**（render）：节点同时设 background-color + background-image → Mesh payload `colors` 非 0（顶点色）+ `texture` 非 0。只设 color → texture=0。只设 image → colors 顶点色=[0,0,0,0]（默认透明，图独立显示）。
- **dirty**（style/dynamic.rs）：background_image/background_size 是视觉字段，改值不触发 layout dirty（同 background_color）。

### 9.2 打包器测试（loomgui_pkg）

- img + bg 同 url → atlas 只入一次（srcs 去重）。
- 纯 bg-image 包（无 img）→ atlas 正确生成 + AtlasSection 含该 src。
- 缺图 → `pack_inner` Err。

## 10. 已知限制 / 后续

- **contain 留白渗色风险**（§6.1）：atlas 无 padding，contain 留白采样可能渗邻图。v1.1 接受，showcase 观察；若明显，批 2 加 atlas padding。
- **默认 Stretch 偏离 CSS auto**：围栏文档须告知 AI（v1-scope §2.1 补一条）。
- **九宫格**（批 2 `-l-slice`）：独立属性，不进 background-size 枚举，v1.1 不预留。
- **background-image tween**：用节点 opacity 淡入；独立通道留 v2（YAGNI）。
