# v1.4-a 包加载模型重构（Package Loading）设计

> **状态**：设计定稿待 review
> **日期**：2026-07-02
> **前置**：v1.3+ 动态树重构已完成（代际 NodeId + slotmap + create_node/append_child/remove_node 等动态 API 已落地）。
> **依据**：brainstorm 会话（2026-07-02）+ fgui 包模型 / Unity prefab 调研 + 打包器/加载侧全量代码扫描。
> **关联**：v1.4 拆成 v1.4-a（本 spec，包加载重构）+ v1.4-b（虚拟化列表 + soft clip，待本 spec 落地后开）。本 spec 推翻 `v1x-deferred.md §1`（旧 `<l-list>` + 单树草稿）。

---

## 0. 为什么有这一项

LoomGUI 当前的包加载模型是 v1 static-tree 时代的简化，有三处根本性错误，v1.4-b（虚拟化列表）的 item 模板要外部包组件、真实游戏 UI 的多界面共存都会撞上它们：

1. **包 = 一整棵树**。当前 `pkg.bin` 是一棵扁平节点数组（`parent_idx` 串成树），无"组件"概念，不能按名取子树。一个包只能是一个完整界面，无法表达"一个系统含多个界面/组件"。
2. **load_package = 重建 scene**。`Stage::load_package`（`stage.rs:106`）直接 `self.scene = Some(scene)`，把加载和实例化耦合。切包 = 重建整棵 scene + 清 MirrorPool + 重订阅事件（showcase `OnDynLoadShowcase` 的代价）。无法"加载包当模板库、按需实例化某组件"，也无法多包/多界面共存。
3. **图集耦合在核心**。打包器解码 PNG + shelf 打图集 + 生成 atlas.png，核心存 `AtlasSection` + `TextureRegistry`，`Stage::load_package` 调 `build_registry`。核心不该知道图集——图集是后端（Unity）实现细节，核心只该知道"这个 img 用哪个路径的图"。

这三处是同一堵墙：**v1 把"加载"和"实例化"耦合、把"资产"和"场景"耦合**。本 spec 把这堵墙拆掉，建立 fgui / Unity prefab 式的干净模型：**包 = 资源池里的组件模板库；实例化 = 从模板克隆子树挂进场景**。地基打好后，v1.4-b 列表 item 模板 = `instantiate(外部包, "item")`，多界面共存 = 多次 instantiate 挂不同 layer。

**设计约束**（项目级，优先于工作量）：LoomGUI 是个人兴趣项目，**设计合理 > 工作量**。架构决策选最干净最严谨的方案，"改动大"不是反对理由。旧 pkg.bin 格式是错架构的产物，**不带兼容包袱，重打**（不做迁移器）。

## 1. 目标与非目标

### 目标

1. **包 = 系统 = 多 HTML 文件**。一个包含多个 HTML，每个 HTML 文件 = 一个具名组件（组件名 = 文件名去 `.html`）。
2. **load_package 只进资源池不建 scene**。包是模板库，`load_package(name, bytes)` 把包存进 `packages` 字典，scene 完全不变。
3. **instantiate 按名克隆组件**。`instantiate(pkg, component) -> NodeId` 从模板克隆一棵子树进当前 scene，返回组件根 NodeId（孤立，调用方挂载）。
4. **核心只知图片 path 不知图集**。打包器只扫 img src / bg-image url 归一化成相对 path，pkg.bin 只存 path；图集完全归 Unity（Sprite Atlas）。
5. **root_size 归 Stage**。设计分辨率是 stage/视口属性，移出包。
6. **Unity 包管理面板**。配置包↔html 映射、res 目录名、输出目录、一键打包、资源校验、智能识别。
7. **showcase 重做为真实游戏 UI 架构**。主界面 + 导航跳页 + 多组件包，借机验证全模型。

### 非目标（推迟）

- **虚拟化列表 / slot 复用 / soft clip**：v1.4-b，建在本 spec 的 instantiate 之上（item 模板 = instantiate）。
- **适配（layer 尺寸适配屏幕）**：用户明确"后面再做，不是这次"。本 spec 建 layer 骨架但不做适配逻辑。
- **runtime CSS 重解析 / class 增删触发整树 rematch**：沿用 v1.3+ 限制（set_style 只改单节点）。
- **旧 pkg.bin 兼容迁移**：不做迁移器，showcase 等现有 pkg 全部按新格式重打。
- **多语言 / 高清分支 / 异步加载 / 热更新**：v2。

## 2. 已定架构决策（brainstorm 确认）

| # | 决策 | 选择 | 关键理由 |
|---|---|---|---|
| D1 | 包的粒度 | 一个包 = 一个系统 = 多个 HTML 文件；每 HTML = 一个具名组件 | 对齐 fgui .fui 多组件 + 真实工程（一系统多界面）。推翻旧"单 HTML 单树" |
| D2 | 组件名规则 | 文件名去 `.html`，不支持 name 属性覆盖 | 文件系统即真相源，最可预测；覆盖致"文件名/组件名不一致" |
| D3 | load_package 语义 | 只进资源池（`packages` 字典），不建 scene | 加载≠实例化（fgui/Unity prefab 模型）；切包/多包不重建 scene |
| D4 | instantiate 语义 | 克隆子树进 scene，返回孤立根 NodeId（调用方 append_child 挂 layer） | 复用 v1.3+ append_child；同组件可挂不同位置 |
| D5 | 组件间静态引用 | **不支持**。组合靠运行时 instantiate + append_child | HTML 标准无文件包含机制；不引入 `<l-include>` 自定义标签（违背 AI 可预测性：AI 无先验 + Chromium 预览塌，同砍 `l-container`/`l-list` 理由） |
| D6 | CSS 作用域 | 每 HTML 自带 CSS（`<style>`/`<link>`/行内均可），打包器归一化 bake 进节点 style；**组件间 CSS 隔离** | CSS 全局命名空间，跨 HTML 共享有重名冲突（`.item` 污染）；scoped CSS 非标准不用 |
| D7 | 伪类规则存储 | 组件级存（ComponentTemplate 自带 dynamic_rules），instantiate 时合并去重进 scene 级规则表 | 组件自包含（模板带自己规则）；合并去重避免多实例重复规则堆积 |
| D8 | 图片管理 | 核心只记 path（res 目录前缀后的相对路径），图集完全归 Unity Sprite Atlas | 图集是后端实现细节；核心不知图集契约最干净；Unity 原生图集系统成熟 |
| D9 | root_size 归属 | 归 Stage，移出 pkg.bin header | 设计分辨率是 stage/视口属性，非包属性；多包共享 stage |
| D10 | res 目录名 | 可配置（编辑器面板项，默认 `res`），打包器按配置去前缀归一化 | 不同工程资源目录命名不同，写死不灵活 |
| D11 | 旧 pkg 兼容 | 不兼容，重打，无迁移器 | 旧格式是错架构产物，不背兼容包袱 |
| D12 | load_inline | **砍掉**。全走 package 路径，单一加载机制 | 双路径分叉是维护负担；inline 无独特价值（单测改用内存 pkg） |
| D13 | AI 工程位置 | 放 Unity 工程内（res 天然同一份），治理 .meta + AssetImporter | 解决 res 同步（零同步）；代价是 .meta 噪音，可治理 |
| D14 | Unity 图集系统 | Unity 原生 Sprite Atlas Asset，LoomGUI 不自建图集 | 复用 Unity 成熟图集系统；核心/打包器都不碰像素 |
| D15 | 打包器调用方式 | Unity 面板起 loomgui_pkg CLI 子进程 | 解耦（Unity 进程不跑 Rust 库），打包不频繁不在乎开销 |
| D16 | showcase 重构 | B 方案多组件（home + 各 page + tips），web 式多页（home 也是页，整层换） | 借机把 showcase 从 static-tree 样张升级真实游戏 UI 架构，验证全模型 |
| D17 | 图片尺寸归属 | **核心知图尺寸**（打包期 PNG IHDR 静态数据，存进 AssetManifest），**不知图集**（运行时纹理/UV 归 Unity） | "核心不知图集"≠"不知图尺寸"——aspect ratio（v1b.2 measure 三档）+ 九宫格 UV（v1.3 border-image-slice）都需图真实尺寸。尺寸是打包期静态数据（PNG IHDR 几字节），不依赖图集。T6 review 发现的 spec 漏洞补丁 |

## 3. pkg.bin 新格式（D1/D2/D6/D8/D9）

### 3.1 打包输入

一个**系统目录**，含多个 `.html` + 配套资源：

```
bag_system/                  ← 一个系统 = 一个包（拖这个目录到 Unity 面板）
  bag_panel.html             ← 组件 "bag_panel"（内联 <style> 或 <link> 引 css）
  bag_item.html              ← 组件 "bag_item"
  mail_row.html              ← 组件 "mail_row"
  tooltip.html               ← 组件 "tooltip"
  res/                       ← 资源目录（目录名可配置，默认 res）
    icons/
      skin.png
      coin.png
```

打包器扫目录顶层所有 `.html`（**不递归**子目录，嵌套的 html 由用户手动拖入包），每个 HTML 独立解析为一个组件。`res/` 固定识别为资源目录（排除出组件列表）。

### 3.2 包内二进制结构

```
Header: magic(u32) + version(u32) + flags(u32) + component_count(u32) + string_count(u32)
        （砍 root_w/root_h——root_size 归 Stage；砍 atlas 引用——图集归 Unity）
StringTable: 字符串 intern 去重（组件名 / text content / img path / classes / id_attr 共用一张表）
ComponentTable: 每组件 { name_idx(u16), root_node_idx(u32), dynamic_rules_blob_len(u32) }
NodeBlock: 所有组件的节点平铺，每节点 {parent_idx(i32), kind_tag(u8), style_blob, text_idx(u16), src_idx(u16),
           classes[], id_attr, flags(u8), tabindex(i32)}
           parent_idx 用 -1 表组件根（组件根无父，instantiate 时由调用方挂载）
           parent_idx 是 NodeBlock 内全局位置索引（跨组件靠 ComponentTable.root_node_idx 切分）
           src_idx 指向归一化后的 img path（不再是 atlas sprite）
PerComponentDynamicRules: 紧跟 ComponentTable 每项后存本组件 dynamic_rules 的 bincode blob
AssetManifest: 本包用到的所有 img path 列表（path_idx 数组 + count），供 Unity 面板校验 res 齐全
```

### 3.3 与旧格式的差异

| 项 | 旧格式 | 新格式 |
|---|---|---|
| 节点组织 | 单树（整包一棵） | 多组件子树（ComponentTable 切分） |
| root_size | header 带 root_w/root_h | 砍，归 Stage |
| AtlasSection | 有（atlases + sprites 坐标） | 砍（图集归 Unity） |
| img src | 指向 atlas sprite | 归一化 path 字符串 |
| 动态规则 | 全包共享一份 DynamicRuleSection | 分组件存（PerComponentDynamicRules） |
| AssetManifest | 无 | 新增（path + 图尺寸 w/h，给 Unity 校验 + 核心 measure/九宫格用） |
| version | 当前 PKG_FORMAT_VERSION | bump（旧包不兼容，重打） |

### 3.4 path 归一化规则 + 图尺寸（D8/D10/D17）

打包器扫 HTML 的 `img src` + `style.background_image url`，归一化：
- 输入：`res/icons/skin.png` / `./res/icons/skin.png` / `res\icons\skin.png`（Win 反斜杠）等。
- 输出：去掉配置的 res 目录前缀 + 统一正斜杠 → `icons/skin.png`。
- res 目录名取自打包配置（默认 `res`，可配置）。
- path 进 StringTable，节点 src_idx 指向它。pkg.bin 只存 path 字符串，不存像素/atlas 坐标。

**图尺寸（D17）**：打包器对每个 manifest 里的 path，读 PNG IHDR chunk（前 8 字节 magic + 13 字节 IHDR，取 width/height，**不引 image crate**——PNG header 解析几十行即可）。AssetManifest 存 `Vec<AssetEntry { path: String, w: u32, h: u32 }>`（path + 真实像素尺寸）。非 PNG 或读失败 → w/h=0（核心 measure fallback 64×64）。

**为何核心知尺寸**：measure 三档（`CSS > 真实像素 > 64×64`，v1b.2）+ 九宫格 UV（`slice_px / src_px`，v1.3）都需图真实尺寸。尺寸是打包期静态数据，不依赖运行时图集。核心知尺寸（静态）+ 不知图集（运行时纹理/UV 归 Unity）——两者正交。

### 3.5 CSS 归一化（D6）

打包器对每个 HTML：
1. 抽出本 HTML 所有 CSS：`<style>` 内联 + `<link rel="stylesheet" href>` 引用的外部 .css 文件内容 + 行内 `style=""`。
2. 合并成本组件的 stylesheet。
3. `resolve_styles` 把样式 bake 进每节点 `style_blob`（已 resolve 的 ResolvedStyle）。
4. `extract_dynamic_rules` 抽本组件伪类规则 → PerComponentDynamicRules 段。
5. pkg.bin **不存原始 CSS 选择器**，只存 resolve 后的 style_blob + 伪类规则。

**组件间 CSS 隔离**：每 HTML 的 CSS 只 resolve 本组件节点，不跨组件。`bag_panel.html` 的 `.item` 与 `mail_row.html` 的 `.item` 互不污染（各自 bake 进各自节点）。

## 4. 核心加载 API + 资源池（D3/D4/D7/D9/D12）

### 4.1 Stage 字段变更

```rust
pub struct Stage {
    pub scene: Option<Scene>,                          // 当前场景（初始空，create_root 建 layer 骨架）
    pub root_size: (f32, f32),                         // 归 Stage（移出包）
    pub packages: HashMap<String, Package>,            // 资源池：pkg_name → Package（多包共存）
    pub pointer_state: PointerState,                   // 不变
    pub tweens: TweenManager,                          // 不变
    // 砍：textures / atlases（图集归 Unity）
    // 砍：load_inline 相关
    // 其余不变
}

/// 一个已加载的包（资源池条目）。
pub struct Package {
    pub name: String,                                  // 包名（load_package 时传入）
    pub components: HashMap<String, ComponentTemplate>,// 组件名(文件名去.html) → 模板
    pub asset_manifest: Vec<String>,                   // 本包用到的图片 path（给 Unity 校验）
}

/// 一个组件的模板（instantiate 的克隆源）。
pub struct ComponentTemplate {
    pub name: String,                                  // 组件名
    pub nodes: Vec<TemplateNode>,                      // 子树节点平铺（parent_idx 串树，根 parent=None）
    pub dynamic_rules: DynamicRuleTable,               // 本组件伪类规则（instantiate 时合并进 scene）
}

/// 模板节点：序列化态（instantiate 时 build 成 live Node）。
/// 与 live Node 的区别：无 NodeId（instantiate 时 slotmap 分配）、无 taffy_id（每帧 solve 重建）。
pub struct TemplateNode {
    pub kind: NodeKind,
    pub style: ResolvedStyle,                          // base_style（已 bake，instantiate 后作 base_style + style 初始）
    pub parent_idx: Option<usize>,                     // 模板内位置索引（None=组件根）
    pub classes: Vec<String>,
    pub id_attr: Option<String>,
    pub draggable: bool,
    pub tabindex: Option<i32>,
}
```

### 4.2 API

```rust
impl Stage {
    /// 加载包进资源池（不碰 scene）。重复 load 同名包 = 替换。
    /// 解析 pkg.bin → Package，存进 self.packages[name]。
    pub fn load_package(&mut self, name: &str, bytes: &[u8]) -> Result<(), String>;

    /// 从包克隆一个组件进当前 scene，返回组件根 NodeId（孤立，无父，调用方 append_child 挂载）。
    /// 复用 v1.3+ create_node 逻辑逐节点建树，按 parent_idx 串子树。
    /// 伪类规则：合并本组件 dynamic_rules 进 scene.dynamic_rules（去重，相同选择器不重复加）。
    pub fn instantiate(&mut self, pkg: &str, component: &str) -> Result<NodeId, String>;

    // create_root / create_node / append_child / remove_node / set_text / set_src / set_style
    // 沿用 v1.3+（不变）。scene 初始由 create_root 建，load_package 不再建 scene。
}
```

### 4.3 典型流程（游戏 UI 启动）

```rust
let mut stage = Stage::new(font, root_size);           // scene = None

// 1. 建层级骨架（v1.3+ create_root/create_node/append_child）
let root = stage.create_root("div", "width:1080px;height:1920px");
let ui_layer = stage.create_node("div", "...");         // 主界面层
stage.append_child(root, ui_layer);
let tips_layer = stage.create_node("div", "...");       // tips 层（在上）
stage.append_child(root, tips_layer);

// 2. 加载包进资源池（scene 不变，仍只有 layer 骨架）
stage.load_package("bag", bag_pkg_bytes)?;              // packages["bag"] = {bag_panel, bag_item, ...}
stage.load_package("mail", mail_pkg_bytes)?;            // 多包共存

// 3. 克隆组件挂 layer
let panel = stage.instantiate("bag", "bag_panel")?;     // 克隆子树 → 孤立根 NodeId
stage.append_child(ui_layer, panel);                    // 挂主界面层

// 4. 同组件多实例（背包网格里多个 item）
let grid = stage.find_node_by_id("bag-grid").unwrap();
for _ in 0..10 {
    let item = stage.instantiate("bag", "bag_item")?;
    stage.append_child(grid, item);
}

// 5. 叠加（tips 永远在上）
let toast = stage.instantiate("bag", "tooltip")?;
stage.append_child(tips_layer, toast);
// ... 定时 stage.remove_node(toast) 摘除
```

### 4.4 instantiate 内部实现要点

1. 查 `self.packages[pkg].components[component]`，拿 ComponentTemplate。
2. 遍历 `template.nodes`，逐个调 v1.3+ 的 `create_node` 逻辑建 live Node（slotmap 分配 NodeId），按 `parent_idx` 串子树（父先建于子，复用已分配的 NodeId）。
3. 组件根 `parent_idx=None` → 不挂任何父，记录根 NodeId 返回。
4. 节点 `style`（base_style）= template 的 baked style；`style`（派生）初始 = base_style，下帧 rematch 填。
5. 伪类规则：遍历 `template.dynamic_rules`，逐条检查 `scene.dynamic_rules` 是否已有相同选择器，无则加入（去重）。规则按 class 匹配，多实例共享同一套 class 规则，hit_test 返回具体 NodeId → 每实例独立命中 `:hover`，不串。
6. id_attr 冲突处理：若实例化的组件 id_attr 与 scene 现有节点 id 冲突，`find_node_by_id` 返回首个匹配——**约定：组件内 id 在单实例下唯一，多实例时业务不应依赖 id_attr 定位**（用返回的 NodeId）。spec 标注此限制，不在核心做 id 去重（YAGNI）。

### 4.5 与当前加载链路的断点

| 当前（要改/砍） | 新 |
|---|---|
| `Stage::load_package` → `self.scene = Some(scene)` | → `self.packages[name] = parsed Package`（不碰 scene） |
| `Stage::load_package` 调 `build_registry` 建 TextureRegistry | 砍（图集归 Unity） |
| `Stage.textures` / `Stage.atlases` | 砍 |
| `Stage::load_inline` | 砍（D12） |
| `read_package` 返 `(Scene, root_size, AtlasSection)` | 返 `Package`（含 components + manifest） |
| `write_package(scene, root_size, atlas, dynamic)` | `write_package(PackageInput)` 多 HTML → 多组件包 |
| Unity `LoomStage.LoadAtlas()` 读 atlas.png → `_texMap[tex_id]` | 改 path → Sprite 查询（Sprite Atlas） |
| Unity `LoadPackageFile` 重建 scene + 清 MirrorPool | 砍（不再切包重建；改为业务侧 instantiate/remove 管界面） |

## 5. Image path 传递 + Unity Sprite 查询（D8/D14/D17）

### 5.1 核心→Unity 契约

**核心只知 path + 图尺寸，不知图集/纹理/UV**。
- **path**：Image 节点持有归一化 path 字符串（如 `icons/skin.png`），render 时把 path 推给 Unity。Unity 按 path 查 Sprite Atlas 拿 Sprite（含 UV + Texture）。
- **图尺寸**（D17）：打包期 PNG IHDR 读出的真实像素 w/h，存进 pkg.bin AssetManifest。核心 `Stage` 持有当前包的尺寸表（`load_package` 时从 Package.asset_manifest 建 `path → (w,h)` 查询表）。`layout::solve` 的 Image measure 用此尺寸算 intrinsic（v1b.2 三档：`CSS > 真实像素(path 尺寸表) > 64×64`）。`render` 的九宫格 UV 用此尺寸算 `slice_px / src_px`（v1.3）。**尺寸不依赖图集**——核心不查纹理、不知 atlas，只查打包期静态尺寸表。

```
核心 render：Image 节点 RenderNode payload 带 path_idx（指向 FrameBlob 字符串表的 path）
                ↓ FrameBlob SOA 传递
Unity MirrorPool：收 path_idx → 取 path 字符串 → 查 path→Sprite → 设 Sprite（UV+Texture）

核心 measure/九宫格：查 Stage 持有的 path→(w,h) 尺寸表（打包期静态，不查 Unity）
```

### 5.2 FrameBlob 传递

path 是变长字符串，进 FrameBlob 字符串表（已有 StringTable 机制），Image 节点行带 `path_idx`。Unity 读 blob 按 path_idx 取 path 字符串。**每帧 path 都在 payload**（hash 比较决定是否重传，沿用 v1 既有 blob hash 增量机制）。path 查 Sprite 在 Unity 侧有 `Dictionary<string, Sprite>` 缓存，开销低。

**图尺寸不进 FrameBlob**：尺寸是核心内部用的静态数据（measure/九宫格），Unity 不需要（Unity 查到 Sprite 自带 textureRect）。尺寸表留核心 Stage 侧。

### 5.3 Unity Sprite Atlas 查询机制

- 开发者用 Unity 原生 Sprite Atlas Asset 系统：建 SpriteAtlas，把 `res/` 下的 Sprite 划进去。
- Unity 侧维护 `Dictionary<string, Sprite>` path→Sprite 缓存（启动时从已注册 SpriteAtlas 扫描填充，或运行时 `SpriteAtlas.GetSprite` 懒查）。
- **多图集**：可有多个 SpriteAtlas，path 路由到对应 atlas 是 Unity 内部事（核心不感知）。Unity 侧用 `SpriteAtlasManager` 或遍历注册的 atlas 查。
- **bg-image 同走 path**：`background-image: url(icons/bg.png)` 同样归一化 path，render 时 Unity 同样查 Sprite 给 UV（与 img 同机制）。
- **path 查不到 Sprite**：Unity fallback（missing texture，如紫色占位），不崩。核心不报错（它不知图集存在）。

### 5.4 与当前 Unity 侧断点

| 当前（要改/砍） | 新 |
|---|---|
| `LoomStage.LoadAtlas()` collect atlas_count/info + load atlas.png → `_texMap[tex_id]` | 砍，改 path→Sprite 缓存查询 |
| `_texMap[tex_id]` MirrorPool 按 tex_id 取 Texture | 改 MirrorPool 按 path 取 Sprite（UV+Texture） |
| blob Image 节点带 tex_id + UV（atlas 坐标） | 改带 path_idx（path 字符串表索引） |
| `Stage::load_package` 返 AtlasSection 给 Unity collect | 砍，Unity 不再 collect atlas |
| `LoomStage._usePackage` / `_pkgFile` 单包切换 | 改多包 load_package(name) + instantiate |

## 7. showcase 重构（D16，验收 demo）

### 7.1 重构目标

当前 showcase 是 static-tree 单页 46 卡长滚动（v1 时代 `load_inline`/单 pkg 一把梭的产物）。借 v1.4-a 之机重做成真实游戏 UI 架构：**主界面 + 导航跳页 + 多组件包**。同时作为 v1.4-a 全模型验收。

### 7.2 包结构（B 方案多组件）

```
showcase/                       ← showcase 包（多组件）
  home.html                     ← 主界面页：标题 + 导航按钮（各功能页入口）
  page_controls.html            ← 控件页：按钮/开关等
  page_text.html                ← 文本页：富文本/字体
  page_image.html               ← 图片页：bg-image/九宫格/ColorFilter
  page_scroll.html              ← 滚动页：ScrollPane/滚动条/滚轮
  page_tween.html               ← 动效页：tween/ease/delay/complete
  page_interact.html            ← 交互页：事件/灯阵/路由/焦点/键盘
  page_dyntree.html             ← 动态树页：建/删/批量/set_style（原 §3.10）
  tips_toast.html               ← tips 组件（演示 tips_layer 叠加）
  res/
    icons/...
```

每页内容从当前 showcase 单页对应 section 抽出（§1-§9 拆成各 page.html）。CSS 各页自带（`<style>`，组件间隔离）。

### 7.3 导航模型（模型 2：web 式多页）

```
Stage 启动
 ├─ create_root 建根
 ├─ 建 ui_layer（主界面层）+ tips_layer（tips 层，在上）
 ├─ load_package("showcase", showcase_pkg_bytes)
 ├─ home = instantiate("showcase", "home") → append_child(ui_layer)   // 首页

点导航按钮（如"动效"）：
 ├─ remove_node(current_page)          // 摘当前页（首次是 home；listener 联动清）
 ├─ current_page = instantiate("showcase", "page_tween") → append_child(ui_layer)  // 新页占满 ui_layer

页内"返回"按钮：
 ├─ remove_node(current_page)
 ├─ current_page = instantiate("showcase", "home") → append_child(ui_layer)

tips 演示（某操作触发）：
 ├─ toast = instantiate("showcase", "tips_toast") → append_child(tips_layer)  // 叠在最上
 ├─ 定时 remove_node(toast)                                            // 摘除
```

- home 也是页（ui_layer 整层换），点导航 remove 当前页 + instantiate 目标页。
- tips 演示 tips_layer 叠加（永远在所有页之上）。
- 各 page 内的交互演示（灯阵订阅 / tween 播放 / 动态树按钮）由 C# driver 在 instantiate 后重新订阅 listener（同 v1.3+ OnDynLoadShowcase 重订阅思路，但不再切包重建 scene）。

### 7.4 C# driver 改造

`LoomShowcaseDriver.cs` 重写：
- 不再 `LoadPackageFile` 切包。
- `Start`：建 layer 骨架 + `load_package` + `instantiate("home")` + 订阅 home 导航按钮。
- 导航回调：`remove_node(current)` + `instantiate(目标页)` + 订阅目标页内按钮。
- 各页订阅逻辑拆成 `SubscribeHome/SubscribeControls/.../SubscribeDynTree` 方法，instantiate 后按页调。
- 事件 listener 随页 remove 联动清（v1.3+ remove_node 已清 anim/scroll/tween/focused_node；事件 listener 在 C# EventHandler 侧，remove 前需显式清对应节点的 listener——**新增约定：driver 维护"当前页 listener 注册表"，切页时批量清**）。

### 7.5 driver listener 清理约定（新增）

v1.3+ remove_node 清了核心侧附属状态，但 **C# EventHandler 的 listener 不在核心**——它按 NodeId 存在 `_listeners` 字典。remove_node 后旧 NodeId 失效，listener 成悬空条目（不崩，但堆积）。driver 侧需：
- 维护 `Dictionary<uint pageRoot, List<(node,type,cb)>>` 当前页注册的 listener。
- 切页（remove_node 当前页）前，遍历该页 listener 表逐个 `RemoveListener`。

EventHandler.Clear() 已存在（切包用），但太粗（清所有）。新约定用细粒度按页清。**这个清理机制是 v1.4-a driver 侧新增的小工作**。

## 8. 验收

### 8.1 核心单测（loomgui_core）

**包格式**：
- `write_package` 多 HTML 输入 → pkg.bin 含正确 component_count + ComponentTable + 分组件 dynamic_rules。
- `read_package` 反解 → Package 含所有组件，组件名=文件名，每组件 nodes 子树 parent 关系正确。

- AssetManifest 列出所有 img path。
- 旧格式 pkg（旧 version）read → 版本协商报错（不兼容）。

**load_package / instantiate**：
- `load_package(name, bytes)` → `packages[name]` 存在，scene 不变（仍是空/骨架）。

- id_attr 多实例：find_node_by_id 返首个匹配（约定限制，不报错）。

**Image path**：
- Image 节点 RenderNode payload 带 path（非 tex_id/UV）。
- bg-image 同样带 path。

- `Stage.textures`/`atlases` 字段移除，编译通过。

**v1 零回归**（关键：渲染等价）：

## 9. 风险与不变量

### 9.1 风险

| 风险 | 缓解 |
|---|---|
| 打包器大改（多 HTML 输入 + 砍 atlas + path 归一化）面广 | 分 task：先 pkg 格式 read/write + ComponentTable，再 path 归一化，再砍 atlas/image crate |
| Unity Sprite Atlas 查询机制首次做，path→Sprite 缓存策略要调 | 先最简（启动全扫填充 Dictionary），性能问题再优化；缺图 fallback 兜底 |
| showcase 8+ 页 HTML 重写 + driver 重写工作量大 | 借机升级架构，值得；按页拆 task 逐个迁 |
| driver listener 清理新约定（按页清）易漏 | 单测/PlayMode 验：切页后旧页 listener 不触发（用灯阵反馈验证） |
| pkg.bin version bump 致所有旧 pkg 失效 | 预期（D11 不兼容），showcase 重打；本机打包脚本同步更新 |
| 多包同名/组件名冲突（两包都有 "item"） | instantiate(pkg, comp) 按 pkg 限定，不跨包冲突；同包内重名文件=文件系统不允许 |
| instantiate 后 id_attr 多实例冲突 | 约定限制（§4.4-6），文档标注，不做核心 id 去重 |
| FrameBlob 传 path 字符串（变长）vs 旧 tex_id（定长）改 SOA 布局 | path 进 StringTable + path_idx，SOA 行带 idx 不带变长串，布局兼容 |

### 9.2 核心不变量（final review 核对清单）

1. **load_package 不碰 scene**：调后 scene 不变，只 packages 字典变。
2. **instantiate 返回孤立根**：根 parent=None，调用方挂载；复用 v1.3+ create_node + append_child。
3. **核心不知图集**：payload 带 path，无 atlas/tex_id/UV 概念。图集是 Unity 内部。
4. **包=多组件**：一个 pkg.bin 含 N 个具名组件，组件名=文件名。
5. **CSS 组件隔离**：每 HTML CSS 只 resolve 本组件，跨组件不污染。
6. **伪类规则组件级 + 合并去重**：ComponentTemplate 自带，instantiate 合并进 scene 不重复堆积。
7. **root_size 归 Stage**：pkg.bin 不带 root_size。
8. **旧 pkg 不兼容**：旧 version read 报错，重打。
9. **remove_node 联动清**（v1.3+ 不变量保留）：anim/scroll/tween/focused_node 清；driver 侧 listener 按页清。
10. **多实例独立**：同组件多次 instantiate，节点/事件/伪类各实例独立不串。
